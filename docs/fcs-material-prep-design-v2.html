1. 业务背景与设计目标

1.1 背景

印尼供应链团队中分出一个专门的配料小组，统一负责所有物料的配料操作。 配料小组按照生产单中的 BOM（物料清单），从各仓库中挑出物料、拣货、放入暂存区、确认配料完成，然后通知下游来领料或配送。

核心业务规则：先配料，再分配任务。配料阶段只跟生产单的 BOM 走，不绑定具体工厂； 配料完成后再进行任务分配——确定由哪个工厂承接该工序。

1.2 设计目标

  1. 在 FCS 下建立独立的配料管理菜单，取代原先散落在各处的配料入口
  2. 按配料对象拆分为 5 个独立视图 + 1 个综合看板
  3. 实现「先配料、再分配」的核心业务顺序
  4. 引入完整的配料生命周期：创建 → 拣货 → 入暂存区 → 确认完成 → 通知下游
  5. 车缝配料受裁片放行数量约束

配料管理是 FCS 下的一个独立菜单模块。向上承接生产单的 BOM，向下驱动下游工厂的领料与配送。

4. 五大配料类型与物料路由规则

4.1 分类定义

生产单 BOM 展开后，每条物料行根据物料类型（面料/辅料/纱线/包材）和上游工序（是否需要印花/染色）自动路由到对应的配料类型。

配料类型物料筛选条件配货仓库下一步动作工厂确定性
染色配料面料 + 需要染色加工染色厂待加工仓染色厂加工领料工艺绑定
印花配料面料 + 需要印花加工印花厂待加工仓印花厂加工领料工艺绑定
裁片配料面料 + 不需要印花/染色中转仓裁片工厂领料工艺绑定
车缝配料辅料 / 纱线辅料仓 / 纱线仓配送至车缝厂分配决定
其他配料包材包材仓配送至承接厂分配决定
工厂确定性说明： 染色/印花/裁片是工艺绑定的——面料只能到对应类型的工厂加工，配料时就能确定目标工厂类型。 车缝/其他配料的目标工厂依赖于任务分配结果，配料时标记为"待分配"，不预先确定配送目的地。

6. 配料记录生命周期（5 态状态机）

6.1 配料单与配料记录的关系

配料单是一个容器，按生产单组织，一个生产单对应一张配料单（每种配料类型）。 配料记录是实际操作对象，一张配料单可以有多个配料记录（分批次操作），每条配料记录独立走完自己的生命周期。

举例：裁片配料单有 3 条物料行（主身面料、里布、拼接面料），配料小组分两批操作：

批次配料记录覆盖物料
第一批配料记录 #1主身面料 200 yard + 里布 150 yard
第二批配料记录 #2拼接面料 80 yard（等印花完成才可配）

8. 领料单状态机（5 态）

配料确认后，下游工厂可以进行领料。领料也有自己的状态流转：

stateDiagram-v2 NOT_PICKABLE : 暂不可领 WAIT_PICKUP : 待领料 REJECTED_WAIT : 打回待仓库处理 PICKUP_DONE : 已领料完结 ACTUAL_CLOSED : 按实完结 NOT_PICKABLE --> WAIT_PICKUP : 配料记录确认 WAIT_PICKUP --> PICKUP_DONE : 下游全部领完 WAIT_PICKUP --> REJECTED_WAIT : 下游打回 REJECTED_WAIT --> WAIT_PICKUP : 仓库重新确认 WAIT_PICKUP --> ACTUAL_CLOSED : 配料单关闭
领料规则：只有"已确认"状态的配料记录才能被领料。领料按整条记录一次性进行，不按物料行零散领取。 领料后物料进入工厂待加工仓，同时写入物料台账。

9. 配料数量计算规则

9.1 核心公式

染色配料 / 印花配料 / 裁片配料

最大可配数量 = min(生产单物料需求量中尚未配齐的部分, 对应仓库的可用库存)

默认配料数量 = 最大可配数量

实际配料数量 ≤ 最大可配数量（由配料操作员在新增记录时确认）

这三种配料类型不受裁片放行约束。

车缝配料

最大可配数量 = min(生产单物料需求量中尚未配齐的部分, 裁片放行件数 × BOM 单品用量, 对应仓库的可用库存)

默认配料数量 = 最大可配数量

受裁片放行数量 + 库存数量的双重约束。

其他配料（包材配料）

最大可配数量 = min(生产单物料需求量中尚未配齐的部分, 对应仓库的可用库存)

不受裁片放行约束。

9.2 车缝配料计算示例

假设生产单需要 500 条拉链（500 件 × 1 条/件）：

场景裁片放行仓库库存最大可配限制因素
A可以做 500 件600 条（充足）500 条BOM 需求（满额）
B部分可以做 320 件600 条（充足）320 条裁片放行不足
C部分可以做 320 件300 条（不足）300 条库存不足
D可以做 500 件0 条（无库存）0 条无库存

10. 车缝配料与裁片放行约束

10.1 业务场景

裁床完成铺布裁剪后，裁床主管在「裁片放行管理」中判断每个生产单能做到多少件。 这个判断直接影响车缝配料的数量上限——如果裁床说"只能做 320 件"，那辅料也只配 320 件的量，配多了用不上。

10.2 数据依赖链路

10.3 约束卡片的展示

车缝配料页面在物料明细表格上方展示一张"裁片放行约束卡片"：

  • 如果该生产单已有裁片放行判断：展示判断结果（可以做 / 部分可以做 / 暂时不能做）、可做件数、判断原因
  • 如果该生产单尚未形成裁片放行判断：展示黄色提示"尚未形成裁片放行判断，车缝配料数量暂以 BOM 需求为准"

11. 暂存区管理

11.1 概念

暂存区是仓库内的一个指定物理区域。配料完成后物料放在暂存区等待下游来取。

配料类型配货仓库暂存区位置通知对象
染色配料染色厂待加工仓染色厂暂存区染色厂
印花配料印花厂待加工仓印花厂暂存区印花厂
裁片配料中转仓中转仓暂存区裁片工厂
车缝配料辅料仓 / 纱线仓相应暂存区任务分配后确定
其他配料包材仓包材仓暂存区任务分配后确定

11.2 暂存区操作

跟单人员执行"确认入暂存区"操作时，系统记录：暂存区域名称（操作人输入）、操作人、操作时间、物料行数量、配料总数量、物料来源仓库列表。

暂存记录有三种状态：暂存中 → 已确认（配料完成时自动切换）或 暂存中 → 已打回（下游打回时自动切换）。

11.3 暂存区业务规则

  • 一个仓库可以有一个或多个暂存区
  • 不同配料类型可以共用或分用暂存区
  • 放入暂存区后，物料不能再被其他配料单占用
  • 领料完成后暂存区清空
  • 每次入暂存区都生成独立的暂存记录，完整保留审计追溯

12. 配料完成时的通知机制

配料小组确认配料完成后，系统根据配料类型生成不同的事件通知：

配料类型事件类型通知对象通知目的
裁片配料中转仓配料完成通知裁片工厂通知来中转仓领料
染色配料染色厂配料完成通知染色厂通知来加工领料
印花配料印花厂配料完成通知印花厂通知来加工领料
车缝配料配料完成通知不直接通知等待任务分配后配送
其他配料配料完成通知不直接通知等待任务分配后配送

通知事件写入运行时事件账本，下游各工厂模块订阅对应事件类型展示待领料列表。

14.1 菜单结构

工厂生产协同系统（FCS）

17. 数据存储与缓存策略

17.1 存储方式

配料管理的数据以键值对形式存储在浏览器本地存储中，包含：所有配料单的种子数据及其物料行、所有配料记录（含拣货人、暂存区、确认人等信息）、所有领料记录、所有打回记录、所有暂存区记录、已关闭的配料单列表。

17.2 数据投影

页面展示的"配料单投影"是一个聚合视图，每次读取时从本地存储水合原始数据，再结合运行时任务列表、运行时事件账本、物料台账等外部数据构建而成。写入操作后立即持久化并触发页面重渲染。

18. 关键业务规则汇总

配料顺序 ⭐

先配料，再分配。配料阶段不绑定具体工厂（车缝/包材），等配料完成后触发任务分配，分配结果回写配料记录。

五态流转 ⭐

每条配料记录必须按 DRAFT → PICKED → STAGED → CONFIRMED 顺序流转。不能跳过状态。REJECTED 状态可以从 CONFIRMED 跳回 STAGED。

整条确认

配料记录的确认对象是整条记录，不是逐行确认。一条配料记录内的所有物料行共享同一个状态。

领料前置条件

只有"已确认"状态的配料记录才能被领料。DRAFT/PICKED/STAGED/REJECTED 状态的记录均不可领料。

工艺绑定 vs 分配决定

染色/印花/裁片任务 → 工厂类型确定，由系统自动匹配；车缝/包装任务 → 工厂待确定，由任务分配模块确定后回写。

车缝双重约束

车缝配料的最大可配数量 = min(未配齐需求, 裁片放行件数 × BOM用量, 仓库库存)。裁片放行件数由裁床主管在裁片放行管理中判断得出。

暂存区管理

每次入暂存区操作生成独立的暂存记录。暂存记录有三种状态：暂存中 → 已确认（配料确认时自动切换）或 暂存中 → 已打回（下游打回时自动切换）。

关闭配料单

关闭配料单时必须填写关闭原因。关闭后该配料单的所有物料不再安排后续配料。关闭前需确认未配齐物料的缺口情况。

事件通知区分

配料确认时，根据配料类型生成不同的事件：裁片→中转仓配料完成通知，染色→染色厂配料完成通知，印花→印花厂配料完成通知，车缝/其他→配料完成通知。
