# FCS 三端工艺链路最终检查说明

## 三端联动最终验收范围

本轮验收覆盖工厂生产协同系统、工艺工厂运营系统、工厂端移动应用三端，重点检查印花、染色、裁片、特殊工艺在列表、详情、状态操作、移动端搜索、统一写回、仓交出回写、平台结果视图和统计中的一致性。

验收只针对原型内的本地 mock 与前端事实源，不新增后端、服务端、数据库，不重做 UI，不改变既有状态机、绑定规则、移动端可见性规则、平台聚合状态规则和打印服务架构。

## 中文流程图

```text
生产单
-> 生成印花染色裁片特殊工艺任务
-> 工艺工厂Web查看和操作
-> 工厂端移动应用查看和操作
-> 共用写回函数
-> 生成仓记录交出记录审核记录差异记录
-> 平台结果视图读取统一事实源
-> 平台侧展示聚合状态风险和跟单动作
-> 统计读取统一事实源
-> 全链路检查脚本和Playwright验证
```

## 中文状态机

```text
待验证
-> 前置脚本校验
-> 数据完整性校验
-> 三端页面校验
-> 状态操作校验
-> 仓交出回写校验
-> 平台结果校验
-> 统计联动校验
-> 最终通过
```

异常流转：

```text
任一校验失败
-> 定位失败模块
-> 修复遗漏
-> 重新执行全部校验
-> 最终通过
```

## 印花全链路验收路径

印花验收以 `PH-20260328-001` 和可操作印花单为核心：

- 工艺工厂 Web 列表与详情显示 `全能力测试工厂（F090）`。
- 详情显示绑定有效移动端任务，Web 可打开移动端执行页。
- 移动端执行列表可通过加工单号、任务号、生产单号、`F090` 和 `全能力测试工厂` 检索到同一任务。
- Web 或移动端执行 `开始打印`、`完成转印`、`发起交出` 后，统一写回服务生成操作记录。
- 完成转印或待送货后进入印花待交出仓。
- 发起交出后生成统一交出记录。
- 回写一致不生成差异记录，回写不一致生成差异记录。
- 平台侧只展示聚合状态、风险提示、下一步动作和当前责任方，主状态不直接展示 `打印中`、`转印中` 等内部细节点。
- 数量字段根据对象显示面料米数或裁片数量，不只写“数量”。

## 染色全链路验收路径

染色验收以完成包装、发起交出、回写差异为核心：

- 工艺工厂 Web 列表与详情显示 `全能力测试工厂（F090）`。
- 染色加工单绑定有效移动端任务，Web 直达和移动端列表搜索一致。
- Web 或移动端执行 `排染缸`、`开始染色`、`完成包装` 等合法动作时共用 `executeProcessAction`。
- 完成包装后生成染色待交出仓记录。
- 发起交出后生成统一交出记录。
- 回写不一致生成差异记录和审核记录。
- 平台侧展示 `待送货`、`待回写`、`异常` 等聚合状态，不直接展示 `染色中`、`烘干中`、`包装中` 作为主状态。
- 染色数量使用面料米数和卷数，页面文案使用“染色统计”，不出现“染色报表”。

## 裁片全链路验收路径

裁片验收重点是原始裁片单、合并裁剪批次、菲票归属和交出差异：

- 裁片任务或单据绑定有效移动端任务。
- 工艺工厂 Web 与移动端读取同一状态和同一操作记录。
- 执行 `开始铺布`、`完成裁剪`、`生成菲票`、`确认入裁片仓` 等合法动作。
- 确认入裁片仓后生成裁片待交出仓记录。
- 发起交出后交出记录关联原始裁片单和菲票。
- 回写差异可追溯菲票。
- 平台侧任务进度看板展示平台聚合状态、裁片数量、菲票数量、交出记录和差异记录。
- 裁片冻结口径保持不变：生产单来源唯一为生产需求单，配料、领料、裁剪安排不得拆分或合并生产单；可裁判断发生在原始裁片单层；合并只发生在裁片执行层；合并裁剪批次只作为执行上下文；菲票永远回落原始裁片单。

## 特殊工艺全链路验收路径

特殊工艺验收覆盖打揽、打条、捆条、烫画、直喷、激光切、洗水：

- 特殊工艺任务单绑定有效移动端任务，工艺类型一致。
- Web 或移动端执行 `确认接收裁片` 后生成待加工仓记录。
- 执行 `开始加工`、`完成加工` 后生成待交出仓记录。
- 发起交出后生成统一交出记录。
- 上报差异后生成差异记录并关联菲票，记录当前裁片数量、累计报废裁片数量或累计货损裁片数量。
- 平台侧展示特殊工艺名称、裁片数量、菲票数量、差异记录和处理差异动作。
- 特殊工艺不得新增开扣眼、装扣子、熨烫、包装作为动作或展示维度。

## 最终总检查脚本说明

最终脚本为 `scripts/check-three-end-process-chain-final.ts`。

该脚本先执行第 1-9 步检查脚本，然后继续做跨模块数据检查：

- F090 工厂一致性。
- 加工单与移动端任务绑定。
- 移动端列表与直达详情一致。
- 平台聚合状态集合与细状态隔离。
- Web 状态操作与移动端共用写回。
- 数量对象和单位。
- 待加工仓、待交出仓、交出、回写、审核、差异联动。
- 平台结果视图读取统一事实源。
- 统计联动读取统一事实源。
- 裁片冻结口径和特殊工艺边界。
- 打印服务没有被本轮改动破坏。

## Playwright 覆盖说明

全链路 Playwright 文件为 `tests/fcs-three-end-process-chain.spec.ts`。

覆盖场景包括：

- 印花三端完整链路。
- 印花差异链路。
- 染色三端完整链路。
- 染色差异链路。
- 裁片三端完整链路。
- 裁片差异链路。
- 特殊工艺三端完整链路。
- 特殊工艺差异链路。
- 不可执行任务不能绕过。
- 统计联动。

测试不是只检查页面是否打开，而是检查工厂、任务号、绑定状态、移动端搜索、平台聚合状态、风险提示、下一步动作、仓记录、交出记录、差异记录、裁片数量、菲票数量等关键业务字段。

## 不可执行任务校验说明

报价、待定标、未接单、工厂不一致、工艺类型不一致的任务不得进入移动端执行列表，不得作为 Web 端可执行移动端任务。

如果通过 URL 直达移动端详情，只允许展示不可执行原因，不显示开工、完工、发起交出等执行按钮。

## 平台聚合状态验收说明

平台侧主状态只允许：

- 待下发
- 待接单
- 待开工
- 准备中
- 加工中
- 待送货
- 待回写
- 待审核
- 异常
- 已完成
- 已关闭

工厂内部细状态可以作为 `工厂内部状态` 辅助字段展示，但不能作为平台主状态。

## 仓、交出、回写、审核、差异联动验收说明

Web 端和移动端都通过统一写回触发仓联动：

- 印花完成转印后进入待交出仓。
- 染色完成包装后进入待交出仓。
- 裁片确认入仓后进入待交出仓。
- 特殊工艺确认接收后进入待加工仓，完成加工后进入待交出仓。
- 发起交出后生成交出记录。
- 接收方回写后生成审核记录或差异记录。
- 差异记录必须关联交出记录。
- 特殊工艺差异必须关联菲票。
- 裁片交出必须关联原始裁片单和菲票。

## 统计联动验收说明

统计页面读取统一事实源：

- 印花统计读取加工单、仓记录、交出记录、差异记录。
- 染色统计读取加工单、仓记录、交出记录、差异记录。
- 特殊工艺统计读取特殊工艺单、菲票、仓记录、交出记录、差异记录。
- 裁片相关看板读取原始裁片单、菲票、仓记录、交出记录、差异记录。

统计字段必须带对象和单位，不只写“数量”。

## 裁片冻结口径验收说明

裁片链路必须保持以下口径：

- 生产单来源唯一：生产需求单。
- 不因配料、领料、裁剪安排而生成、拆分或合并生产单。
- 可裁判断发生在原始裁片单层。
- 合并只发生在裁片执行层。
- 合并裁剪批次只作为执行上下文。
- 菲票永远回落原始裁片单。
- 裁片交出必须关联原始裁片单和菲票。
- 不把合并裁剪批次作为菲票归属主体。
