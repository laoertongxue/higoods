# BOM 缩水 / 洗水需求与准备工序联动

## 范围

本轮只处理技术包物料清单中的“缩水需求”“洗水需求”，以及它们对工序工艺准备阶段的自动生成关系。不改纸样管理、捆条维护、裁片明细、菲票打印、特殊工艺任务生成和工厂端执行闭环。

## 字段口径

- 缩水需求：取值为“是 / 否”，默认“否”。
- 洗水需求：取值为“是 / 否”，默认“否”。
- 物料清单保存时保留 `shrinkRequirement`、`washRequirement`。
- 技术包版本快照保留 BOM 原始需求和自动生成的准备阶段工序。

## 联动规则

- 任一 BOM 行“缩水需求 = 是”，准备阶段自动生成 `PREP_SHRINKING / 缩水`。
- 任一 BOM 行“洗水需求 = 是”，准备阶段自动生成 `PREP_WASHING / 洗水`。
- 自动生成工序统一标记：
  - `sourceType = BOM`
  - `triggerField = shrinkRequirement / washRequirement`
  - `isAutoGenerated = true`
  - `canRemoveAutomatically = true`
- 全部 BOM 行取消需求后，如果自动生成工序没有人工维护内容，则自动移除。
- 如果已有人工作业备注、人工标记或人工维护字段，不静默删除，页面提示用户选择保留或移除。

## 业务边界

- 洗水不再作为裁片部位级特殊工艺。
- 缩水 / 洗水不进入裁片明细逐片特殊工艺选择器。
- 准备阶段洗水通过 BOM 的洗水需求触发，后续生产单可消费技术包快照中的准备工序。

## 流程图

维护物料清单
-> 选择缩水需求或洗水需求
-> 系统扫描全部物料行
-> 自动生成准备阶段缩水或洗水
-> 工序工艺页展示自动生成来源
-> 保存技术包
-> 技术包版本快照保留需求和工序

## 状态机

未配置
-> BOM已选择需求
-> 准备工序已自动生成
-> 已进入技术包快照
-> 可被生产单消费

## 取消状态机

准备工序已自动生成
-> BOM需求取消
-> 检查人工维护内容
-> 无人工内容自动移除
-> 有人工内容待确认
-> 保留或移除
