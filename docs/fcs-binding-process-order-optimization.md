# FCS 捆条加工单优化研发说明

## 目标

本需求优化裁床模块的 `捆条加工单`，覆盖页面展示、记录裁剪、捆条菲票、库存查询、打印预览和性能验收。

核心目标：

- 捆条加工单必须能看到是否领料、货架位置、布料图片、捆条宽度、切割方式和加工记录。
- 每个捆条规格只能有一种切割方式，不允许同一规格同时维护直切、横切、斜切三套长度。
- `切割长度` 不允许手工直接填总长，必须按 `切割长度 = 每卷长度 × 实切卷数` 自动计算。
- 任一功能按钮点击后响应速度不得高于 200ms，满足这一规则才算完成页面功能开发。

## 切割方式口径

| 印尼语 | 中文业务口径 | 页面短标签 | 代码枚举 |
| --- | --- | --- | --- |
| Potongan serong/miring | 斜裁捆条 | 斜切 | `斜切` |
| Potongan lurus | 直裁捆条 | 直切 | `直切` |
| Potongan melintang | 横纹捆条 | 横切 | `横切` |

研发约束：

- `BindingStripCuttingMethod` 只能取 `斜切 | 直切 | 横切`。
- 一个捆条规格只对应一个 `cuttingMethod`。
- 记录裁剪时录入 `每卷长度` 和 `实切卷数`。
- 系统自动计算 `切割长度 = 每卷长度 × 实切卷数`。
- 根据 `cuttingMethod` 将计算出的切割长度落到对应字段：
  - `直切` -> `straightCutLength`
  - `横切` -> `crossCutLength`
  - `斜切` -> `biasCutLength`

## 关键字段

| 字段 | 含义 | 录入/计算 |
| --- | --- | --- |
| `rollLength` | 每卷长度，单位 m/卷 | 操作员录入 |
| `actualRollCount` | 实切卷数 | 操作员录入 |
| `actualLength` | 实际完成总长度 | 按每卷长度 × 实切卷数计算 |
| `straightCutLength` | 直切切割长度 | 直切规格自动写入 |
| `crossCutLength` | 横切切割长度 | 横切规格自动写入 |
| `biasCutLength` | 斜切切割长度 | 斜切规格自动写入 |

## 页面要求

### 记录裁剪弹窗

必须展示：

- 接收布料长度
- 每卷长度
- 实切卷数
- 切割长度
- 记录时间
- 操作员工
- 备注

交互规则：

- `每卷长度` 和 `实切卷数` 是录入项。
- `切割长度` 是只读计算结果。
- 输入每卷长度或实切卷数时，只局部更新当前行切割长度，不允许整页重绘。
- 页面必须显示公式文案：`切割长度 = 每卷长度 × 实切卷数`。

### 列表和详情

必须展示：

- 领料状态、货架位置、布料图片。
- 捆条宽度、切割方式。
- 捆条需要长度、需要布料长度、接收布料长度。
- 每卷长度、实切卷数、切割长度、实际完成总长度。
- 菲票状态、入仓状态、装袋交出状态、差异状态。

不允许：

- 显示 `直切长度 / 横切长度 / 斜切长度` 三列给操作员填写。
- 把入仓、装袋、交出状态混入加工主状态。

## 下游要求

### 捆条菲票

路由：

```text
/fcs/craft/cutting/binding-fei-tickets
```

要求：

- `打印菲票` 必须进入 `捆条菲票打印` 菜单。
- 菲票列表展示每卷长度、实切卷数和切割长度公式。
- 捆条菲票页只构建捆条打印对象，不得无条件构建普通部位菲票数据包。

### 打印预览

打印模板必须展示：

- 每卷长度
- 实切卷数
- 切割公式
- 切割长度

打印预览必须先按捆条菲票记录查找，命中后不得继续构建普通菲票全集投影。

### 库存查询

路由：

```text
/fcs/craft/cutting/warehouse-management/wait-handover?inventoryType=binding
```

要求：

- `inventoryType=binding` 必须走捆条库存轻量路径。
- 不得进入普通待交出仓完整聚合逻辑。

## 性能验收

所有功能按钮点击后响应不得高于 200ms。捆条加工单至少验收：

- 捆条加工单列表渲染低于 200ms。
- 捆条菲票打印页渲染低于 200ms。
- 捆条库存查询页渲染低于 200ms。
- 捆条菲票打印预览渲染低于 200ms。
- 记录裁剪弹窗打开低于 200ms。
- 弹窗输入每卷长度或实切卷数时无整页重绘。

推荐命令：

```bash
npm run check:cutting-binding-strip-flow
npm run check:cutting:all
npm run build
```
