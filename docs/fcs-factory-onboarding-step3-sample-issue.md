# 工厂入驻第 3 步：样衣验证对象与平台发样衣

## 背景

本步骤建立在入驻模型纠偏和平台初审逻辑纠偏之后。平台初审通过只说明入驻资料可进入下一阶段，不代表工厂已经具备正式合作资格，因此必须先进入样衣验证。

## 样衣验证对象

新增 `FactorySampleVerification` 作为样衣验证对象，关联 `FactoryOnboardingApplication`。样衣验证对象记录样衣批次、款号、样衣名称、样衣件数、样衣说明、验证目的、发放方式、快递信息、发放时间、发放人、预计收样时间、预计提交样衣审核时间、平台参考照片、平台参考视频和平台参考资料。

入驻申请通过字段关联样衣验证：

- `sampleVerificationId`：样衣验证 ID
- `sampleStatus`：当前样衣状态
- `sampleIssuedAt`：样衣发放时间
- `sampleExpectedSubmitAt`：预计提交样衣审核时间

## 平台发样衣

平台在 `/fcs/factories/onboarding` 的“工厂入驻管理”中，只能对“待样衣验证”的申请执行“登记并发放样衣”。弹窗字段包括样衣批次号、款号、样衣名称、样衣件数、样衣说明、验证目的、发放方式、快递公司、快递单号、发放时间、发放人、预计收样时间、预计提交样衣审核时间和平台参考资料。

发放样衣后：

- 创建 `FactorySampleVerification`
- 样衣验证状态为“待工厂确认收样”
- 入驻申请状态变为“待工厂确认收样”
- 入驻申请当前节点保持“样衣验证”
- 写入入驻申请 `sampleVerificationId`、`sampleStatus`、`sampleIssuedAt`、`sampleExpectedSubmitAt`
- 写入入驻申请动作记录“平台登记并发放样衣”

## 为什么仍不能生成工厂档案

发放样衣只是进入样衣验证过程，工厂还没有确认收样、提交样衣审核资料，也没有通过平台样衣审核。工厂档案只能在后续“样衣审核通过待转正式”阶段允许生成，本步骤不生成工厂档案。

## 为什么仍不能开放业务页

执行、交接、仓管、结算只允许“已转正式合作”的工厂进入。“待工厂确认收样”仍属于入驻流程中的样衣验证阶段，不具备正式业务准入资格。

## 本次不实现内容

本次不实现工厂确认收样、工厂提交样衣审核、平台样衣审核和转正式合作，也不新增后端、服务端或数据库。
