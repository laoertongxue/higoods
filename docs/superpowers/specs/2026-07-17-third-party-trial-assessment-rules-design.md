# 三方车缝工厂试产考核与评级规则规格

日期：2026-07-17

## 1. 背景

三方车缝工厂评级已经从工厂档案移到独立菜单“三方工厂评级”，并且派单策略、结算策略已经开始由结构化规则生成。当前仍存在 4 个需要继续收敛的问题：

1. 规则和策略不能依赖长段文案说明，必须能被代码直接判断。
2. 不良率中的“不良”必须来自质检单事实，不能凭空编造瑕疵样例。
3. 试产考核不是普通履约记录。一般情况下，每个工厂只有一个首轮试产单；业务人员延长考核一次，才允许产生下一轮试产单。
4. 评级列表和详情必须展示试产单情况，不能只展示当前评级结果。

本规格把“三方车缝工厂评级”拆成两层：

- 试产考核记录：记录每一轮试产单的派单、完成时效、质检、不良率、系统评级和人工结论。
- 评级快照：读取最新一轮试产考核记录，生成当前生效评级、合作状态、派单规则和结算规则。

## 2. 产品目标

本期目标是形成一个可执行、可追溯、可检查的试产评级闭环：

1. 试产单情况成为独立结构化数据，不再混在评级快照文案里。
2. 自动评级直接读取试产完成时效和质检事实。
3. 不良率按统一公式计算：返工数量 + 工厂责任瑕疵原因数量。
4. 延长考核允许多次，每次延长只开放下一轮一个试产单。
5. 列表展示最新试产单结果，详情展示所有试产考核轮次。
6. 派单、结算和页面展示全部消费结构化字段，不消费自由文本策略。

## 4. 核心业务口径

### 4.1 试产单与轮次

三方车缝工厂进入考核时，默认只允许第 1 轮试产单。

每一轮试产考核只能有一个试产单：

- 第 1 轮：首单试产。
- 第 2 轮：第 1 轮评级后，业务人员选择“延长考核”才开放。
- 第 3 轮及以后：上一轮评级后继续选择“延长考核”才开放。

如果当前轮次已有试产单未完成，不能再派同轮次的新试产单。

### 4.2 延长考核

延长考核不是新的合作状态，也不是独立流程。

延长考核的业务含义是：

- 工厂继续保持“考核中”。
- 派单控制继续为 `TRIAL_ONLY`。
- 结算不做黑名单拦截。
- 下一轮仍只允许试产单。
- 下一轮试产单数量继续受车位规模派生的试产上限控制。
- 新一轮试产完成后，系统重新自动评级。

延长考核允许多次，但每次延长只开放一轮新的试产资格。

### 4.3 规则和策略

规则、策略必须是结构化字段或函数。

允许展示短标签和简短说明，但说明不能作为规则来源。

禁止将以下内容作为业务判断依据：

- `dispatchPolicyLabel` 这类自由文本字段。
- `settlementPolicyLabel` 这类自由文本字段。
- 页面上的长段说明文案。
- mock 中手写的“评级原因”自然语言。

业务判断必须读取：

- `dispatchControl`
- `settlementControl`
- `allowedDocumentTypes`
- `trialLimitQty`
- `assessmentDecision`
- `assessmentRound`
- `autoRatingGrade`
- `effectiveDecision`
- 试产考核记录中的时效和质检字段

## 5. 工厂责任瑕疵原因

### 5.1 责任原因清单

计入三方车缝工厂评级不良率的工厂责任瑕疵原因，只能来自现有结构化清单：

- 做工原因
- 脏污
- 抽纱
- 做错
- 做毁
- 破洞

该清单对应当前代码中的 `SEWING_FACTORY_LIABILITY_REASONS`。

### 5.2 不计入评级不良率的原因

后道质检原因字典中还存在其他原因，例如：

- 布料原因
- 色差
- 缺辅料
- 半套
- 印花
- 缺辅料不补

这些可以出现在完整质检事实中，但不得计入“三方车缝工厂责任不良率”。

### 5.3 Mock 数据约束

试产考核 mock 数据中的工厂责任瑕疵原因必须只使用：

```text
做工原因 / 脏污 / 抽纱 / 做错 / 做毁 / 破洞
```

不得出现字典外样例，例如：

- 针距问题
- 缝线不齐
- 质量异常
- 做工瑕疵
- 工艺问题

## 6. 不良率计算

### 6.1 输入字段

不良率必须从质检单事实计算，不允许手工写死。

每轮试产考核记录必须包含：

| 字段 | 含义 |
| --- | --- |
| `qcOrderNo` | 质检单号 |
| `inspectedQty` | 质检数量 |
| `qualifiedQty` | 合格数量 |
| `reworkQty` | 返工数量，来自质检单 |
| `factoryLiabilityDefectReasonItems` | 工厂责任瑕疵原因明细 |

`factoryLiabilityDefectReasonItems` 的每一项包含：

| 字段 | 含义 |
| --- | --- |
| `reasonName` | 瑕疵原因名称，必须属于工厂责任原因清单 |
| `qty` | 该原因数量 |

### 6.2 计算公式

```text
工厂责任瑕疵数量 = sum(factoryLiabilityDefectReasonItems.qty)

不良数量 = reworkQty + 工厂责任瑕疵数量

不良率 = 不良数量 / inspectedQty
```

计算边界：

- `inspectedQty` 必须大于 0。
- `reworkQty` 不能小于 0。
- 责任瑕疵原因数量不能小于 0。
- `qualifiedQty + reworkQty + 其他不合格处置数量` 不在本期强制闭合，但试产考核详情必须展示实际质检字段。

## 7. 试产考核记录模型

新增数据对象：`trialAssessmentRecords`。

每条记录代表某个三方车缝工厂的一轮试产考核。

| 字段 | 含义 |
| --- | --- |
| `assessmentId` | 考核记录 ID |
| `factoryId` | 工厂主档 ID |
| `factoryCode` | 工厂编码 |
| `factoryName` | 工厂名称快照 |
| `assessmentRound` | 考核轮次，从 1 开始 |
| `trialOrderNo` | 试产单号 |
| `productionOrderNo` | 关联生产单 |
| `dispatchQty` | 试产派单数量 |
| `plannedDeliveryAt` | 计划交出时间 |
| `actualDeliveryAt` | 实际交出时间 |
| `delayDays` | 延期天数，由计划和实际交出时间计算 |
| `qcOrderNo` | 质检单号 |
| `inspectedQty` | 质检数量 |
| `qualifiedQty` | 合格数量 |
| `reworkQty` | 返工数量 |
| `factoryLiabilityDefectReasonItems` | 工厂责任瑕疵原因明细 |
| `factoryLiabilityDefectQty` | 工厂责任瑕疵数量合计 |
| `defectiveQty` | 不良数量 |
| `defectRate` | 不良率 |
| `timelinessGrade` | 时效命中等级 |
| `qualityGrade` | 质量命中等级 |
| `autoRatingGrade` | 系统自动评级等级 |
| `autoRatingDecision` | 系统建议结论 |
| `manualDecision` | 业务人工结论 |
| `manualReason` | 人工调整原因 |
| `effectiveDecision` | 最终生效结论 |
| `status` | 当前记录状态 |

记录状态建议：

| 状态 | 含义 |
| --- | --- |
| `WAIT_TRIAL_DISPATCH` | 等待派出本轮试产单 |
| `TRIAL_DISPATCHED` | 本轮试产单已派出 |
| `WAIT_QC` | 已交出，等待质检 |
| `AUTO_RATED` | 系统已自动评级 |
| `MANUAL_CONFIRMED` | 业务人员已确认最终结论 |

页面展示时使用中文状态，不直接显示英文状态码。

## 8. 自动评级规则

自动评级读取两个维度：

- 完成时效：`delayDays`
- 质量情况：`defectRate`

### 8.1 时效规则

| 命中等级 | 条件 |
| --- | --- |
| S | `delayDays <= 0` |
| A | `delayDays <= 1` |
| B | `delayDays <= 3` |
| C | `delayDays > 3` |

### 8.2 质量规则

| 命中等级 | 条件 |
| --- | --- |
| S | `defectRate <= 2%` |
| A | `defectRate <= 5%` |
| B | `defectRate <= 10%` |
| C | `defectRate > 10%` |

### 8.3 综合评级

综合评级取时效等级和质量等级中的较差等级。

例如：

| 时效等级 | 质量等级 | 综合评级 |
| --- | --- | --- |
| S | A | A |
| A | B | B |
| B | S | B |
| C | A | C |

### 8.4 系统建议结论

| 综合评级 | 系统建议 |
| --- | --- |
| S | 转正 |
| A | 转正 |
| B | 延长考核 |
| C | 拉黑 |

业务人员可以覆盖系统建议，但必须填写 `manualReason`。

最终生效结论：

```text
effectiveDecision = manualDecision ?? autoRatingDecision
```

## 9. 评级快照读取规则

评级快照只表达当前生效状态，不保存试产事实明细。

评级快照来源：

| 快照字段 | 来源 |
| --- | --- |
| 工厂名称 / 编码 | 工厂主档 |
| 车缝车位数 | 工厂主档 / 产能资料 |
| 工厂规模 | 车缝车位数派生 |
| 当前评级 | 最新已确认试产考核记录 |
| 当前考核轮次 | 最新试产考核记录 |
| 最新试产单号 | 最新试产考核记录 |
| 最新试产结果 | 最新试产考核记录 |
| 最新不良率 | 最新试产考核记录 |
| 合作状态 | 最新最终结论派生 |
| 派单策略 | 最新最终结论派生 |
| 结算策略 | 最新最终结论派生 |
| 下一单规则 | 最新最终结论 + 车位规模派生 |

### 9.1 生效结论到快照状态

| `effectiveDecision` | 合作状态 | 派单策略 | 结算策略 |
| --- | --- | --- | --- |
| 转正 | 正常合作 | `ALLOW` 或 `PRIORITY` | `ALLOW` |
| 延长考核 | 考核中 | `TRIAL_ONLY` | `ALLOW` |
| 拉黑 | 黑名单 | `BLOCKED` | `BLOCK_NEW_STATEMENT` |

S 级转正可以展示 `PRIORITY`，A 级转正展示 `ALLOW`。

详情必须展示全部试产考核轮次。

### 11.1 当前评级摘要

展示：

- 合作状态。
- 当前派单策略。
- 当前结算策略。
- 当前考核轮次。
- 下一单允许单据。
- 下一单上限。

### 11.2 试产考核记录

按轮次倒序展示。

每轮展示：

- 试产单号。
- 生产单号。
- 试产数量。
- 计划交出时间。
- 实际交出时间。
- 延期天数。
- 质检单号。
- 质检数量。
- 合格数量。
- 返工数量。
- 工厂责任瑕疵原因明细。
- 不良数量。
- 不良率。
- 时效命中等级。
- 质量命中等级。
- 系统自动评级。
- 系统建议。
- 人工结论。
- 人工原因。

### 11.3 规则命中

每轮试产考核必须展示结构化命中结果：

| 维度 | 展示 |
| --- | --- |
| 时效 | `delayDays` 命中 S/A/B/C 哪一档 |
| 质量 | `defectRate` 命中 S/A/B/C 哪一档 |
| 综合 | 取较差等级 |
| 建议 | 由综合等级派生 |

不得用长段文案替代规则命中表。

## 12. 延长考核交互

本期不做完整审批流，但必须表达人工决策结果。

业务人员在每轮系统自动评级后，可以确认：

- 转正。
- 拉黑。
- 延长考核。

选择“延长考核”时：

- 必须填写人工原因。
- 当前工厂继续保持考核中。
- 系统开放下一轮试产资格。
- 下一轮仍只能派试产单。
- 下一轮上限继续按车缝车位数派生。

同一工厂当前存在未完成试产考核记录时，不允许再创建下一轮。

## 13. 派单联动

派单模块必须读取评级快照和试产考核记录派生结果。

### 13.1 考核中

规则：

- 只允许试产单。
- 常规单阻断。
- 本轮试产数量不能超过 `nextTrialLimitQty`。
- 当前轮已有未完成试产单时，不能再派试产单。
- 只有上一轮最终结论为“延长考核”时，才能派下一轮试产单。

### 13.2 正常合作

规则：

- S 级优先派单。
- A 级正常允许派单。
- B 级黄牌必须风险确认后才允许派单。

### 13.3 黑名单

规则：

- 禁止派单。
- 不允许通过页面确认放行。

## 14. 结算联动

结算模块继续读取结构化结算策略。

规则：

- 正常合作：允许发起结算。
- 考核中：允许按账本发起结算。
- 黑名单：禁止发起新结算。
- 黑名单历史账本仍可查看。

结算规则不读取试产说明文案。

## 15. Mock 数据要求

mock 数据必须覆盖以下场景：

1. 第 1 轮试产后转正。
2. 第 1 轮试产后拉黑。
3. 第 1 轮试产后延长考核，第 2 轮重新评级。
4. 多次延长考核。
5. B 级黄牌需要人工确认。
6. 黑名单阻断派单和新建结算。
7. 考核中试产单超量阻断。

每条试产考核记录必须有：

- 试产单号。
- 生产单号。
- 质检单号。
- 试产数量。
- 计划交出时间。
- 实际交出时间。
- 返工数量。
- 工厂责任瑕疵原因明细。

mock 数据禁止：

- 使用字典外瑕疵原因。
- 写死不可追溯的 `defectQty`。
- 只写自然语言“质量波动”但没有质检数量。
- 只在详情里有试产单，列表没有试产单摘要。

## 16. 检查与验收

专项检查必须覆盖：

1. `SEWING_FACTORY_LIABILITY_REASONS` 是评级不良率的唯一工厂责任原因来源。
2. 试产考核记录中的工厂责任瑕疵原因全部属于该清单。
3. 不良数量等于 `reworkQty + 工厂责任瑕疵数量`。
4. 不良率等于 `不良数量 / inspectedQty`。
5. 自动评级按时效等级和质量等级取较差等级。
6. 系统建议按综合评级派生。
7. 人工结论覆盖系统建议时必须有人工原因。
8. 每个工厂每轮最多一个试产单。
9. 延长考核后仍是 `考核中 + TRIAL_ONLY + 试产单`。
10. 评级快照读取最新试产考核记录。
11. 列表展示试产单号、轮次、完成时效、不良率。
12. 详情展示全部试产考核轮次。
13. 派单阻断考核中常规单和超量试产单。
14. 派单阻断当前轮未完成时重复派试产单。
15. 结算阻断黑名单新建结算。
16. 页面不直接展示自由文本策略字段作为规则来源。

## 17. 规格自检

### 17.1 完整性检查

本文档不包含未完成标记、空字段或未命名模块。

### 17.2 一致性检查

- 试产考核记录是事实来源。
- 评级快照只读取最新结果。
- 派单和结算只读取结构化策略。
- 不良率只读取质检事实。
- 延长考核保持考核中，不改变为独立状态。

### 17.3 范围检查

本规格聚焦三方车缝工厂试产考核、评级快照、列表详情展示和规则检查。真实后端、审批流、规则配置页和版本管理不在本期范围。

### 17.4 模糊性处理

- “不良”明确为返工数量 + 工厂责任瑕疵数量。
- “工厂责任瑕疵原因”明确为 6 个既有原因。
- “延长考核”明确为允许下一轮试产，不是独立状态。
- “当前评级”明确读取最新已确认试产考核记录。
