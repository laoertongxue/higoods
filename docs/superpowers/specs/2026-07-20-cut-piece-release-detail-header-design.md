# 裁片放行矩阵详情页头设计

## 背景

裁片放行矩阵通过新窗口打开后，当前页面直接从矩阵业务卡片开始，只显示「PO14671 裁片放行矩阵」。页面缺少详情页级别的返回入口、一级标题和业务对象摘要，用户无法快速判断当前位置，也不能像「捆条加工单详情」一样返回对应列表。

## 目标

为裁片放行矩阵独立详情页补齐稳定的详情页头，同时保持矩阵计算、目标选择、版本历史和补料跳转逻辑不变。

## 页面结构

详情页从上到下分为两层：

1. 页面级页头：
   - 返回入口：`返回裁片放行管理`。
   - 一级标题：`裁片放行矩阵详情`。
   - 对象摘要：`生产单号 · SPU 编码 · 款式名称`，例如 `PO14671 · ASYSA26060310 · 女式基础圆领短袖`。
2. 矩阵业务卡片：
   - 保留现有 `PO14671 裁片放行矩阵` 二级标题。
   - 保留当前版本、更新历史、目标选择、缺口、多余裁片和补料入口。

页面级页头位于矩阵卡片之外，避免把导航和业务动作混在同一卡片内。

## 返回行为

点击 `返回裁片放行管理` 后，当前详情窗口跳转到 `/fcs/craft/cutting/cut-piece-release` 列表页。使用项目既有的 `href` 与 `data-nav` 导航协议，支持普通点击和链接语义，不增加关闭窗口或跨窗口通信逻辑。

## 边界与异常

- 只有带生产单查询参数且成功找到矩阵记录的独立详情页展示该页头。
- 列表页保持现有标准列表页标题，不重复展示详情页头。
- 矩阵记录找不到时沿用现有列表降级行为，本次不扩展错误页。
- 不修改 Mock 数据、矩阵版本、目标快照及补料管理逻辑。

## 验收

- 新窗口详情页显示唯一的一级标题 `裁片放行矩阵详情`。
- 页头显示 `返回裁片放行管理`，点击后到达裁片放行管理列表。
- 页头摘要显示生产单号、SPU 编码和款式名称。
- 原有矩阵二级标题及 4 种颜色矩阵继续正常显示。
- 刷新详情页后页头仍存在。
- 1366×768 和 1280×720 下页面主体无横向溢出。
