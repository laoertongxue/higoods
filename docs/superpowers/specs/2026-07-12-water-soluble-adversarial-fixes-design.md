# 水溶工序对抗式审查缺口修复设计

## 1. 文档信息

- 修复对象：`codex/water-soluble-process` 分支对抗式审查发现的 2 个 Critical、9 个 Important、3 个 Minor。
- 修复原则：修根因，不扩大水溶工序既定产品范围。
- 业务规则沿用：仅水溶生成独立水溶加工单；同一物料水溶加染色只生成一张含水溶染色加工单；固定同厂、先水溶后染色；水溶不进入生产准备时效。
- 端型：FCS 管理端、PFOS 染厂主管端、PDA 员工执行端与交接端。

## 2. 修复目标

1. 独立水溶加工单必须真实来源于正式技术包快照中“仅水溶”的 BOM 物料。
2. 新建染色移动任务的任务号、二维码、来源与数量单位必须一致。
3. 含水溶染色的所有领域动作均不能绕过水溶、染色和后处理前序节点。
4. PDA 必须阻断跨厂读取、离线写入和错误扫码交出。
5. FCS、PFOS、PDA 与交接继续读取同一加工单、任务、状态、数量和单位事实。
6. 用真实反例测试替换会锁死错误行为的测试和 Mock。

## 3. 明确不变的范围

- 不新增水溶需求单。
- 不新增水溶专用 PDA 导航、仓库、结算、计价或批次模块。
- 不支持水溶和染色分厂。
- 不改变普通染色、印花、预缩、水洗等工序的生成规则。
- 不重构通用任务、权限或交接框架，只修正水溶链路调用和必要的共用校验。

## 4. 数据来源与演示数据

### 4.1 正式产物边界

独立水溶加工单只消费满足以下条件的产物：

- 来源为生产单正式技术包快照。
- 产物类型为 `TASK`，工序为 `WATER_SOLUBLE`。
- 关联一条明确 BOM 物料。
- BOM 物料 `waterSolubleRequirement = 是`。
- 同一 BOM 物料不需要染色。
- 计划数量和原 BOM 单位完整有效。

`DICT-*` 字典覆盖产物只用于字典和通用产物展示，不得进入水溶加工单、统一加工单、PDA 或交接。

### 4.2 真实演示生产单

在既有正式技术包演示数据中增加至少一条真正的“仅水溶”BOM 物料，并通过正式快照和真实拆解入口生成独立水溶产物。不得在水溶领域层人工补单。

演示数据同时保留：

- 仅水溶。
- 仅染色。
- 同一物料水溶加染色。
- 不同物料分别水溶和染色。
- 均未选择。

### 4.3 来源版本冻结

水溶加工单的 `techPackVersionId` 只能来自生成产物对应的正式快照版本。生产单当前选择的草稿或新版本不得刷新既有加工单来源。

来源产物暂时消失时：

- 已进入执行或已产生交接事实的加工单不得静默删除。
- 尚未执行的无效演示产物可以在明确同步时移除。
- 本次原型至少保证正式快照切换不会重置或改写既有执行事实。

## 5. 任务身份、二维码与单位

### 5.1 新建任务身份

从染色需求创建加工单时，可以复用任务模板的展示配置，但必须重新生成：

- `taskId`、`taskNo`。
- `taskQrValue`，且二维码必须编码当前新 taskId。
- 与模板任务有关的审计日志和运行状态。

二维码、任务详情和扫码定位必须指向同一新任务。

### 5.2 单位规则

- 正式 BOM 缺少单位时阻断产物生成，返回明确中文原因；不得默认补成“米”。
- 染色需求、加工单、移动任务、PDA 输入、执行节点和最终交接统一保留原 BOM 单位。
- 通用任务需要标准枚举时，必须同时保留权威显示单位；交接优先读取权威显示单位，不能由 `METER` 推断覆盖。
- 所有数量输入提示必须包含当前单位。

## 6. 状态机与数量防错

### 6.1 前序节点硬校验

领域层而非页面层负责校验顺序：

~~~text
备料与染缸安排 → 水溶 → 染色 → 脱水 → 烘干 → 定型 → 打卷 → 包装 → 最终交出
~~~

`startDyeNode`、`completeDyeNode` 及通用写回入口都必须确认当前状态和上一个节点已完成。即使通过历史 DOM、直接模块调用或伪造 actionCode，也不能跳过前序节点。

### 6.2 零产出

水溶实际完成数量允许为 0：

- 小于 0 或非有限数继续阻断。
- 0 必须填写原因。
- 独立水溶和含水溶染色均进入生产暂停。
- 主管可选择继续补做或退回重做；0 不能选择按实际数量交出或开始染色。

## 7. PDA 与 PFOS 现场防错

### 7.1 跨厂读取

PDA 水溶详情在渲染任何生产单、物料、数量和日志前，必须校验：

- 当前存在有效 PDA session。
- 任务已分配工厂等于 session 工厂。
- 任务在当前工厂可见。

失败时只显示无权限提示，不泄露任务事实。

### 7.2 离线交出

水溶交出提交在生成记录和改变加工单状态前检查真实浏览器在线状态：

- 离线时不写领域、不生成记录、不消耗确认令牌。
- 保留扫码值、数量和焦点。
- 提示“当前网络不可用，请联网后重试”。
- 恢复联网后原输入可提交一次，重复提交不二写。

### 7.3 扫码匹配

水溶交出扫码值必须匹配当前交出单允许的结构化标识之一：

- 当前任务二维码或任务号。
- 当前水溶加工单号。
- 当前 BOM 物料编码。

其他非空值均阻断，并提示当前应扫描的对象。扫码值不能只写入备注后放行。

### 7.4 角色矩阵

页面、handler 和领域统一使用同一矩阵：

- 操作员：到料、开始水溶、完成水溶。
- 生产主管：处理生产暂停，不执行普通水溶动作。
- 交接员：进入通用交接并交出。
- 管理员：主管和交接异常兜底；普通现场加工仍由操作员执行。

页面不得展示最终必然被领域拒绝的主动作。

## 8. FCS 与 PFOS 追溯闭环

### 8.1 FCS

独立水溶列表除分配染厂和查看详情外，按状态提供：

- 查看统一任务详情。
- 查看执行与异常记录。
- 已生成交接单时查看交接详情。

入口必须使用真实 taskId、waterOrderId 和 handoverOrderId，不另造页面状态。

### 8.2 PFOS

从水溶动作日志中解析并展示最近 PDA 操作人、动作和时间。没有操作人时显示“未记录”，不能在已有事实时固定显示未记录。

### 8.3 无关 Mock 回归

新增水溶能力工厂应增加或扩展专门演示记录，不覆盖既有“定向裁＋后道包装”等无关多能力场景。

## 9. 测试设计

所有修复遵循红—绿循环，必须先看到反例测试失败。

### 9.1 领域测试

- `DICT-*` 水溶产物不能生成加工单。
- 正式快照仅水溶 BOM 生成且重复拆解幂等。
- 同物料水溶加染色不生成独立水溶单。
- 未派厂加工单不会跟随草稿版本变化。
- 新任务二维码等于 `buildTaskQrValue(newTaskId)`，且不等于模板二维码。
- 公斤、码、卷单位贯穿移动任务和最终交接。
- 缺单位阻断生成。
- 直接完成 PACK、跳过染色或跳过任一后处理均失败且无副作用。
- 水溶完成 0 有原因进入生产暂停，无原因阻断。

### 9.2 浏览器测试

- 外厂账号直达水溶任务 URL 只能看到无权限提示。
- 离线交出无领域副作用，联网重试只写一次。
- 错误扫码值阻断，任务码、加工单号和物料码允许。
- 操作员、主管、交接员、管理员各自只看到允许动作。
- FCS 可以从水溶单进入任务和交接详情。
- PDA 数量输入明确显示公斤、码或卷。

### 9.3 回归与治理

- 三条水溶专项检查。
- 水溶页面与 PDA Playwright 全量用例。
- 技术包路线、正式快照、染色工作流、PDA 详情与通用交接回归。
- 生产准备时效隔离检查。
- 原型治理检查与构建。
- 更新原型审查记录，使结论只描述已被新鲜证据证明的行为。

## 10. 完成标准

- 两个 Critical 和全部 Important 均有先失败后通过的直接测试。
- 当前独立水溶页面展示的每张单都能追溯到正式快照中的“仅水溶”BOM。
- 新任务二维码、任务号、单位、工厂与交接事实一致。
- 跨厂读取、离线写入、错误扫码和节点绕过均被阻断且无副作用。
- FCS、PFOS、PDA 和交接形成可点击、可追溯的闭环。
- 无生产准备时效、仓库、结算、批次或分厂范围扩张。
- 原型审查记录与最终运行证据一致。
