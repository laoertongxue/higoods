# 裁片配料与领料退回可视化联动设计

## 背景

裁床从中转仓领料后，物料进入裁床待加工仓。开工前发现物料异常时，裁床在领料管理中按物料行退回到中转仓。

当前原型已经有退回动作，但页面体验断裂：

- 裁片配料和领料管理看起来不像同一套数据。
- 退回记录藏在领料记录区域，列表层没有明显提示。
- 裁片配料详情顶部摘要放在 Tab 外，与各 Tab 内容重复。
- 领料管理退回记录缺少反向跳转，用户很难定位同一配料单。

## 范围

本次只处理配料和领料两个模块：

- 裁片配料列表、详情。
- 裁前准备 - 领料管理列表、详情。
- 配料、领料、退回必须来自同一套 mock 数据和同一套投影。

不处理：

- 中转仓收回确认。
- 待质检判定。
- 换料重配。
- 无法补料后续。
- 真实后端、真实库存数据库和接口联调。

## 业务口径

### 退回不是配料主状态

退回是领料后的物料行事实，不是配料单主状态。

一个配料单可以同时存在：

- 有物料仍待配。
- 有物料已配齐。
- 有物料已领料。
- 有物料已退回。

因此不新增“已退回”主状态 Tab。

### 已退回统计口径

列表中的 `已退回` 使用物料行数口径：

`已退回 = 当前配料单中退回数量大于 0 的物料行数`

不是退回记录数。

数量口径在明细行展示：

- 已配数量。
- 已领数量。
- 已退数量。
- 待加工仓剩余数量。

### 配料记录不回滚

退回发生后，原配料记录仍保持已确认事实，不回退为被打回。

退回事实挂在：

- 领料记录。
- 配料记录。
- 物料行。

## 页面设计

### 裁片配料列表

保留现有主状态 Tab：

- 待配料 - 无库存可配。
- 待配料 - 部分有库存可配。
- 待配料 - 全部都有充足库存。
- 被打回重配。
- 已配齐。
- 已关闭。

新增轻量筛选：

- `只看有退回`

不新增退回主 Tab。

列表行中补充：

- 物料行统计增加 `已退回：N`。
- 领料状态旁展示 `已退：X 单位`。
- 最近操作纳入退回人和退回时间。
- 当存在退回时，在状态说明中提示 `含退回待中转仓处理`。

### 裁片配料详情

移除 Tab 外的大摘要区。

把摘要信息下沉到对应 Tab：

- `生产需求信息`：生产单、款式、SPU、计划数量、BOM 来源。
- `当前库存与上游`：配料状态、物料行、缺料缺口、库存和上游进度。
- `按任务查看`：任务分配、分配回写。
- `配料记录`：配料记录、仓库拣货、完成通知。
- `领料 / 退回记录`：领料、已退、待加工仓剩余、退回原因和退回状态。

把原来的 `与配料记录关联的领料记录` 改名为：

`领料 / 退回记录`

每条领料记录下展示：

- 领料记录号。
- 配料记录号。
- 物料行。
- 已领数量。
- 已退数量。
- 待加工仓剩余数量。
- 入库库区 / 库位。
- 退回原因。
- 退回人。
- 退回时间。
- 状态：`已退回待中转仓处理`。
- 图片凭证，若无则不显示。

### 领料管理详情

`退回处理` 中每条记录补充定位信息：

- 配料单。
- 配料记录。
- 领料记录。
- 物料行。

增加跳转入口：

`查看裁片配料`

跳转到同一个配料单，并默认打开：

`领料 / 退回记录`

跳转目标只进入配料模块，不进入仓储或质检模块。

## 数据设计

裁片配料与领料管理必须读取同一套数据源：

- `MaterialPrepRecord`
- `PickupRecord`
- `MaterialPickupReturnRecord`
- `MaterialPrepOrderProjection`

禁止在裁片配料和领料管理中各自维护独立 mock。

投影必须包含：

- `totalReturnedQty`
- `returnedLineCount`
- 每条物料行的 `returnedQty`
- 每条领料记录的 `returnQty`
- 每条领料记录的 `waitProcessAvailableQty`
- 最近退回操作时间和操作人

退回记录参与最近操作：

- 若退回时间晚于配料、领料、关闭时间，最近操作显示退回人和退回时间。

## Mock 数据覆盖

至少 12 条 mock 数据，必须覆盖以下场景：

1. `待配料 - 无库存可配`，无退回。
2. `待配料 - 部分有库存可配`，无退回。
3. `待配料 - 全部都有充足库存`，无退回。
4. `被打回重配`，无退回。
5. `已配齐`，未领料。
6. `已领料完结`，无退回。
7. `已领料完结`，部分退回。
8. `已领料完结`，全部退回。
9. 多物料配料记录中只有 1 行退回。
10. 同一领料记录多次部分退回。
11. 退回后仍有待加工仓剩余可用数量。
12. 退回后待加工仓剩余为 0。

每条 mock 必须能在两个页面中对上：

- 同一个配料单号。
- 同一个配料记录号。
- 同一个领料记录号。
- 同一个物料行。
- 同一组已领、已退、剩余数量。

## 筛选规则

`只看有退回` 的过滤口径：

- 只展示 `returnedLineCount > 0` 的配料单。
- 不改变主状态 Tab 的归属。
- 与关键词搜索叠加。

不做：

- 按退回原因筛选。
- 按退回人筛选。
- 按退回时间筛选。
- 按仓储处理状态筛选。

## 验证要求

需要保留最小可运行检查：

- 同一退回记录在领料管理详情可见。
- 同一退回记录在裁片配料详情 `领料 / 退回记录` 可见。
- 裁片配料列表能显示 `已退回` 物料行数。
- `只看有退回` 只展示有退回的配料单。
- 退回后的最近操作显示退回人和退回时间。
- mock 数据至少 12 条，并且配料单、配料记录、领料记录、物料行能互相对上。

## 规格自检

- 占位符：无。
- 内部一致性：退回被定义为领料后的物料行事实，不作为配料主状态。
- 范围：仅覆盖裁片配料和领料管理，不扩展到仓储、质检或补料后续。
- 模糊性：`已退回` 已明确为物料行数，退回数量在明细展示。
