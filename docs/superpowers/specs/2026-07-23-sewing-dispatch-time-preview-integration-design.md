# 车缝派单履约时效预览整合设计

## 1. 目标与范围

将「车缝履约时效预览」分支的完整业务范围整合到最新主线：独立车缝任务与车缝到后道任务在直接派单、改派时均可预览交付完成、30% 回货、70% 回货和 100% 回货的具体截止时间。

本次整合必须保留主线现有的工厂风控、风险确认、主管指定改派和主工厂确认能力。时效预览不改变工厂准入规则，也不绕过现有派单防错。

## 2. 统一业务口径

1. 业务分配时间是履约起点。时间为空、格式无效或晚于当前操作时间时，不得提交派单或改派。
2. 时效仅适用于独立车缝和车缝到后道任务；其他任务维持原有派单规则和展示。
3. 截止时间按连续满 24 小时计算；回货进度仅以接收方确认实收为准。
4. 直接派单与改派必须使用同一套预览规则，提交后的正式履约要求必须与提交前预览一致。
5. 改派以未完成的剩余数量为计算基数；已确认实收数量不得重新分配或重复计入。

## 3. 页面行为

### 3.1 直接派单

- 选择承接工厂并填写业务分配时间后，展示适用的履约类型和 4 个截止时间。
- 变更业务分配时间时，仅更新时效预览、主工厂选择区和错误提示，不关闭弹窗、不丢失已选择的 SKU、工厂和主工厂。
- 确认按钮仅在所有条件同时满足时可用：工厂准入通过、风险确认完成、主工厂已明确、业务分配时间有效。

### 3.2 改派

- 弹窗展示原工厂、已确认实收数量和剩余可改派数量。
- 选择目标工厂和业务分配时间后，立即展示改派后剩余数量对应的时效预览。
- 保留主线的工厂风险提示、主管指定改派和主工厂选择；时效预览仅增加校验条件，不替代这些条件。
- 提交成功后，新承接工厂只承担剩余数量及其新的履约时效；历史分配和已实收事实保持不变。

### 3.3 车缝到后道连续任务

- 仅对实际覆盖「车缝到后道」范围的连续任务展示履约时效预览。
- 其他连续工序保持原有页面和操作，不新增时效预览或额外校验。

## 4. 防错与异常处理

| 场景 | 页面处理 |
|---|---|
| 未填写业务分配时间 | 显示提示，确认按钮不可用 |
| 业务分配时间晚于当前操作时间 | 显示提示，确认按钮不可用 |
| 工厂不具备承接资格 | 保留主线提示与阻断，不展示可提交状态 |
| 需风险确认或主管指定改派 | 保留主线确认项；未完成时不可提交 |
| 不适用时效的任务 | 不展示预览，不改变既有派单流程 |
| 改派任务无剩余数量 | 沿用主线阻断，不生成新的时效要求 |

## 5. 整合策略

采用组合整合，不直接覆盖主线页面：

1. 保留主线的工厂政策判断、风险确认、主管指定改派和局部刷新机制。
2. 引入统一的履约时效预览能力，由直接派单和改派共用。
3. 将确认按钮的判断合并为共同条件，而非选择任一分支的旧逻辑。
4. 保留分支中的产品需求文档、审查记录和专项检查；检查口径同步到主线的工厂稳定标识规则。

## 6. 验收标准

1. 独立车缝与车缝到后道任务均可在直接派单前查看 4 个截止时间。
2. 改派仅按剩余数量预览和冻结新的履约要求。
3. 时间无效、工厂不合规、风险确认未完成或主工厂未明确时，确认按钮均不可用。
4. 变更时间或工厂时，时效预览与主线工厂风控提示同步局部刷新。
5. 不适用任务不显示时效预览，也不受本次校验影响。
6. 专项检查、页面交互回归、列表治理、原型设计治理和生产构建均通过。
