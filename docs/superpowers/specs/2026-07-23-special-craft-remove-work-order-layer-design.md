# 特殊工艺域去掉子加工单层设计

## 日期

2026-07-23

## 背景

特殊工艺域（辅助工艺 + 特种工艺）的数据模型中存在"任务单 → 子加工单"两层结构。一张生产单按工艺拆分为任务单后，`buildWorkOrdersFromTaskOrders` 再按裁片部位（`partName`）拆分为子加工单，状态和数量操作均挂在子加工单上。

但这个拆分没有业务意义：

- 同一张任务单的不同部位，承接工厂是同一家，不存在分包场景
- 成衣类工艺（烫画、直喷）的 `targetObject` 为"成衣"，所有明细归到同一个部位，一张任务单永远只产生一张子加工单（1:1）
- 子加工单的存在增加了 3 层页面跳转（列表 → 任务详情 → 子加工单 tab → 加工单详情），操作路径不可见

## 现状

- 特殊工艺域共 13 个工艺：绣花、抽条、压褶、打缆、烫画、直喷、贝壳绣、曲牙绣、直牙绣（辅助工艺），模板工艺、激光袋、花样机、橡筋定长（特种工艺）
- 水溶不在此域内
- 操作页面三层：任务单列表页 → 任务详情页 → 加工单详情页（操作面板在此）

## 方案

### 一、数据模型变更（阶段 1）

**删除：**

- "子加工单"概念 —— `SpecialCraftTaskWorkOrder` 类型和所有相关函数
- 子加工单行明细 —— `SpecialCraftTaskWorkOrderLine` 类型
- 子加工单业务类型 —— `SpecialCraftWorkOrderBusinessType`
- 所有子加工单的查询、构建、状态变更函数

**任务单直接承担执行：**

任务单原来只记录"谁做、做什么"，现在同时承担状态和进度：

| 领域 | 变化 |
|------|------|
| 状态流转 | 状态直接挂在任务单上，不在子加工单中转 |
| 数量进度 | 计划、已收、已完成、报废、货损、待交出等数量字段迁入任务单 |
| 仓库记录（待加工仓、待交出仓） | 关联对象从子加工单 ID 改为任务单 ID |
| 交出记录、差异记录 | 关联对象从子加工单 ID 改为任务单 ID |
| 菲票绑定 | 从绑子加工单改为绑任务单 |
| 生产单 → 任务单自动生成 | 不变 |
| 需求明细行 | 不变，仍挂在任务单下 |

### 二、Web 页面变更（阶段 2）

**任务单列表页（重写为标准列表页）：**

- 使用 `renderStandardListPage`、`renderStandardListTable`、`renderTablePagination`
- 摘要卡片 48px 单行布局
- 列排序、列选择、列拖拽、冻结列全部支持
- 列显示/顺序/冻结列/每页条数按路由持久化；当前页和排序不持久化
- 1366×768 为标准分辨率
- 行内快捷操作按钮（当前状态允许的操作，不离开列表即可推进状态）

**任务详情页（改造）：**

- 左侧：任务信息、明细、流转记录（Tab 结构保留）
- 右侧：操作面板（原先在加工单详情页的 7 个状态操作：出库、收货、开工、完工、差异上报、发起交出、差异后重交）
- 去掉"子加工单"Tab，不再展示子加工单列表

**加工单详情页（删除）：**

- 页面文件删除
- 路由 `/special-craft/{slug}/work-orders/{id}` 重定向到 `/special-craft/{slug}/tasks/{taskId}`

### 三、PDA 端适配（阶段 3）

PDA 端操作对象从子加工单改为任务单：

| 涉及点 | 变化 |
|------|------|
| PDA 执行页操作按钮 | 指向任务单而非子加工单 |
| PDA 仓库领料/交出按钮 | 关联任务单 ID |
| PDA 任务列表 | 展示任务单作为任务项 |
| 成衣 SKU 逐行输入 | 按任务单的行明细操作，输入方式不变 |
| 操作流程（扫菲票→收货→开工→完工→交出） | 不变 |
| 仓库视图库存结构 | 不变 |

### 四、脚本和测试适配（阶段 4）

所有引用子加工单符号或 `work-orders/{id}` 路由的检查脚本和测试文件全部适配新结构。

## 不变项

- 工艺字典定义、运营分类、专属工厂
- 仓储流转规则
- 生产单 pipeline 任务单生成逻辑
- 菜单和路由顶层结构

## 标准列表页合规要求

列表页重写必须满足：
- 顶部声明 `// @page-pattern: list`
- 使用 `renderStandardListPage` / `renderStandardListTable` / `renderTablePagination`
- 48px 单行摘要卡片
- 三态排序图标、列选择/拖拽/冻结
- 必需列不可隐藏，操作列固定右侧
- 偏好按路由持久化（不含当前页和排序）
- `npm run check:list-page-governance` 通过

## 验证

每个阶段完成后 `npm run build` 通过；全部完成后：
- `npm run check:prototype-design-governance -- --all` 通过
- `npm run check:list-page-governance` 通过
- 页面可正常浏览和操作
