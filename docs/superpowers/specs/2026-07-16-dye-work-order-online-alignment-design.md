# 单张染色加工单线上对齐设计

## 1. 背景与目标

当前线上运行并获得业务认可的染色加工单，以“列表集中查看、弹窗人工维护、日志追溯、流程卡打印”为核心。现有原型则更强调执行节点、移动端绑定、交出审核、配方和统计，页面重心与线上业务人员的使用习惯不同。

本次目标是让单张染色加工单在业务对象、字段、状态、操作和打印资料上与线上基本一致，同时继续遵守当前原型仓库的标准列表页规范。

本次设计只调整单张染色加工单及其与 PDA 共用的状态口径，不修改合并染色任务、合并染色分配规则或合并染色历史。

### 1.1 版本覆盖关系

本设计是 `2026-07-15-print-dye-requirement-removal-and-combined-dyeing-design.md` 在“单张染色加工单页面表达和跨端状态协作”范围内的后续规格。两份文档同时有效，按以下规则解释：

- 染色需求单、印花需求单及其业务单号已经删除，本设计不得重新引入需求单号、需求满足明细或需求单跳转。
- 单张染色加工单页面不展示合并染色徽标、任务号和分配结果；合并染色仍在独立菜单和独立业务对象中保留全部规则与历史。
- 加工单自动生成、按备货创建、平台加工单号唯一、生产单变更快照和合并染色领域规则继续沿用前一份规格。
- 本设计新增的七个执行状态不替代“工厂分配状态”；两者是不同维度。

## 2. 角色与页面定位

| 项目 | 定义 |
| --- | --- |
| 系统 | PFOS 工艺工厂运营视角，关联 PDA 现场执行端 |
| 页面 | 染色加工单 |
| 页面路径 | `/fcs/craft/dyeing/work-orders` |
| 主要角色 | 染厂业务人员、染厂主管、现场执行人员 |
| 主要任务 | 查询加工单、人工维护加工信息和状态、查看修改历史、导出数据、打印染整生产流程卡 |
| 上游来源 | 正式生产单自动生成，或业务人员按备货创建 |
| 下游去向 | PDA 接单与执行、交出、接收方入库 |

PFOS 页面属于主管和业务操作端，可以保持较高信息密度；PDA 仍是现场动作入口，只显示当前对象、当前动作和必要数量。

## 3. 设计原则

1. 线上业务字段和操作方式优先，不机械复制旧页面视觉。
2. 列表使用当前仓库的标准列表能力，支持分页、排序、列显示、列顺序、冻结和右侧固定操作栏。
3. 平台加工单号是唯一染色加工单号，PFOS、PDA、日志和打印卡不得生成或展示另一套工厂加工单号。
4. PFOS 允许人工维护状态；PDA 通过现场动作按顺序推进同一个状态字段。
5. 查看、编辑和日志都在当前列表页用弹窗完成，不再进入独立复杂详情页。
6. 单张加工单页面不承载合并染色任务信息、配方管理、执行节点详情、移动端绑定校验、交出审核和统计分析。

## 4. 页面整体结构

页面由五部分组成：

1. 查询区：承载线上已有的业务查询条件。
2. 批量操作区：导出、导出备料数据、导出超期未完结、批量打印流程卡。
3. 汇总区：展示当前筛选范围内的主要数量。
4. 染色加工单标准列表：集中展示线上认可的业务字段。
5. 弹窗：查看、编辑、日志；打印流程卡使用独立打印页面。

宽表只能在表格容器内部横向滚动。平台加工单号固定在左侧，操作栏固定在右侧；页面主体不得产生横向溢出。

## 5. 查询与批量操作

### 5.1 查询条件

查询区保留以下条件：

- 查询项：染色加工单号、任务单号、生产单号、采购单号、商品编码等。
- 查询内容：与所选查询项配合使用。
- 状态：等待处理、取消、染色中、染色完成、待审核、部分入库、已完成。
- 销售类型。
- 生产工厂。
- 染色工艺。
- 面料接收人。
- 是否纱线。
- 是否补料。
- GTG 仓是否有库存。
- 物料类型。
- 染色色号。
- 时间类型、开始时间、结束时间。
- 成分、幅宽、克重。
- 每页条数。

“查询”应用当前条件，“重置”恢复稳定默认条件。筛选输入不得在每次输入时触发整页重绘。

### 5.2 批量操作

| 操作 | 规则 |
| --- | --- |
| 导出 | 导出当前筛选结果，不要求勾选 |
| 导出备料数据 | 导出当前筛选结果中的备料相关字段 |
| 导出超期未完结 | 导出预计完成时间已过且状态不是“取消”或“已完成”的加工单 |
| 批量打印染整生产流程卡 | 仅打印已勾选加工单；未勾选时阻止并明确提示 |

批量打印按照用户勾选顺序生成，每张加工单单独分页。

## 6. 汇总区

汇总区展示当前筛选结果的：

- 计划数量。
- 原料使用数量。
- 完成数量。
- 损耗数量。
- 采购单数量。

所有数量必须带单位。不同单位不能直接相加；如果筛选结果存在多种单位，应按单位分别汇总，不生成误导性的总数。

- 平台染色加工单号。
- 任务单号。
- 生产单号。
- 染色色号。
- 销售类型。
- 当前状态。
- 工厂分配状态：待分配工厂或已分配。
- 是否补料。

平台加工单号为必需列、只读且不可改号。

- 采购单号。
- 采购类型。
- 面料接收人库存。
- GTG 仓库存。

库存数量使用红色只表达短缺或阻断，不把普通库存值全部染红。

### 7.3 原料与面料信息

- 面料图片。
- 面料名称。
- 原料 SKU。
- 颜色 SKU。
- 成分。
- 幅宽。
- 克重。
- 当前库存。
- 待染数量。

图片、名称、颜色和数量共同帮助业务人员识别面料，不能只依赖长编码。

### 7.4 属性、工厂和时间

- 染色工艺。
- 深浅。
- 温度。
- 生产工厂。
- 下单时间。
- 预计完成时间。
- 完成时间。
- 交货时间。

### 7.5 数量与附加信息

- 计划数量。
- 原料使用数量。
- 原料卷数。
- 备料数量。
- 备料重量。
- 完成数量。
- 损耗数量。
- 面料接收人。
- 交出单号。
- 头缸/复染。
- 待回写数量。
- 差异数量。
- 异议数量。
- 备注。

差异必须直接表达“多、少或一致”，不能只显示正负数。

### 7.6 行操作

固定操作栏只保留：

- 查看。
- 编辑。
- 日志。
- 打印流程卡。

不再提供“查看配方”“查看统计”或跳转独立复杂详情页的入口。

## 8. 查看与编辑弹窗

### 8.1 查看弹窗

查看弹窗只读展示与编辑弹窗相同的业务字段，并补充加工单号、任务单号、生产单号、商品、采购和面料识别信息。弹窗内不展示执行节点、移动端绑定、配方、交出审核和统计。

### 8.2 编辑弹窗

编辑字段如下：

| 字段 | 编辑规则 |
| --- | --- |
| 预计完成时间 | 人工维护 |
| 生产工厂 | 从具备有效染色能力的工厂主数据中人工选择；未选择时显示“待分配工厂” |
| 面料接收人 | 人工填写 |
| 状态 | 人工选择七个统一状态中的任意一个 |
| 深浅 | 浅色或深色 |
| 温度 | 190、200 或 205 |
| 计划数量 | 由生产单或备货来源带出，只读 |
| 原料数量 | 人工填写，不得为负数 |
| 原料卷数 | 人工填写，必须为非负整数 |
| 完成数量 | 人工填写，不得为负数 |
| 损耗数量 | 人工填写，不得为负数 |
| 备注 | 人工填写 |

编辑弹窗不根据排缸、备料或内部执行节点限制 PFOS 人工选择状态。取消状态或把状态回退到前序状态时必须二次确认，并提示该修改会影响 PDA 当前可执行动作。

生产工厂和执行状态是两个独立维度。未分配工厂时，加工单仍然生成，工厂分配状态为“待分配工厂”，执行状态为“等待处理”；此时 PDA 没有归属工厂，不允许接单。分配工厂后，执行状态保持不变，由 PDA 明确接单。

## 9. PFOS 与 PDA 共用状态

PFOS 和 PDA 读取并更新同一个加工单状态，统一使用以下七个状态：

- 等待处理。
- 取消。
- 染色中。
- 染色完成。
- 待审核。
- 部分入库。
- 已完成。

PDA 通过现场动作顺序推进，PFOS 可以人工选择任意状态。

```mermaid
flowchart LR
  A["加工单生成：等待处理"] --> B["PDA 接单：仍为等待处理"]
  B --> C["PDA 开工：染色中"]
  C --> D["PDA 完工：染色完成"]
  D --> E["PDA 交出：待审核"]
  E --> F["部分入库：部分入库"]
  E --> G["全部入库：已完成"]
  F --> G
  A --> H["主管取消：取消"]
  B --> H
  C --> H
```

具体映射：

| PDA 或接收动作 | 加工单状态 | 业务结果 |
| --- | --- | --- |
| 加工单生成、尚未接单 | 等待处理 | PDA 主动作是“接单” |
| 接单 | 等待处理 | 记录接单人和接单时间，PDA 下一动作变为“开工” |
| 开工 | 染色中 | PFOS 同步显示染色中 |
| 完工 | 染色完成 | 记录完成数量、损耗数量和完成时间 |
| 交出 | 待审核 | 记录交出人、时间、数量和交出单号 |
| 接收方部分入库 | 部分入库 | 展示已入库、待入库和差异数量 |
| 接收方全部入库 | 已完成 | 流程结束 |
| 主管取消 | 取消 | PDA 不再允许继续开工或交出 |

PDA 必须阻止跳过接单直接开工、未完工直接交出、重复接单、重复开工和重复交出。

PFOS 人工将状态调回前序状态时，只改变当前状态，不删除已经存在的 PDA 操作历史。PDA 根据最新状态重新判断可执行动作。

### 9.1 PFOS 人工状态修改影响矩阵

PFOS 人工改状态是主管兜底方式，只修改线上执行状态、更新时间、必要的状态时间字段和永久日志，不伪造不存在的完成记录、交出单、收货记录或入库记录。

| 人工修改 | 必要处理 | 不得产生的结果 |
| --- | --- | --- |
| 改为等待处理 | 二次确认；保留接单、开工、完成、交出和入库历史 | 不删除历史，不自动把底层任务重置为未执行 |
| 改为染色中 | 记录状态和日志 | 不自动生成开工凭证或交出单 |
| 改为染色完成 | 记录状态、完成时间和日志 | 不自动生成交出单 |
| 改为待审核 | 记录状态、交货时间和日志 | 不自动生成交出单或收货记录 |
| 改为部分入库 | 二次确认；保留已有收货事实 | 不伪造实收数量 |
| 改为已完成 | 二次确认；保留已有收货事实 | 不伪造全部入库记录 |
| 改为取消 | 除“已完成”外允许；二次确认；PDA 立即阻断 | 不删除任何历史事实 |
| 从取消恢复 | 二次确认；恢复为所选状态；根据当前状态重新判断 PDA 动作 | 不自动恢复或伪造业务事实 |

“已完成”是终态，不允许再改为取消或其他状态。部分入库可以人工取消，但已经存在的入库事实永久保留。所有取消、从取消恢复和状态回退都属于高风险修改。

### 9.2 工厂改派

- 未执行和进行中的加工单均允许主管改派工厂。
- 改派不回退当前执行状态，也不删除原染厂已形成的执行事实。
- 保存改派后，原染厂立即失去该任务的操作权限；加工单和 PDA 任务的接单责任同时重置。
- 新染厂必须重新接单。重新接单不改变当前执行状态，并从该状态允许的下一现场动作继续。
- 改派前后工厂、操作人、时间和原因写入永久日志。

## 10. 操作日志

所有 PFOS 编辑和 PDA 状态动作写入同一份操作日志。每条日志必须记录：

- 操作时间。
- 操作人。
- 操作端：PFOS 或 PDA。
- 具体动作。
- 修改前状态和修改后状态。
- 每个被修改字段的修改前值和修改后值。
- 必要的备注或原因。

日志永久保留。状态回退、取消、重复修改不能覆盖或删除历史记录。

## 11. 染整生产流程卡

流程卡采用线上业务认可的中印双语结构，二维码绑定平台加工单 ID，卡片显示的平台加工单号与列表、编辑和日志完全一致。

流程卡包含：

- 卡片序号和二维码。
- 下单日期、是否加急、开单日期、生产单号或备货来源。
- 色样图片和色样备注。
- 纱线 RAINBOW 色卡、TPG 色卡、辅料 GCC 色卡、TCX 色卡。
- 样衣 SPU、布料样品 SPU、翻单 SPU。
- 布料供应商、批号、原料名称。
- 染色色号、原料 SKU、数量、平台加工单号、颜色 SKU、卷数。
- 成分、幅宽、克重。
- 备料要求：松布、装袋、卷支、络筒。
- 现场工序签认区：复样、染色、脱水、开幅、烘干、定型、包装、出货。

生产单来源的加工单只展示生产单号，不展示或推导需求单号。备货加工单对应位置展示“备货创建”，不伪造生产单号或需求单号。

## 12. 数据流与一致性

1. 正式生产单生成后，系统自动生成染色加工单，并带出生产单、商品、采购、面料和计划数量快照。
2. 按备货创建的加工单沿用现有入口，来源明确显示为备货。
3. PFOS 编辑、PDA 动作和接收方入库更新同一张加工单。
4. 列表、查看、导出、日志和流程卡均读取同一张加工单及其不可删除的操作历史。
5. 加工单号、生产单号、任务单号和计划数量不得在编辑弹窗中修改。
6. 列表、导出和流程卡不得读取或展示已删除的染色需求单号；历史 `sourceArtifactIds` 不作为需求单号回退来源。

## 13. 异常与防错

- 数量格式错误或出现负数时阻止保存，并说明具体字段和修改方式。
- 原料卷数不是整数时阻止保存。
- PDA 状态不允许当前动作时阻止操作，并提示下一步应做什么。
- PFOS 取消或回退状态必须二次确认。
- PFOS 从取消恢复、改为部分入库或改为已完成必须二次确认。
- 已完成状态不得取消或回退。
- PFOS 修改状态后，必须明确提示 PDA 当前动作可能随之改变。
- 批量打印未勾选加工单时阻止操作。
- 重复提交不得产生重复业务结果，但要给出可理解的提示。

## 14. 原型数据要求

演示数据至少覆盖：

- 等待处理且尚未接单。
- 已接单但仍为等待处理。
- 染色中。
- 染色完成。
- 待审核。
- 部分入库并存在待入库数量。
- 已完成。
- 已取消。
- 原料库存不足。
- 补料加工单。
- 超期未完结加工单。
- PFOS 人工回退状态并保留 PDA 历史。
- 生产单来源和备货来源。

## 15. 非本次范围

- 不修改合并染色任务、成员锁定、分配、更正或删除规则。
- 不把多张染色加工单合并成一张加工单。
- 不新增后端、接口、权限或真实数据库。
- 不建设独立配方管理、排缸管理或执行节点管理页面。
- 不调整印花加工单。

## 16. 验收标准

1. 列表字段、查询、汇总和四个批量操作与线上业务口径一致。
2. 页面通过标准列表治理，具备分页、三态排序、列显示、列顺序、冻结、右侧固定操作栏和路由级偏好持久化。
3. 查看、编辑和日志均在弹窗内完成，不再进入独立复杂详情页。
4. PFOS 可人工选择七个状态，PDA 可通过接单、开工、完工、交出和入库推进同一个状态。
5. PDA 状态动作映射和顺序阻断正确。
6. 平台加工单号在 PFOS、PDA、日志和流程卡中完全一致且不可修改。
7. 中印双语流程卡可单张打印和批量打印。
8. 1366×768 和 1280×720 下主要操作可见，宽表只在表格容器内部滚动。
9. 原型治理检查、列表治理检查、染色业务专项检查和构建通过。
10. 未分配工厂与执行状态分维度展示；未分配工厂不能进入 PDA 接单范围。
11. 列表、导出和流程卡不展示需求单号。
12. 取消、恢复、回退、部分入库和已完成等人工修改符合影响矩阵，不伪造下游业务事实。
13. 进行中改派保持当前执行状态，原染厂失去操作权，新染厂重新接单后继续。

## 17. 设计审查结论

本方案参考以下设计规范：

- 角色与端类型：PFOS 按主管/业务操作端设计，PDA 按员工执行端设计。
- 现场任务：PDA 保留接单、开工、完工和交出等动作，不让员工人工选择完整状态机。
- 数量与状态：所有数量带单位，差异直接表达，PFOS/PDA 共用状态。
- 防错与追溯：PDA 顺序阻断，PFOS 高风险修改二次确认，所有修改永久留痕。
- UI 与设备：标准列表承载管理端密度，操作栏固定，兼顾低分辨率。
- 跨端协作：列表、PDA、入库、日志和打印读取同一张加工单。

设计阶段结论：通过。当前没有需要突破项目级原型规范的例外。

## 18. 统一事实型 Mock 补充设计（2026-07-17）

### 18.1 事实来源

Mock 不得在列表展示层单独伪造状态或数量。生产单快照确定加工单的生产单号、商品、物料和计划数量；执行节点确定实际投入、实际产出与损耗；交出记录确定交出数量；接收记录确定累计实收、差异与待入库数量；PFOS 和 PDA 只读取这些共同事实。

### 18.2 数量不变量

- 正常场景满足 `计划量 ≥ 实际投入 ≥ 实际产出 ≥ 交出量 ≥ 累计实收量 ≥ 0`。
- 损耗等于实际投入减实际产出。
- 待入库数量等于交出数量减累计实收数量。
- 未开工不得出现实际投入、实际产出、损耗、交出或实收数量。
- 染色中允许只有实际投入；染色完成必须有实际投入和实际产出。
- 部分入库必须已有大于 0 的实收数量，且实收小于交出数量。
- 已完成必须足额收货，且待入库数量为 0。
- 取消可以保留取消前已形成的执行、交出和收货事实，但不得凭状态修改新增事实。

### 18.3 场景矩阵

可见 Mock 至少覆盖：待分配工厂、已分配待接单、已接单待开工、正常染色中、补料染色中、染色完成待交出、已交出待审核、少收后部分入库、收货差异后取消、足额入库完成、备货创建、超期未完结、含水溶加工单以及合并染色成员加工单。

其中，待分配工厂的加工单保持“等待处理”且不能被 PDA 接单；备货加工单不生成生产单号；合并染色成员仍是独立加工单，不在单染色列表伪造合并任务数量。

### 18.4 稳定演示要求

- 所有场景必须通过真实列表读取可见，不能只存在于不可达的种子数据中。
- 超期与正常交期使用相对当前日期生成的稳定演示时间，避免固定日期导致全部超期。
- 至少覆盖两个有效染厂和一个待分配工厂场景，工厂归属必须与 PDA 任务一致。
- 专项检查逐条验证场景存在、数量不变量、来源合法性和跨端同号。
