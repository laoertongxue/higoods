# 生产对象总览跨系统编号点击规格

日期：2026-07-01
状态：方案 B 已确认，待书面规格审查

## 1. 背景

当前系统已经有“生产对象总览”和“查生产”的雏形，但入口主要集中在 FCS Web 的部分页面。业务侧现在需要进一步统一：无论用户在 FCS、WMS、PFOS、PMS、PCS、PDA 或其他生产相关系统中看到生产单号、生产需求单号或上下游业务单据编号，都应该能直接点击编号，在当前页面右侧打开同一个“生产对象总览”。

这个需求的核心不是新增很多单据详情页，而是把“任意编号 -> 找到生产主线 -> 展示当前生产状态、卡点、责任和证据”做成跨系统一致能力。

业务前提：

- 一线和跟单人员不愿意多点一步，也很难接受“复制编号再搜索”的操作。
- 印尼现场人员综合培训成本高，界面必须直观，看到编号就能点。
- 生产链路跨多个系统，单独在某个列表页平铺所有信息会导致页面膨胀，也会造成数据口径混乱。
- 生产对象总览只负责看清楚，不负责替代 WMS、PFOS、PMS、PCS 等源系统的业务操作。

## 2. 产品目标

### 2.1 一句话目标

所有生产相关业务编号都成为可点击的“生产对象入口”，点击后在右侧侧边弹窗展示统一的生产对象总览。

### 2.2 用户要得到的答案

用户点击任意业务编号后，必须能快速知道：

- 这个编号是什么单据。
- 它属于哪个生产单或生产需求。
- 当前生产是否能继续。
- 如果不能继续，卡在哪里。
- 责任方是谁。
- 下一步应该谁处理、处理什么。
- 这个判断来自哪些单据和数据。

### 2.3 设计原则

- 点编号本身，不额外增加“查看总览”操作列。
- 所有系统的可交互生产相关编号使用同一套入口规则。
- 右侧总览是只读诊断层，不在里面修改源系统业务数据。
- 第一屏优先展示判断和卡点，完整明细放到二级 Tab。
- 页面不因为点击编号而跳转或刷新，用户原页面筛选、滚动和上下文必须保留。
- 窄屏和 PDA 使用同一生产对象总览语义，可以从右侧滑出并占满屏，但不另做一套业务逻辑。

## 3. 适用范围

### 3.1 系统范围

该能力适用于所有生产相关系统中的可交互页面：

- FCS：生产单、生产需求、任务生成、任务跟踪、配料、裁片、印花、染色、车缝、后道、质检相关页面。
- WMS：配料、领料、发料、入库、退料、调拨、仓配执行页面。
- PFOS：工厂执行、裁片、铺布、印花、染色、车缝、后道、质检、交接、回货页面。
- PMS：采购、供应商订单、面辅料到货、采购异常页面。
- PCS：商品、款式、技术包、样衣和生产前资料页面中与生产单有关的编号。
- PDA：现场执行页面中的生产相关编号，使用移动适配抽屉或全屏右滑层，语义与 Web 总览一致。
- 旧系统或迁移页面：只要出现可识别的生产主线编号，也应按同一规则打开总览。

### 3.2 页面例外

以下页面不强制提供点击总览：

- 纯打印结果页。打印纸面本身不可交互，但进入打印前的列表或详情页必须支持点击编号。
- 登录页、系统设置页、权限页、基础字典页等没有生产上下文的页面。
- 只展示图片、PDF、外部文件预览的静态页面。

- 不能关联生产主线的普通商品资料编号。

## 4. 核心概念

### 4.1 生产对象

生产对象是任何可以回溯到生产主线的业务对象。它可以是生产单本身，也可以是一个配料单、领料单、裁片单、质检单或入库单。

生产对象必须至少包含：

- 对象类型。
- 对象编号。
- 来源系统。
- 展示名称。
- 当前状态。
- 关联生产单或关联生产需求。

### 4.2 生产主线

生产主线是总览的归口对象，优先级如下：

1. 生产单。
2. 已生成生产单的生产需求。
3. 尚未生成生产单的生产需求。
4. 能关联到 SPU / SKU 但无法确认生产单的商品或物料资源。

总览的任务、物料、时间、数量、异常都围绕生产主线聚合。

### 4.3 被点击对象

用户点击的编号必须保留在总览顶部，例如：

> 当前查看：领料记录 PICK-20260701-001，关联生产单 143852。

这样用户不会感觉自己点了领料单，却被带到一个陌生的生产单页面。

### 4.4 生产对象总览

生产对象总览是一个跨系统只读抽屉。它展示源系统事实和判断摘要，但不直接提交业务动作。

允许展示：

- 当前判断。
- 任务进度。
- 物料配领。
- 关键时间。
- 关键数量。
- 异常和责任。
- 来源单据。
- 跳转源系统的入口。

不允许在总览中直接做：

- 修改 WMS 库存。
- 修改 PFOS 工厂执行状态。
- 修改 PMS 采购单。
- 修改 PCS 技术包。
- 审批、付款、结算等源系统动作。

## 5. 用户体验设计

### 5.1 编号点击规则

所有生产相关编号在页面上统一表现为可点击文本：

- 蓝色。
- 等宽字体优先。
- 鼠标悬停显示下划线。
- 点击不跳页。
- 点击后右侧打开生产对象总览。
- 当前页面的筛选、分页、滚动位置保留。

禁止同一页面出现两种入口：

- 不要既把编号做成可点击，又额外加“查看总览”按钮。
- 不要每个系统自己做一套不同的详情弹窗。

### 5.2 右侧抽屉形态

桌面 Web：

- 从右侧打开。
- 覆盖当前页面一部分宽度。
- 背景有遮罩但主页面上下文仍可识别。
- 抽屉内部滚动，不影响主页面滚动。
- 关闭后回到原页面位置。

低分辨率 Web：

- 1366x768 下必须可用。
- 抽屉可占更大宽度。
- 不允许出现三栏挤压、横向滚动、透明面板。

PDA / 窄屏：

- 仍从右侧打开，但可以占满屏。
- 第一屏只保留现场需要的信息：当前对象、生产单、状态、下一步、关键数量。
- 复杂 Tab 可以收起或折叠，但对象解析和判断逻辑必须与 Web 一致。

打印页：

- 打印成品页面不提供交互。
- 打印来源页面的编号仍应可点击。

### 5.3 抽屉第一屏

第一屏按以下顺序展示：

1. 当前判断：能否继续、不能继续原因、责任角色、责任人、下一步动作。
2. 被点击对象：对象类型、对象编号、来源系统、状态。
3. 生产主线：生产单号、生产需求单号、SPU、产品图、计划数量、主加工厂、跟单。
4. 四类关键证据：任务、物料、时间、数量。
5. 最高优先级异常：缺料、延期、数量冲突、质检未闭环、数据冲突。

第一屏不追求完整，而是让用户马上知道“现在该找谁、处理什么”。

### 5.4 Tab 结构

总览抽屉包含以下 Tab：

| Tab | 作用 | 内容 |
| --- | --- | --- |
| 总览 | 第一屏诊断 | 当前判断、主线信息、关键证据、最高风险 |
| 任务 | 完整任务单 | 工序、任务单号、工厂、状态、待办动作、责任方 |
| 物料 | 完整配领信息 | 物料、需求、采购、到仓、配料、领料、缺口 |
| 时间 | 关键节点 | 计划时间、实际时间、延期状态、更新时间来源 |
| 数量 | 关键数量 | 计划、裁片、送货、回货、质检、返工、瑕疵、入库 |
| 单据关系 | 上下游链路 | 从被点击对象到生产单的关系、来源单据、下游单据 |
| 异常与责任 | 问题闭环 | 异常类型、责任角色、证据、建议处理入口 |

### 5.5 默认定位规则

用户点击不同编号后，总览默认打开不同位置：

| 点击对象 | 默认 Tab | 高亮对象 |
| --- | --- | --- |
| 生产单号 | 总览 | 生产单 |
| 生产需求单号 | 总览 | 生产需求 |
| SPU / SKU | 总览或物料 | 对应商品、SKU 或生产主线 |
| 采购单、配料单、领料单 | 物料 | 对应物料行或仓储单据 |
| 裁片、铺布、飞票、印花、染色、车缝、后道任务单 | 任务 | 对应任务行 |
| 质检总单、质检单、复检单 | 数量 | 对应质检节点 |
| 交接单、回货单、入库单 | 数量 | 对应数量节点 |
| 异常单、退料单、退货单 | 异常与责任 | 对应异常行 |

## 6. 业务场景

### 6.1 跟单查生产单

跟单在 FCS 生产单列表看到生产单号 `143852`。点击后右侧打开总览，第一屏显示：

- 当前判断：暂不能继续。
- 原因：面料 A 欠 47.76 米，后道无法完整排产。
- 责任方：仓库。
- 下一步：补齐配料或确认替代物料。
- 证据：配料单、领料记录、采购到货状态。

跟单不需要进入 WMS 页面就能先判断问题归属。

### 6.2 仓库查配料单

仓库人员在 WMS 配料列表点击某个配料单。右侧总览默认打开“物料”Tab，并高亮这个配料单对应的物料行。

用户能看到：

- 该配料单属于哪个生产单。
- 该生产单总共需要哪些物料。
- 当前物料已配多少、已领多少、还缺多少。
- 缺料是否会影响裁片、车缝或后道。

### 6.3 工厂反馈裁片单

印尼工厂人员反馈裁片单号。国内业务人员在 PFOS 或 FCS 页面点击裁片单号，右侧总览默认打开“任务”Tab。

用户能看到：

- 裁片任务由哪个工厂执行。
- 当前裁片状态。
- 上游物料是否齐套。
- 下游印花、染色、车缝任务是否已经生成。
- 当前卡点是否属于工厂、仓库还是跟单。

### 6.4 后道质检分批执行

业务人员在后道质检列表点击质检单号。右侧总览默认打开“数量”Tab，并高亮本次质检单。

用户能看到：

- 生产单质检总单。
- 当前质检单的质检数量。
- 合格、返工、瑕疵数量。
- 如果 PDA 已填瑕疵数量但 Web 未补齐瑕疵原因数量，显示“质检未闭环”。
- 是否允许该生产单进入完成或入库判断。

### 6.5 管理层查异常

管理人员在异常看板看到一个退料单或质检异常单。点击编号后右侧总览默认打开“异常与责任”Tab。

用户能看到：

- 异常是否影响生产继续。
- 异常来自哪个源系统。
- 责任角色和处理人。
- 建议回到哪个系统处理。

## 7. 数据与解析逻辑

### 7.1 统一对象索引

所有可点击编号都必须进入统一生产对象索引。索引字段：

| 字段 | 说明 |
| --- | --- |
| objectType | 对象类型，如生产单、需求、物料、仓储单据、工序单、质检单 |
| objectId | 对象编号 |
| displayTitle | 展示标题 |
| sourceDomain | 来源系统，如 FCS、WMS、PFOS、PMS、PCS、PDA |
| primaryNo | 主要编号 |
| secondaryNo | 辅助编号 |
| relatedProductionOrderNo | 关联生产单号 |
| relatedDemandNo | 关联生产需求单号 |
| statusText | 当前状态中文 |
| ownerRole | 责任角色 |
| quantityText | 数量摘要 |
| routePath | 源系统页面入口 |
| defaultTab | 点击后默认打开的 Tab |
| highlightKey | 抽屉中需要高亮的对象 |
| updatedAt | 数据更新时间 |
| keywords | 搜索关键词 |

### 7.2 编号解析顺序

点击编号后按以下顺序解析：

1. 精确匹配 `objectId`。
2. 精确匹配 `primaryNo`。
3. 匹配历史编号或别名。
4. 匹配生产需求。
5. 匹配生产单。
6. 匹配物料或 SKU。
7. 匹配 SPU。
8. 无法确认时进入“未关联生产主线”状态。

### 7.3 多生产单匹配

一个物料 SKU 或 SPU 可能对应多个生产单。处理规则：

- 如果用户所在行已经有生产单上下文，优先使用当前行上下文。
- 如果没有上下文，抽屉展示匹配到的生产单列表，让用户选择。
- 不允许系统静默选一个不确定的生产单。

### 7.4 未关联状态

如果编号存在但无法关联生产单，抽屉不显示空白，必须展示：

- 当前编号。
- 识别到的对象类型。
- 来源系统。
- 当前状态。
- 不能关联的原因。
- 可尝试的搜索关键词。
- 源系统入口。

示例提示：

> 已找到领料记录 PICK-001，但未找到关联生产单。可能原因：历史数据未回写生产单号，或该记录来自非生产领料。

### 7.5 数据冲突

当不同系统数据冲突时，总览必须显式提示，不做静默覆盖。

典型冲突：

- FCS 显示已送货，PFOS 没有交接记录。
- WMS 显示已领料，工厂未接收。
- PDA 已填质检瑕疵数量，Web 未补齐瑕疵原因数量。
- 入库数量大于质检合格数量。
- 下游数量大于上游数量。

冲突展示规则：

- 第一屏展示最高风险冲突。
- “异常与责任”Tab 展示完整冲突列表。
- 每条冲突展示来源系统、来源单据、更新时间、建议核对方向。

## 8. 当前判断逻辑

总览第一屏的“当前判断”由以下规则综合得出：

1. 有阻断性异常时，显示“暂不能继续”。
2. 没有阻断性异常但有待确认事项时，显示“需要确认后继续”。
3. 关键任务、物料、时间、数量都满足时，显示“可以继续”。

阻断性异常包括：

- 缺料影响当前或下一工序。
- 必须完成的上游任务未完成。
- 当前任务没有工厂或负责人。
- PDA 或 Web 质检未闭环。
- 关键数量不守恒。
- 入库、交接、回货数量存在明显冲突。

判断必须包含：

- 状态：可以继续、需要确认、暂不能继续。
- 原因：一句话说明。
- 责任角色：采购、仓库、工厂、跟单、待确认。
- 责任人或责任组织。
- 下一步动作。
- 来源证据。

## 9. 数量与质检逻辑

生产对象总览中的数量口径必须保持守恒关系。

基础数量链路：

```text
计划数量 -> 裁片数量 -> 送货数量 -> 回货数量 -> 质检数量 -> 合格数量 / 返工数量 / 瑕疵数量 -> 入库数量
```

质检规则：

```text
质检数量 = 合格数量 + 返工数量 + 瑕疵数量
各种瑕疵原因数量之和 = 瑕疵数量
```

PDA 与 Web 分工：

- PDA 允许只填写瑕疵数量。
- Web 需要补齐有瑕疵数量的质检单对应的瑕疵原因数量。
- Web 未补齐前，该质检单不能算完成。
- 总览中必须显示“质检未闭环”，并把它作为可能阻断生产完成或入库的异常。

## 10. 来源系统边界

### 10.1 总览可以做的事

- 展示跨系统摘要。
- 显示当前判断。
- 汇总证据。
- 展示单据关系。
- 提供跳转源系统的入口。
- 提醒数据冲突。

### 10.2 总览不能做的事

- 不替 WMS 执行配料、领料、入库。
- 不替 PFOS 执行工厂任务。
- 不替 PMS 修改采购。
- 不替 PCS 修改技术包。
- 不替 PDA 执行现场扫码。
- 不做真实权限、审批、结算、付款。

### 10.3 源系统处理入口

当用户需要处理问题时，总览只提供“去源系统处理”的入口，例如：

- 去 WMS 查看配料。
- 去 PFOS 查看工厂任务。
- 去 PMS 查看采购单。
- 去质检单补齐瑕疵原因。

入口文案必须明确具体，不使用笼统的“去处理”。

## 11. 跨系统页面接入规范

### 11.1 页面展示规范

页面中出现生产相关编号时，应按以下优先级处理：

1. 生产单号、需求单号必须可点击。
2. 当前行内的业务单据编号必须可点击。
3. 上下游来源单据编号必须可点击。
4. 数量、状态、人员、工厂名称不默认可点击。
5. 如果编号无法关联生产主线，仍可点击打开未关联状态。

### 11.2 列表页

列表页不新增“生产对象总览”操作列。编号自身就是入口。

列表页点击编号后：

- 不改变当前筛选。
- 不改变当前分页。
- 不跳路由。
- 关闭抽屉后回到原列表状态。

### 11.3 详情页

详情页中的所有上下游单据编号也应可点击。点击后仍在当前页右侧打开总览，不覆盖详情主体。

### 11.4 弹窗和抽屉内

如果现有业务弹窗或抽屉内出现生产相关编号，也可以点击打开生产对象总览。打开方式：

- 优先在最外层右侧打开总览。
- 不在业务弹窗内部再嵌套小弹窗。
- 若层级过深，先关闭或压暗当前业务弹窗，避免遮挡。

### 11.5 PDA 页面

PDA 页面坚持现场执行优先：

- 只展示现场执行必要编号。
- 编号可点击打开同一生产对象总览的移动版。
- 移动版第一屏只展示当前对象、生产单、状态、下一步、关键数量。
- 不把复杂后台 Tab 全部堆在 PDA 首屏。

## 13. 首期建议范围

为了避免一次性改成大工程，首期按最小可用闭环推进：

1. 保留现有生产对象总览和查生产能力。
2. 把对象范围从 FCS 扩展为跨系统设计口径。
3. 统一编号点击规范。
4. 优先覆盖现有原型里已经出现的生产单、需求单、物料、仓储、加工、质检、交接编号。
5. 补齐缺失对象类型的索引。
6. 补充未关联、数据冲突、多生产单匹配三类状态。
7. 验证桌面、低分辨率、PDA 窄屏的可用性。

首期不做：

- 后端接口。
- 真权限。
- 真数据库。
- 真实跨系统写回。
- 每个源系统独立详情页。
- 大型架构重构。

## 14. 验收标准

### 14.1 功能验收

- 任意系统中生产相关编号可点击打开生产对象总览。
- 点击编号后右侧抽屉打开，不跳页，不刷新当前页面。
- 生产单号、需求单号、物料 SKU、仓储单据、加工任务单、质检单、交接单都能识别。
- 点击不同对象时默认 Tab 正确。
- 被点击对象在总览顶部明确展示。
- 无法关联生产单时展示未关联状态，不空白。
- 多生产单匹配时要求用户选择，不静默误判。
- 数据冲突时明确展示冲突来源。

### 14.2 业务验收

- 跟单能通过任意编号判断当前卡点。
- 仓库能通过配料 / 领料编号看到生产主线和缺口。
- 工厂相关人员能通过任务编号看到上下游状态。
- 管理人员能通过异常单据编号看到责任和证据。
- 质检未补齐瑕疵原因数量时，总览能提示未闭环。

### 14.3 体验验收

- 1366x768 下总览可用。
- 抽屉背景不透明。
- 抽屉内部可滚动。
- 不出现横向滚动。
- PDA 窄屏可打开移动版总览。
- 关闭抽屉后保留当前页面上下文。

### 14.4 代码与原型验收

- 复用现有生产对象总览、搜索索引和编号按钮能力。
- 不新增重复的详情弹窗体系。
- 不新增真实后端、权限、数据库或 API。
- 不把完整台账复制到每个业务页面。
- CodeGraph 修改后保持索引同步。

## 15. 风险与处理

| 风险 | 处理 |
| --- | --- |
| 页面里编号类型多，容易漏接 | 先按现有原型页面扫一遍编号，再用统一按钮能力替换 |
| 物料或 SPU 对应多个生产单 | 优先使用行上下文，无上下文时让用户选择 |
| 跨系统数据口径不一致 | 总览显示冲突，不静默覆盖 |
| PDA 信息过重 | PDA 只展示移动版第一屏，复杂信息后置 |
| 实现变成大重构 | 复用现有总览、索引、按钮，按页面逐步接入 |
| 用户误以为能在总览处理业务 | 文案明确“只读诊断”，处理动作跳源系统 |

## 16. 与当前项目的对应

现有可复用能力：

- 生产对象总览右侧抽屉。
- 生产全局搜索。
- 生产单 / 需求单编号按钮。
- 生产对象索引。
- 生产对象总览数据聚合。

需要设计约束的新增方向：

- 从 FCS Web 扩展为所有生产相关系统。
- 编号点击成为统一交互规范。
- 增加更多业务单据对象类型。
- 为不同对象设置默认 Tab 和高亮对象。
- 补齐未关联、多匹配、数据冲突状态。
- PDA 使用同一语义的移动适配展示。
