# 裁片放行四颜色矩阵与详细版本历史设计

## 1. 目标与边界

本次只增强裁片放行原型的演示数据和版本历史可读性：生产单 `PO14671` 的当前矩阵展示 Black、White、Navy、Red 四种成衣颜色，并提供 10 个可追溯版本。继续使用本地 Mock 仓储，不接入真实后端、权限或状态机。

页面服务裁床办公室文员和裁床主管。用户需要先看清当前能齐套多少，再判断目标与补料差异；历史用于解释某次铺布完成裁剪如何改变累计矩阵。

## 2. 不变的业务口径

- 矩阵从有效裁片事实计算，不从计划数量或目标数量倒推。
- 每个颜色、尺码、物料的可做成衣数由该物料所需裁片部位折算；颜色尺码的齐套数量取全部必需物料的最小值。
- 目标只能从当前尺码下各物料的可做成衣数量中选择，不允许自由输入。
- 目标确认保存候选矩阵快照；后续版本不能覆盖历史快照。
- 多余裁片是系统记录的事实，但车缝最低回货承诺只采用目标与齐套口径。
- 裁片单冻结后保留最后有效数量参与计算，并明确展示“不再更新”；关闭后迟到的铺布数据进入待处理区，不静默计入。
- 补料管理仍可独立创建，裁片放行只提供可选的目标快照入口。

## 3. 当前四颜色矩阵

四种颜色均使用 M、L、XL 三个尺码和 A、B、C、D 四种必需物料，形成 48 个比较点。计划数量只作参考。

| 颜色 | 物料 A | 物料 B | 物料 C | 物料 D | 当前齐套 | 确认目标 |
| --- | --- | --- | --- | --- | --- | --- |
| Black | 220 / 358 / 532 | 200 / 350 / 500 | 208 / 364 / 520 | 200 / 350 / 500 | 200 / 350 / 500 | 208 / 350 / 520 |
| White | 190 / 280 / 340 | 180 / 270 / 330 | 185 / 290 / 350 | 180 / 275 / 335 | 180 / 270 / 330 | 185 / 280 / 340 |
| Navy | 170 / 260 / 340 | 175 / 265 / 345 | 180 / 270 / 350 | 175 / 265 / 345 | 170 / 260 / 340 | 170 / 260 / 340 |
| Red | 160 / 240 / 315 | 150 / 235 / 300 | 165 / 250 / 320 | 155 / 238 / 305 | 150 / 235 / 300 | 165 / 250 / 320 |

表中每格均按 M / L / XL 排列。场景差异如下：

- Black：延续用户截图数据，目标确认后同时出现需补、刚好和多余。
- White：通过两次铺布累计，V7 前仍处于等待完整目标判断的过程态；V10 统一确认目标后产生混合差异。
- Navy：目标取当前齐套下限，目标确认后没有补料缺口，主要体现刚好和多余。
- Red：目标选择较高候选值，多个物料形成集中补料缺口。

生产单的目标状态是整体状态，因此 V10 确认时必须一次覆盖 4 色 12 个颜色尺码点；不能让同一生产单在当前版本中出现某颜色已确认、某颜色待确认。

## 5. 历史抽屉信息结构

采用已确认的 A 方案“详细事件卡片”，每页 5 条，默认按版本倒序。

卡片收起态直接展示：

- 版本、事件名称、时间、操作人。
- 裁片单和铺布单来源。
- 本次影响的颜色、尺码数量。
- 更新前与更新后的齐套数量摘要。
- 本次发生变化的物料点数量。

点击卡片后局部展开，不触发整页重绘。展开态展示：

- 每个受影响颜色尺码的齐套数量：更新前 → 更新后，并显示增量。
- 每个变化物料点的可做成衣数量：更新前 → 更新后，并显示增量。
- 该版本的来源状态和业务原因。
- V8 展示“已冻结，不再更新；最后有效数量继续参与计算”。
- V10 展示 12 个已确认目标和需补 / 刚好 / 多余的汇总数量。

裁片部位、片数、单件用量仍保留在矩阵单元格的点击详情中，不直接铺满历史卡片。

## 6. 数据与计算实现

Mock 仓储按事件顺序增量写入裁片事实，每次事件调用现有矩阵计算后保存快照。历史展示通过相邻版本快照计算差异，避免手写一份可能与矩阵不一致的历史摘要。

相邻版本差异计算规则：

- 物料点变化 = 当前版本可做成衣数量 − 上一版本同颜色、尺码、物料数量。
- 齐套变化 = 当前版本齐套数量 − 上一版本同颜色、尺码齐套数量。
- 新出现的颜色或单元格以上一版本数量 0 作为比较基准。
- 冻结、目标确认等无数量变化事件使用事件元数据和当前快照说明，不伪造数量增量。

- 当前矩阵同时展示 Black、White、Navy、Red 四个颜色分组。
- 标题显示当前版本 V10，目标已确认且覆盖 12 个颜色尺码点。
- Black 的当前数量和目标与本设计表一致。
- 历史共 10 条、每页 5 条、两页可翻阅。
- V1 至 V7、V9 均显示铺布完成裁剪的来源、数量变化和更新后齐套；V8 显示冻结语义；V10 显示目标汇总。
- 展开历史卡片只局部更新历史抽屉，不替换页面根节点。
- 1366×768 可读，矩阵横向滚动限制在容器内部。
- 构建、裁片放行专项检查、列表治理和原型设计治理通过。
