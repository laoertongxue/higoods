# 面料需求看板产品设计

## 背景

新增页面：仓储物流系统 WLS / 仓储管理 / 面料需求看板。

页面用于仓储主管和计划协同人员查看面料需求、库存分布、加工在途、采购在途和缺料/待调拨异常。页面是原型看板，不做真实调拨单、采购单、接口或状态机。

## 页面目标

- 按面料 SKU 识别当前生产需要的面料。
- 看到目标面料是否需要印花、是否需要染色。
- 看到目标面料及其原料面料在中央仓、目的仓和在途环节的数量。
- 明确哪些面料缺料，哪些只是待调拨。
- 明确异常触发原因、解除条件和差额。

## 页面入口

- 系统：WLS 仓储物流系统。
- 菜单分组：仓储管理。
- 菜单名称：面料需求看板。
- 建议路径：`/wls/fabric-demand-board`。

## 页面布局

页面从上到下分为三个板块：

1. 数据搜索区。
2. 数据统计区。
3. 数据展示区。

### 数据搜索区

搜索区放在页面顶部，用于先缩小面料范围。

支持字段：

- 关键词：面料名称、面料 SKU、面料 SPU。
- 面料类型：全部、直裁面料、印花面料、染色面料。
- 是否需印花：全部、需印花、不需印花。
- 是否需染色：全部、需染色、不需染色。
- 异常预警：全部、缺直裁面料、缺印花原料、缺染色原料、直裁待调拨、印花待调拨、染色待调拨。
- 仓库/目的仓：全部、中央仓面料仓、中转仓、印花厂待加工仓、染色厂待加工仓。

搜索区只影响当前看板统计和列表，不创建新业务对象。

### 数据统计区

统计区随搜索条件变化。

面料种类：

- 总数：当前筛选结果中的目标面料 SKU 数。
- 印染数量：当前筛选结果中需印花或需染色的目标面料 SKU 数。
- 直裁数量：当前筛选结果中不需印花且不需染色的目标面料 SKU 数。

面料长度：

- 印花中米数：目标面料需印花，且原料已进入印花加工中的米数。
- 染色中米数：目标面料需染色，且原料已进入染色加工中的米数。
- 裁剪中米数：目标面料已进入裁床或中转仓并被裁剪占用的米数。
- 采购中米数：目标面料或其原料仍处于采购中的米数。
- 库存米数：当前筛选结果中可用于后续动作的库存米数合计。

所有长度展示单位为米，避免和面料种类数量混淆。

## 数据展示区

列表采用宽表。一行代表一个目标成品面料 SKU。

### 面料信息

展示：

- 面料图片。
- 面料名称。
- 面料 SPU。
- 面料 SKU。
- 面料类型。
- 是否需印花。
- 是否需染色。

目标成品面料 SKU 是列表主对象。需要印花或染色时，原料面料不是主行，而是在当前行中作为辅助库存信息展示。

### 原料库存

仅当目标面料需印花或需染色时展示。

展示：

- 原料面料名称。
- 原料面料 SKU。
- 中央仓面料仓库存。
- 对应目的仓待加工库存。
- 原料库存是否覆盖本次加工需求。

目的仓定义：

- 需印花：印花厂待加工仓。
- 需染色：染色厂待加工仓。
- 直裁：中转仓。

### 库存数据

库存可能分布在多个仓，不展示成单一仓库字段。

展示：

- 总库存米数。
- 中央仓面料仓库存。
- 中转仓库存。
- 印花厂待加工仓库存。
- 染色厂待加工仓库存。
- 其他在途或待入库库存。

列表中优先展示有数量的仓，空仓不强行占满信息密度。

### 印花数据

展示：

- 待领料米数。
- 印花中米数。
- 待入库米数。

待领料表示原料仍未进入印花厂待加工仓或待加工仓不足。

### 染色数据

展示：

- 待领料米数。
- 染色中米数。
- 待入库米数。

待领料表示原料仍未进入染色厂待加工仓或待加工仓不足。

### 采购数据

展示：

- 采购中米数。
- 转运中米数。
- 待入库米数。

采购数据用于解释缺料为什么还未解除，不在看板首版中提供采购单创建能力。

## 异常预警规则

异常固定为六类：

- 缺直裁面料。
- 缺印花原料。
- 缺染色原料。
- 直裁待调拨。
- 印花待调拨。
- 染色待调拨。

### 缺料与待调拨的区别

待调拨：中央仓面料仓有料，但目的仓不足，需要把原料调拨到目的仓。

缺料：中央仓面料仓和目的仓合计仍不足，需要采购、补料或调整需求。

### 缺直裁面料

触发条件：

- 目标面料不需印花且不需染色。
- 中央仓面料仓库存 + 中转仓库存 < 直裁需求米数。

解除条件：

- 中央仓面料仓库存 + 中转仓库存 >= 直裁需求米数。

展示说明：

- 缺少可直接裁剪的目标面料。
- 展示缺口米数。

### 缺印花原料

触发条件：

- 目标面料需印花。
- 中央仓面料仓中的原料库存 + 印花厂待加工仓原料库存 < 印花需求米数。

解除条件：

- 中央仓面料仓中的原料库存 + 印花厂待加工仓原料库存 >= 印花需求米数。

展示说明：

- 缺少用于印花的原料面料。
- 展示原料 SKU 和缺口米数。

### 缺染色原料

触发条件：

- 目标面料需染色。
- 中央仓面料仓中的原料库存 + 染色厂待加工仓原料库存 < 染色需求米数。

解除条件：

- 中央仓面料仓中的原料库存 + 染色厂待加工仓原料库存 >= 染色需求米数。

展示说明：

- 缺少用于染色的原料面料。
- 展示原料 SKU 和缺口米数。

### 直裁待调拨

触发条件：

- 目标面料不需印花且不需染色。
- 中转仓库存 < 直裁需求米数。
- 中央仓面料仓库存 > 0。
- 中央仓面料仓库存 + 中转仓库存 >= 直裁需求米数。

解除条件：

- 中转仓库存 >= 直裁需求米数。

调拨方向：

- 中央仓面料仓 -> 中转仓。

### 印花待调拨

触发条件：

- 目标面料需印花。
- 印花厂待加工仓原料库存 < 印花需求米数。
- 中央仓面料仓原料库存 > 0。
- 中央仓面料仓原料库存 + 印花厂待加工仓原料库存 >= 印花需求米数。

解除条件：

- 印花厂待加工仓原料库存 >= 印花需求米数。

调拨方向：

- 中央仓面料仓 -> 印花厂待加工仓。

### 染色待调拨

触发条件：

- 目标面料需染色。
- 染色厂待加工仓原料库存 < 染色需求米数。
- 中央仓面料仓原料库存 > 0。
- 中央仓面料仓原料库存 + 染色厂待加工仓原料库存 >= 染色需求米数。

解除条件：

- 染色厂待加工仓原料库存 >= 染色需求米数。

调拨方向：

- 中央仓面料仓 -> 染色厂待加工仓。

## 数据建模边界

首版使用本地 Mock 数据和现有仓储/配料数据聚合。

需要补充的原型字段：

- `materialSpu`：面料 SPU。
- `requiresPrint`：是否需印花。
- `requiresDye`：是否需染色。
- `rawMaterialSku`：原料面料 SKU。
- `rawMaterialName`：原料面料名称。
- `centralFabricStockQty`：中央仓面料仓库存。
- `transferWarehouseStockQty`：中转仓库存。
- `printFactoryWaitProcessStockQty`：印花厂待加工仓库存。
- `dyeFactoryWaitProcessStockQty`：染色厂待加工仓库存。
- `purchaseInProgressQty`：采购中米数。
- `inTransitQty`：转运中米数。
- `waitingInboundQty`：待入库米数。

不新增真实后端、真实库存流水、真实调拨单状态机。

## 交互

- 搜索和筛选只更新统计卡片与列表。
- 异常标签可作为筛选入口。
- 行内可展示规则说明和差额。
- 首版不提供创建调拨单、创建采购单、编辑库存、批量处理。

## 验证范围

实现后需要验证：

- `/wls/fabric-demand-board` 页面可访问。
- 菜单中出现仓储管理 / 面料需求看板。
- 搜索筛选后统计和列表同步变化。
- 多仓库存能展示多个仓的库存。
- 需印花/需染色行能展示原料库存。
- 六类异常标签触发、解除和差额说明符合本文档规则。
- 页面不出现英文状态码。
- 运行 `npm run build`。
- 运行 `npm run check:prototype-design-governance`。
