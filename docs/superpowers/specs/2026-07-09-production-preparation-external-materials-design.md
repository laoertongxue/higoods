# 生产准备时效非系统内物料维护设计

## 1. 背景

生产准备时效页面的「确认生产准备工作项」已经要求跟单维护本次用料和做款 / 打板要求。当前物料选择只覆盖系统内已有物料，但业务上存在一批印尼本地常用面料、里布、网纱、雪纺等临时物料，它们还没有进入系统物料资料。

本次设计只解决生产准备时效原型里的非系统内物料选择问题，不扩展成完整物料主数据管理。

## 2. 目标

1. 跟单可以在生产准备时效页面维护非系统内物料清单。
2. 非系统内物料只维护物料名称，系统自动生成序号。
3. 跟单确认工作项时，每一行本次用料都可以独立选择物料来源。
4. 同一条准备记录允许同时使用系统内物料和非系统内物料。
5. 新增的非系统内物料刷新页面后仍然保留。

## 3. 不做范围

- 不维护非系统内物料编码、类别、颜色、图片、单位、库存、采购、价格。
- 不把非系统内物料同步到正式物料主数据。
- 不新增独立菜单或独立管理页。
- 不做删除能力，避免误删影响已确认记录。
- 不改变染色要求维护入口；潘通色号和染色说明仍在染色准备项的独立确认入口维护。

## 4. 页面入口

在生产准备时效页面右上角新增入口：`非系统内物料`。

入口行为：

- 点击后打开弹窗。
- 不跳转页面。
- 不影响当前筛选、分页、详情抽屉或确认工作项弹窗状态。

## 5. 非系统内物料弹窗

弹窗标题：`非系统内物料`。

弹窗内同时提供查看和新增：

| 区域 | 内容 |
| --- | --- |
| 新增区 | 输入框：`物料名称`；按钮：`新增` |
| 列表区 | 表格列：`序号`、`物料名称` |

新增规则：

- 物料名称必填。
- 序号从 `1` 开始，按 `1` 递增。
- 初始化数据按业务提供的顺序生成序号。
- 新增物料序号 = 当前最大序号 + 1。
- 不复用历史序号。
- 空名称不允许新增。

## 6. 初始化数据

非系统内物料初始化为业务提供的名称列表。每一行只保留完整物料名称，不拆中文名、英文名、规格型号。

示例：

| 序号 | 物料名称 |
| --- | --- |
| 1 | 印花雪纺Printing seruti S388-1 |
| 2 | 白色雪纺white seruti S388-1 |
| 3 | 黑色雪纺black seruti S388-1 |
| 4 | 黑色里布 black furing S256 |
| 5 | 白色里布white furingS256 |

完整初始化清单进入 mock 数据文件，顺序保持业务给出的原始顺序。

## 7. 本地保存

非系统内物料使用生产准备时效现有运行时存储方式保存。

保存内容：

| 字段 | 说明 |
| --- | --- |
| `serialNo` | 自增序号 |
| `materialName` | 物料名称 |

页面渲染时合并：

1. 初始化非系统内物料。
2. 浏览器本地新增的非系统内物料。

如果本地新增物料和初始化物料名称重复，仍允许保留；原型阶段不做去重规则，避免误判同名不同料。

## 8. 确认工作项弹窗调整

「确认生产准备工作项」弹窗的第 3 步「维护物料和做款要求」中，每一行物料新增 `物料来源`。

来源选项：

- `系统内物料`
- `非系统内物料`

每一行独立选择来源。

### 系统内物料行

保持现有行为：

- 从系统内物料下拉数据源选择。
- 展示图片、物料名称、物料编号、物料类型。
- 保存现有物料字段。

### 非系统内物料行

切换为 `非系统内物料` 后：

- 选择框数据源切换为非系统内物料清单。
- 展示序号和物料名称。
- 保存物料来源、序号、物料名称。
- 图片、系统编码、物料类型不强行造假。

## 9. 详情展示

准备详情里的「本次用料」需要展示物料来源。

系统内物料展示：

- 图片
- 物料名称
- 物料编码
- 物料类型
- 数量信息

非系统内物料展示：

- 来源：`非系统内物料`
- 序号
- 物料名称

非系统内物料不展示假的图片、编码、类别和库存数量。

## 10. 交互约束

- 新增非系统内物料后，弹窗列表立即出现新行。
- 确认工作项弹窗重新打开时，可以选择刚新增的非系统内物料。
- 切换物料来源时，只切换当前行，不影响其他物料行。
- 同一准备记录可以同时存在系统内物料行和非系统内物料行。
- 输入框交互不触发整页重绘。

## 11. 辅料下单操作

`辅料下单` 不是上传凭证型准备项，不要求上传文件或图片。

操作入口仍作为准备项操作按钮展示。点击后打开单独弹窗，维护面辅料采购单号。

弹窗规则：

- 支持填写多个面辅料采购单号。
- 支持修改已填写的面辅料采购单号。
- 每次提交记录更新时间。
- 最后一次更新时间作为 `辅料下单` 的实际完成时间。
- 只要存在至少一个有效面辅料采购单号，`辅料下单` 可视为已完成。
- 单号为空时不能提交完成。

详情展示：

- 展示全部面辅料采购单号。
- 展示最后更新时间。
- 不展示上传记录空态。

## 12. Mock 数据覆盖

需要覆盖以下场景：

1. 非系统内物料初始化列表可见。
2. 新增非系统内物料后序号递增。
3. 确认工作项弹窗里一行选择系统内物料，一行选择非系统内物料。
4. 详情里系统内物料和非系统内物料展示口径不同。
5. 刷新后新增的非系统内物料仍可选。
6. `辅料下单` 已完成记录展示多个面辅料采购单号和最后更新时间。
7. `辅料下单` 未完成记录可以通过操作弹窗填写多个面辅料采购单号。
8. `辅料下单` 不出现上传凭证要求。

## 13. 验收标准

1. 生产准备时效页面右上角存在 `非系统内物料` 入口。
2. 弹窗内可以查看初始化非系统内物料。
3. 弹窗内可以新增物料名称，序号自增。
4. 新增物料刷新后仍保留。
5. 确认工作项弹窗的每一行物料都有来源选择。
6. 来源为系统内物料时，沿用原有系统内物料选择和展示。
7. 来源为非系统内物料时，只展示序号和物料名称。
8. 同一条准备记录可以同时保存两种来源的物料。
9. 详情展示不为非系统内物料伪造图片、编码或类型。
10. `辅料下单` 操作弹窗支持多个面辅料采购单号。
11. 修改面辅料采购单号后，最后一次更新时间同步为完成时间。
12. `辅料下单` 不要求上传凭证，也不展示上传记录空态。
13. 现有染色要求确认、准备项依赖、上传和统计逻辑不受影响。
