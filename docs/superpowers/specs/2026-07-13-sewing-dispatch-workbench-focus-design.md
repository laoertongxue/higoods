# 车缝分配工作台精简与 SKU 分工厂设计

## 1. 文档目的

本文档用于明确车缝分配工作台的页面职责、直接派单交互、SKU 分工厂规则、主工厂规则、时效展示方式、性能要求和连续工序任务演示数据要求。

本设计在既有“含车缝任务交付与回货时效”规则基础上，进一步收敛车缝分配工作台的操作范围，解决以下问题：

1. 直接派单弹窗内容过多，重复展示主列表已经提供的判断信息。
2. 一个生产单可能按 SKU 分配给多家车缝工厂，但现有交互一次只能选择一家工厂。
3. 页面和弹窗存在不必要的整页刷新，无法稳定满足按钮响应要求。
4. 连续工序任务分配页面缺少能够直接演示的“车缝到后道”任务数据。

## 2. 页面定位

| 项目 | 内容 |
| --- | --- |
| 所属系统 | 工厂生产协同系统 |
| 页面名称 | 车缝分配工作台 |
| 主要角色 | 跟单、生产计划员 |
| 端类型 | 管理端 |
| 核心任务 | 将独立车缝任务中的每个 SKU 指定给一家承接工厂 |
| 上游来源 | 独立车缝任务、裁片与配料准备结果、工厂档案、工厂状态 |
| 下游去向 | 工厂接单、任务执行、交出、确认实收、交付与回货履约 |

页面必须遵循以下职责分工：

- 主列表负责帮助用户判断哪些 SKU 可以进入分配。
- 直接派单弹窗只负责完成“分给谁、谁是主工厂、何时起算、何时到期”。
- 工厂端负责接单、执行和交出。
- 接收方确认实收后，数量才进入履约统计。

## 3. 核心业务边界

### 3.1 独立车缝任务按 SKU 分配

1. 分配工厂的最小业务粒度是 SKU。
2. 同一个 SKU 的全部任务数量只能分配给一家工厂。
3. 同一个 SKU 不允许再按数量拆分给不同工厂。
4. 一个生产单可以包含多个 SKU，不同 SKU 可以分配给不同工厂。
5. 一个 SKU 完成分配后，不得再次出现在普通待分配范围中。
6. 若需要更换承接工厂，必须通过改派处理，不得新增一条并行分配覆盖原结果。

示例：

| SKU | 任务数量 | 承接工厂 |
| --- | ---: | --- |
| SKU-A | 400 件 | 工厂甲 |
| SKU-B | 400 件 | 工厂甲 |
| SKU-C | 400 件 | 工厂乙 |
| SKU-D | 400 件 | 工厂乙 |

该生产单存在两家车缝承接工厂，但每个 SKU 始终只有一家承接工厂。

### 3.2 连续工序任务不得在分配页拆分

含车缝连续工序任务继续保持整任务分配：

- 车缝到包装连续任务由同一家工厂完整承接。
- 裁片到包装连续任务由同一家工厂完整承接。
- 连续工序任务不得在分配页面按 SKU 拆分。
- 若业务需要不同 SKU 由不同工厂承接，必须在上游任务形成阶段生成范围不同的任务，不得在连续任务分配页面临时拆分。

### 3.3 主工厂保持唯一

1. 一个生产单在任意时点只能有一个主工厂。
2. 一个生产单可以有多家车缝承接工厂。
3. 其他车缝承接工厂属于 SKU 分配关系，不得并列成为主工厂。
4. 只有一家承接工厂时，系统自动将其确定为主工厂。
5. 存在多家承接工厂时，必须明确选择唯一主工厂。
6. 已有有效主工厂且该工厂仍承接本生产单 SKU 时，默认保留原主工厂。

## 4. 页面信息架构

### 4.1 主列表职责

主列表负责展示分配前判断信息，包括：

- 车缝任务与生产单。
- 款式和 SKU 范围。
- 每个 SKU 的任务数量。
- 当前是否可以分配。
- 不能分配时的主要原因。
- 已分配和待分配状态。
- 已指定的承接工厂。

裁片、毛织片、特种工艺裁片、配料准备等详细判断信息可以在主列表或任务详情中查看，但不得在直接派单弹窗中重复铺开。

主列表必须让用户在进入弹窗前完成“能不能分”的判断。

### 4.2 直接派单与竞价入口分离

主列表保留两个独立动作：

- 直接派单。
- 发起竞价。

两个动作分别打开不同弹窗，不在弹窗内提供分配方式切换。

本设计重点定义直接派单弹窗。竞价弹窗继续遵守“中标不等于接单，工厂确认后才启动履约时钟”的既有规则，但不得与直接派单字段混在同一个界面。

## 5. 直接派单弹窗

### 5.1 弹窗只保留五类内容

直接派单弹窗只展示：

1. 业务分配时间。
2. SKU 与承接工厂。
3. 唯一主工厂。
4. 交付时效要求。
5. 按比例回货时效要求。

底部只保留：

- 取消。
- 确认直接派单。

### 5.2 明确删除的内容

直接派单弹窗不得重复展示：

- 当前齐套数量。
- 裁床判断详情。
- 风险摘要。
- 配料前置明细表。
- 毛织片明细。
- 特种工艺裁片明细。
- 辅料库存明细。
- 实际操作时间输入框。
- 大段接单时效规则解释。
- 大段节点计算说明。

这些信息已由主列表或任务详情承担，弹窗重复展示会干扰用户完成分配动作。

## 6. SKU 与承接工厂交互

### 6.1 表格字段

| 字段 | 展示要求 | 是否可编辑 |
| --- | --- | --- |
| SKU | 展示 SKU 编码 | 否 |
| 颜色 / 尺码 | 展示当前 SKU 的颜色和尺码 | 否 |
| 任务数量 | 展示该 SKU 的完整任务数量和单位 | 否 |
| 承接工厂 | 选择该 SKU 的唯一承接工厂 | 是 |

不提供“本次分配数量”输入框。

### 6.2 批量设置工厂

1. 用户可以勾选一个或多个 SKU。
2. 选择“批量设置承接工厂”后，为所有勾选 SKU 填入同一家工厂。
3. 批量设置后，用户仍可逐行修改承接工厂。
4. 批量设置不改变 SKU 任务数量。
5. 未选择承接工厂的 SKU 必须突出显示，但不使用大段错误说明。

### 6.3 工厂选项

工厂选项只展示当前可承接车缝任务的工厂。

- 不具备车缝能力的工厂不展示。
- 黑名单工厂不进入可选范围。
- 当前不可派单的工厂不进入可选范围，或置灰并用短句说明原因。
- 不在弹窗内展开完整评级、风险和能力档案。

## 7. 业务分配时间

1. 业务分配时间可以回填。
2. 业务分配时间不得晚于提交动作发生时的当前时间。
3. 直接派单成功后，工厂自动接单。
4. 接单时间同步为业务分配时间。
5. 本次操作分给多家工厂时，各工厂使用同一个业务分配时间起算。
6. 修改业务分配时间后，所有具体时效时间必须立即刷新。
7. 实际操作时间由系统记录，不作为可编辑字段在弹窗中展示。

## 8. 具体时效时间展示

### 8.1 展示原则

弹窗不得只展示“接单后第几天”或“接单后多少小时”的抽象规则，必须根据业务分配时间展示具体截止日期和时间。

必须展示到年、月、日、时、分，因为所有节点按连续满 24 小时滚动计算，不能只展示日期。

### 8.2 独立车缝任务示例

业务分配时间为 `2026-07-12 14:55` 时：

| 节点 | 具体要求 |
| --- | --- |
| 交付完成 | `2026-07-21 14:55` 前完成 100% |
| 30% 回货 | `2026-07-16 14:55` 前确认实收达到 30% |
| 70% 回货 | `2026-07-20 14:55` 前确认实收达到 70% |
| 100% 回货 | `2026-07-21 14:55` 前确认实收达到 100% |

### 8.3 多工厂履约口径

1. 多家工厂的截止时间可以相同。
2. 每家工厂只按其承接 SKU 的任务数量计算目标数量和履约比例。
3. 不得把多家工厂的确认实收数量合并后判断某一家工厂是否达标。
4. 只有接收方确认实收的数量进入履约统计。
5. 超量实收正常累计，履约比例允许超过 100%。

## 9. 提交校验与结果

### 9.1 提交前校验

满足以下条件后，才允许确认直接派单：

1. 业务分配时间已填写且不晚于当前操作时间。
2. 每个 SKU 都已选择一家承接工厂。
3. 每个 SKU 只对应一家承接工厂。
4. 存在多家承接工厂时，已确定唯一主工厂。
5. 所选工厂在提交时仍处于可派单状态。

### 9.2 数据变化处理

主列表已经完成分配前判断，因此弹窗不重复展示判断明细。

若打开弹窗后，上游事实发生变化导致某个 SKU 已不可分配，只显示一条短提示：

> SKU-C 当前不可分配，请返回列表重新确认。

发生该情况时阻止提交，不在弹窗中重新展开全部裁片或配料明细。

### 9.3 提交结果

系统按承接工厂自动汇总 SKU：

```text
工厂甲
- SKU-A：400 件
- SKU-B：400 件
合计：800 件

工厂乙
- SKU-C：400 件
- SKU-D：400 件
合计：800 件
```

每家工厂分别形成独立的实际分配结果，并分别记录：

- 承接 SKU。
- 承接总数量。
- 业务分配时间。
- 接单时间。
- 交付截止时间。
- 30%、70%、100% 回货截止时间。
- 主工厂或其他承接工厂关系。

## 10. 页面性能要求

### 10.1 主列表

1. 主列表按任务分页，不一次性渲染全部任务。
2. 主列表只展示任务级摘要，不在每一行默认展开所有 SKU 的全部齐套明细。
3. 详细裁片、毛织片、特种工艺和配料信息进入详情查看。
4. 搜索输入不得在每次输入后触发整页重绘。
5. 翻页、筛选和选择任务只更新列表和必要统计区域。

### 10.2 直接派单弹窗

1. 打开和关闭弹窗不得重绘整页外壳。
2. 单行选择工厂只更新当前行、主工厂候选和时效区。
3. 批量设置工厂只更新被选择的 SKU 行和主工厂候选。
4. 修改业务分配时间只更新具体时效时间。
5. 选择主工厂只更新主工厂区域。
6. 图标初始化只作用于新插入或更新的区域。
7. 任一功能按钮点击后的可见反馈时间不得超过 200 毫秒。

## 11. 连续工序任务 Mock 要求

连续工序任务分配页面必须具备可直接演示的任务数据，不得用空状态代替核心场景。

至少覆盖：

1. 车缝到后道连续任务，待分配，可直接派单。
2. 车缝到后道连续任务，竞价中。
3. 车缝到后道连续任务，已直接派单并自动接单。
4. 裁片到车缝再到后道的连续任务，归入其他连续工序任务。
5. 不含车缝的其他连续工序任务。

Mock 数据必须保证：

- 覆盖工序名称与任务名称一致。
- 车缝到后道任务能够出现在默认页签。
- 待分配样例能够真实打开直接派单和竞价弹窗。
- 已分配样例能够展示承接工厂和接单结果。
- 连续任务始终按整任务分配，不提供 SKU 分工厂操作。

## 12. 验收标准

### 12.1 业务验收

- 一个生产单的不同 SKU 可以选择不同承接工厂。
- 同一个 SKU 不能选择多家工厂，也不能按数量拆分。
- 每个 SKU 的任务数量只读。
- 多家工厂承接时只能确认一个主工厂。
- 直接派单成功后所有承接工厂自动接单。
- 每家工厂独立计算交付与回货履约。

### 12.2 页面验收

- 直接派单和发起竞价使用两个独立弹窗。
- 直接派单弹窗只展示五类核心内容。
- 弹窗不再展示当前齐套、风险摘要和配料明细。
- 支持逐行选择工厂和批量设置工厂。
- 只有一家承接工厂时不要求用户重复选择主工厂。
- 时效区域展示具体日期和时间，不只展示规则天数。

### 12.3 性能验收

- 打开、关闭弹窗不发生整页闪烁和滚动位置丢失。
- 修改时间和工厂后，输入焦点保持稳定。
- 列表、弹窗和主工厂区域采用局部更新。
- 功能按钮点击后 200 毫秒内出现可见反馈。

### 12.4 Mock 验收

- 连续工序任务默认页签至少展示一条真实的车缝到后道任务。
- 默认页签不得只展示空状态说明。
- 其他连续工序任务页签至少展示含裁片和不含车缝两类任务。

## 13. 设计结论

本次设计最终确定：

> 主列表负责判断“能不能分”，直接派单弹窗只负责完成“每个 SKU 分给谁、谁是唯一主工厂、何时起算、何时到期”。

独立车缝任务按 SKU 指定工厂，同一个 SKU 不再按数量拆分；含车缝连续工序任务继续保持整任务由一家工厂承接。
