# 车缝派单自定义分配时间与时效预览设计

## 1. 文档状态

- 状态：已完成方案确认，待书面规格审查
- 日期：2026-07-17
- 适用系统：工厂生产协同系统（FCS）
- 适用页面：车缝分配工作台、连续工序任务分配

## 2. 背景

车缝相关任务在直接派单和改派时，需要允许业务人员回填真实的业务分配时间，并以该时间为统一起点，自动展示交付完成时间以及 30%、70%、100% 回货截止时间。

当前项目已经具备时效分类、满 24 小时滚动计算、业务分配时间校验和正式时效快照能力，但不同派单入口的页面表现不一致：

- 独立车缝直接派单已经展示完整时效。
- 独立车缝改派只有业务分配时间，没有时效预览。
- 车缝到后道直接派单和改派只展示回货节点，没有单独展示交付完成时间。
- 部分弹窗仍展示实际操作时间，偏离“精简、聚焦”的页面方向。
- 各页面各自维护预览代码，容易继续产生规则和展示差异。

## 3. 目标

本次调整必须实现：

1. 独立车缝任务和车缝到后道连续任务，在直接派单和改派时均可填写业务分配时间。
2. 业务分配时间默认当前时间，允许回填，但不得晚于提交动作发生时的当前时间。
3. 修改业务分配时间后，页面立即重新计算并展示具体截止日期和时间。
4. 四个入口统一展示交付完成、30% 回货、70% 回货和 100% 回货。
5. 预览结果与提交后生成的正式时效快照保持一致。
6. 页面只局部刷新时效区域，任一相关交互响应不超过 200ms。

## 4. 范围

### 4.1 本次覆盖

| 页面 | 动作 | 时效类型 |
|---|---|---|
| 车缝分配工作台 | 直接派单 | 车缝 |
| 车缝分配工作台 | 改派 | 车缝 |
| 连续工序任务分配 | 直接派单 | 车缝到后道 |
| 连续工序任务分配 | 改派 | 车缝到后道 |

### 4.2 本次不覆盖

- 发起竞价。
- 竞价定标和中标工厂确认接单。
- 裁片到包装。
- 非车缝任务分配。
- 已接单后的履约进度页面。
- 配料、产能和工厂能力等其他分配条件。
- SKU 分配粒度和主工厂业务规则。

竞价链路继续维持既有规则：发起竞价不启动履约时钟，中标工厂确认接单后再按实际接单时间生成正式时效。

## 5. 方案决策

采用“共享时效预览组件”方案。

保留现有任务页面、弹窗状态和派单提交逻辑，只抽取业务分配时间校验、时效预览模型和统一展示。独立车缝按 SKU 选择工厂，连续工序按整任务选择工厂，两种分配粒度不合并为通用派单弹窗。

未采用的方案：

- 分别在每个弹窗补代码：改动快，但会继续形成重复计算和展示漂移。
- 统一重写所有派单弹窗：独立车缝和连续任务的分配粒度不同，改动范围超过本次目标。

## 6. 时效规则

所有时间均从业务分配时间开始，按照满 24 小时滚动计算，不按自然日零点截断。

### 6.1 车缝

| 项目 | 计算规则 |
|---|---|
| 交付完成 | 业务分配时间 + 9×24小时 |
| 30% 回货 | 业务分配时间 + 4×24小时 |
| 70% 回货 | 业务分配时间 + 8×24小时 |
| 100% 回货 | 业务分配时间 + 9×24小时 |

### 6.2 车缝到后道

| 项目 | 计算规则 |
|---|---|
| 交付完成 | 业务分配时间 + 10×24小时 |
| 30% 回货 | 业务分配时间 + 5×24小时 |
| 70% 回货 | 业务分配时间 + 9×24小时 |
| 100% 回货 | 业务分配时间 + 10×24小时 |

“交付完成”和“100% 回货”即使时间相同也必须分别展示，因为两者代表不同的业务要求。

## 7. 统一页面结构

四个入口统一按照以下顺序组织信息：

1. 业务分配时间。
2. SKU 与承接工厂，或者连续任务的承接工厂。
3. 唯一主工厂确认。
4. 交付时效与按比例回货要求。
5. 取消和确认动作。

实际操作时间由系统在提交时记录，只用于时间校验和审计，不作为弹窗字段展示。

### 7.1 时效区域

时效区域固定展示：

```text
交付时效与按比例回货要求                   车缝

交付完成      2026-07-26 10:00 前完成 100%
30% 回货      2026-07-21 10:00 前确认实收达到 30%
70% 回货      2026-07-25 10:00 前确认实收达到 70%
100% 回货     2026-07-26 10:00 前确认实收达到 100%

按满 24 小时滚动；仅接收方确认实收计入回货比例。
```

车缝到后道任务的右上角类型名称显示为“车缝到后道”。

为了保持弹窗精简，预览区域不展开每家工厂的 30% 和 70% 目标件数。正式派单后，系统仍按每个有效分配记录的实际分配数量分别冻结目标件数。

## 8. 共享组件设计

新增轻量业务组件，建议位置：

- `src/components/sewing-delivery-sla-preview.ts`

组件只负责：

- 接收任务、业务分配时间和本次分配数量。
- 判断任务属于车缝还是车缝到后道。
- 调用现有时效计算规则。
- 生成交付完成、30%、70%、100% 四个具体时间。
- 输出统一的校验结果和展示内容。

组件不负责：

- 选择承接工厂。
- 确认主工厂。
- 提交派单或改派。
- 保存正式时效快照。
- 计算已接单任务的履约进度。
- 处理竞价任务。

现有 `src/data/fcs/sewing-delivery-sla.ts` 继续作为规则和快照的唯一事实来源，不在页面或组件中重复编写 9 天、10 天及各比例节点的计算公式。

## 9. 数据流

```mermaid
flowchart LR
    A["打开直接派单或改派弹窗"] --> B["业务分配时间默认当前时间"]
    B --> C["共享时效预览组件"]
    C --> D["识别车缝或车缝到后道"]
    D --> E["计算交付及30%/70%/100%时间"]
    E --> F["弹窗展示具体日期时间"]

    G["用户修改业务分配时间"] --> C

    F --> H["点击确认派单或改派"]
    H --> I["提交时再次校验不得晚于当前时间"]
    I --> J["自动接单时间同步为业务分配时间"]
    J --> K["生成正式时效快照"]
```

预览过程不得写入运行数据。只有派单或改派提交成功后，才生成正式分配记录和时效快照。

## 10. 页面调整

### 10.1 车缝分配工作台直接派单

- 使用共享组件替换页面内部的独立时效展示函数。
- 保持现有按完整 SKU 选择工厂的规则。
- 保持生产单唯一主工厂逻辑。
- 同一次操作中的全部 SKU 和承接工厂使用同一个业务分配时间。
- 修改时间时只刷新时效区域和时间错误提示。

### 10.2 车缝分配工作台改派

- 在业务分配时间下方增加共享时效预览。
- 预览对应的分配范围为原分配数量减去已经确认实收的数量。
- 修改时间时只刷新时效区域和时间错误提示。
- 确认改派时继续使用现有改派事务，生成剩余数量的新分配记录和新时效快照。

### 10.3 连续工序任务直接派单

- 使用共享组件替换现有只展示 30%、70%、100% 的局部预览。
- 补充独立的“交付完成”行。
- 页面时效类型显示“车缝到后道”。
- 删除实际操作时间展示。
- 删除与主列表重复的分配数量卡片。
- 保持连续工序任务按整任务分配。

### 10.4 连续工序任务改派

- 与直接派单共用同一个时效预览组件。
- 使用剩余未实收数量生成预览和正式快照。
- 删除实际操作时间展示。
- 保持现有改派原因和主工厂确认逻辑。

### 10.5 其他连续任务

不属于车缝到后道的连续工序任务不接入本次共享预览，不改变现有页面和提交规则。

## 11. 业务分配时间校验

- 弹窗打开时默认当前时间。
- 允许回填过去时间，不设置最早日期。
- 业务分配时间不能为空。
- 业务分配时间不得晚于点击确认时的系统当前时间。
- 页面变更时即时校验，提交时再次校验。
- 页面展示精确到分钟，正式记录保留到秒。
- 实际操作时间由系统在提交时生成并写入审计记录。

出现以下情况时不得继续显示上一次的具体截止时间：

| 场景 | 页面处理 |
|---|---|
| 时间为空 | 提示“请选择业务分配时间” |
| 时间格式错误 | 提示“业务分配时间格式不正确” |
| 时间晚于当前时间 | 提示“业务分配时间不能晚于当前时间” |
| 时效计算异常 | 隐藏具体截止时间并显示明确错误 |
| 任务不属于本次范围 | 不接入共享预览，不改变原页面 |

时间无效时确认按钮不可提交，提交逻辑仍必须执行相同校验，不能只依赖按钮状态。

## 12. 改派处理

改派时：

1. 读取原任务当前生效的分配记录。
2. 汇总截至当前操作时间已经由接收方确认实收的数量。
3. 计算剩余未实收数量。
4. 使用新业务分配时间预览新的交付和回货截止时间。
5. 提交时再次计算剩余数量，避免弹窗打开后又发生实收。
6. 以最终剩余数量生成新分配记录和新时效快照。
7. 原任务和原时效快照保留为历史，不覆盖原履约事实。

如果没有剩余数量，不允许提交改派。

## 13. 局部刷新与性能

用户修改业务分配时间时：

- 不提交表单。
- 不关闭或重建弹窗。
- 不替换页面根节点。
- 不改变已经选择的工厂、SKU、主工厂和改派原因。
- 只更新时效区域、错误提示和确认按钮可用状态。
- 响应时间不超过 200ms。

## 14. 自动化测试

### 14.1 时效规则

覆盖：

- 车缝四个截止时间。
- 车缝到后道四个截止时间。
- 跨月、跨年和闰年。
- 业务分配时间等于、早于和晚于当前时间。
- 交付完成和 100% 回货时间相同但展示为两行。

### 14.2 共享组件

检查：

- 固定展示交付完成、30%、70%、100% 四行。
- 显示具体日期和时间。
- 显示正确的任务类型名称。
- 不展示实际操作时间。
- 无效时间不保留旧预览。
- 不属于范围的任务不会错误套用规则。

### 14.3 四入口交互

分别验证：

1. 独立车缝直接派单。
2. 独立车缝改派。
3. 车缝到后道直接派单。
4. 车缝到后道改派。

每个入口检查：

- 打开弹窗时默认当前时间。
- 修改时间后四个截止时间同步变化。
- 弹窗不闪烁、不关闭、不丢失其他输入。
- 页面根节点保持不变。
- 局部刷新不超过 200ms。
- 提交后自动接单时间等于业务分配时间。
- 正式时效快照与提交前预览一致。

### 14.4 回归范围

确认没有改变：

- 发起竞价和中标确认接单规则。
- 裁片到包装任务。
- 非车缝任务分配。
- SKU 不可按数量拆给多家工厂。
- 生产单唯一主工厂。
- 仅确认实收计入回货比例。
- 超量实收比例允许超过 100%。

## 18. 完成标准

只有同时满足以下条件才算完成：

1. 四个入口均可自定义业务分配时间。
2. 四个入口均展示交付完成、30%、70%、100% 的具体时间。
3. 修改时间后预览在 200ms 内局部更新。
4. 时间无效时不能提交且不展示旧截止时间。
5. 提交后的自动接单时间和正式时效快照与预览一致。
6. 竞价、裁片到包装及非车缝任务没有被改变。
7. 自动化检查、构建和浏览器验收全部通过。
