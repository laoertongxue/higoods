# 物料资源总览产品设计规格

## 背景

当前项目已有 `生产对象总览` 和 `生产全局搜索`：

- `生产对象总览` 是业务单据视角，用于判断一张生产单、需求单或工艺任务当前推进到哪、卡在哪里、谁负责下一步。
- `生产全局搜索` 是入口能力，用于按业务单据号、物料、异常关键词快速定位供应链对象。

本次设计补充点击 `物料编码` 后的产品边界。物料不是只挂在某一张生产单下的明细，而是跨生产单、需求单、采购、仓储、配料、领料、发料复用的供应链资源。因此点击物料编码不应只打开静态物料档案，也不应把所有物料全局信息塞回生产对象总览。

## 设计结论

采用 `A+B` 组合：

- `A`：新增独立的 `物料资源总览`，作为点击物料编码后的主页面。
- `B`：`生产对象总览 / 面辅料与仓储` 保留当前生产对象下的物料履约切片。

两者关系：

| 页面 | 主对象 | 解决问题 |
|---|---|---|
| 生产对象总览 | 一张业务单据 | 这张单怎么推进、卡在哪里、下一步谁处理 |
| 物料资源总览 | 一个物料编码 | 这个物料被谁占用、够不够、在哪里、影响哪些单 |
| 物料档案 | 一个物料主数据 | 这个物料的静态资料是什么 |

## 入口规则

| 点击对象 | 打开页面 | 默认定位 |
|---|---|---|
| 业务单据号 | 生产对象总览 | 总览 |
| 物料编码 | 物料资源总览 | 供需总览 |
| 物料档案入口 | 物料档案或档案区块 | 静态主数据 |

从生产对象总览点击物料编码时，打开 `物料资源总览` 并带入来源上下文：

- 来源单据号，例如 `PO-202603-0001`
- 来源位置，例如 `生产对象总览 / 面辅料与仓储`
- 当前物料行置顶、高亮
- 同一物料在其他生产单、需求单、采购、仓储中的占用仍然可见

禁止把来源单据作为全局过滤条件。来源只用于高亮和返回，不用于隐藏其他占用。

## 物料资源总览首屏

首屏定位为供需判断页，不做传统档案页。

首屏展示 5 个区块：

| 区块 | 字段 | 目的 |
|---|---|---|
| 物料身份 | 物料编码、名称、类型、规格、颜色、单位、主供应商 | 确认是否点错物料 |
| 当前判断 | 库存是否足够、缺口、影响单据数、最早影响时间 | 直接给业务结论 |
| 来源上下文 | 来源单据、本单需求、本单配料、本单领料、本单状态 | 保留用户来路 |
| 供需概览 | 总需求、可用库存、已锁定、在途采购、待检、缺口 | 判断供需是否成立 |
| 影响范围 | 影响生产单、影响任务、高优先级数量、最早交期 | 判断先处理谁 |

示例：

```text
FAB-2024-001｜T-shirt 面料｜面料｜白色 180g｜yd
当前判断：缺口 300 yd，影响 3 张生产单，最早影响 2026-07-02 裁片计划

来源：PO-202603-0001
当前单据：需求 600 yd｜已配料 600 yd｜待领料 600 yd｜状态：待领料

供需：总需求 2,400 yd｜可用 800 yd｜锁定 600 yd｜在途 1,000 yd｜待检 300 yd｜缺口 300 yd
影响：生产单 3 张｜裁片任务 2 张｜高优先级 1 张｜最早交期 2026-07-02
```

## 物料资源总览 Tab

第一版设置 5 个 Tab：

1. 供需总览
2. 业务占用
3. 库存与在途
4. 配料 / 领料 / 发料
5. 异常与档案

### 供需总览

展示汇总判断，不展示所有流水。

字段：

- 总需求
- 可用库存
- 已锁定
- 待检
- 在途采购
- 预计可用
- 缺口
- 最早影响交期
- 最近采购到仓
- 当前建议

业务结论示例：

```text
当前物料可用库存不足。PO-202603-0001 需 600 yd，已配料 600 yd，但裁片厂尚未领料；PO-202603-0002 仍缺 400 yd，需等待 PUR-20260629-003 到仓。
```

### 业务占用

回答谁在用这个物料。

字段：

- 业务类型：生产单、需求单、样衣单、补单
- 业务单据号
- SPU
- 颜色尺码
- 需求数量
- 已配料
- 已领料
- 缺口
- 交期
- 优先级
- 当前状态

交互：

- 来源单据置顶、高亮
- 点击业务单据号回到生产对象总览
- 支持筛选：全部、当前来源、缺料、待领料、近 7 天交期

### 库存与在途

回答货在哪里。

库存批次字段：

- 仓库
- 批次
- 总库存
- 可用
- 锁定
- 待检
- 冻结
- 单位

采购在途字段：

- 采购单号
- 供应商
- 采购数量
- 已到仓
- 未到仓
- 预计到仓
- 状态

入库记录字段：

- 入库单号
- 来源单据
- 到仓数量
- 仓库
- 到仓时间
- 质检状态

库存判断必须区分可用、锁定、待检、在途、冻结，不允许只展示总库存。

### 配料 / 领料 / 发料

回答物料在生产履约中走到哪一步。

字段：

- 业务单据号
- 工序 / 工厂
- 需求数量
- 已配料
- 待配料
- 已领料
- 待领料
- 已发料
- 待发料
- 缺口
- 下一动作

状态定义：

| 状态 | 含义 |
|---|---|
| 配料 | 仓库已为业务单据预留或分配物料 |
| 领料 | 工厂或执行方已经领取物料 |
| 发料 | 仓库已经发出物料 |
| 到仓 | 物料进入仓库 |
| 可用 | 到仓后可被生产使用，且不在待检、冻结或已被其他单占用状态 |

### 异常与档案

异常在前，档案在后。用户点击物料编码通常先处理履约风险，再查看静态资料。

异常字段：

- 异常类型：缺料、未到仓、待检未释放、配料不足、待领料、发料不足、超耗、退料、替代料待确认、库存冻结
- 影响单据
- 影响数量
- 责任方
- 发生时间
- 要求完成时间
- 当前状态
- 建议处理

档案字段：

- 物料编码
- 名称
- 类型
- 规格
- 成分
- 颜色
- 单位
- 默认供应商
- 采购周期
- 最小采购量
- 默认损耗率
- 替代料
- 适用品类 / SPU
- 主数据状态

## 生产对象总览中的物料切片

`生产对象总览 / 面辅料与仓储` 只展示当前生产对象下的物料履约切片，不展示跨单占用大表。

每个物料行字段：

- 物料类型：面料、辅料、纱线、包材
- 物料编码：可点击，进入物料资源总览
- 物料名称
- 需求数量
- 已采购
- 已到仓
- 已配料
- 待配料
- 已领料
- 待领料
- 已发料
- 缺口
- 当前状态：缺料、未到仓、待配料、待领料、已发料
- 下一步：仓库配料、工厂领料、采购跟进等

允许给轻提示：

```text
该物料另有关联占用 2 张单
```

但不在生产对象总览内展开其他单据占用。点击提示或物料编码进入物料资源总览查看。

## 生产全局搜索

搜索结果按对象类型分组，不混排。

搜索物料编码时：

1. 物料资源
   - 物料编码、名称、类型
   - 当前判断：缺口、影响单据数
   - 关键数字：总需求、可用、在途、缺口
   - 点击进入物料资源总览
2. 相关生产对象
   - 生产单、需求单、工艺任务
   - 单据号、SPU、状态、当前卡点、交期
   - 点击进入生产对象总览
3. 相关采购与仓储
   - 采购单、入库单、配料单、领料单、发料单
   - 数量、预计 / 实际时间、状态、责任方
4. 异常线索
   - 缺料、未到仓、待检、待领料、冻结、替代料待确认
   - 影响对象、影响数量、责任方、建议动作

搜索业务单据号时：

1. 最佳匹配：生产对象
2. 该对象相关物料
3. 该对象相关任务 / 仓储 / 采购
4. 异常线索

搜索异常关键词时，例如 `缺料`、`待领料`、`未到仓`：

1. 当前卡点
2. 影响生产对象
3. 影响物料资源
4. 责任方 / 下一步

点击规则：

| 搜索结果类型 | 点击后 |
|---|---|
| 生产单 / 需求单 / 工艺单 | 生产对象总览 |
| 物料编码 | 物料资源总览 |
| 采购 / 入库 / 配料 / 领料 / 发料单 | 对应执行对象总览；若第一版未独立实现，则落到生产对象总览并定位证据 |
| 异常关键词 | 进入对应对象的异常视图，并定位该异常 |

## 第一版范围

要做：

- 新增 `物料资源总览`
- 物料编码点击进入物料资源总览
- 从生产对象进入时带来源上下文并高亮当前记录
- 生产对象总览保留本单物料履约切片
- 生产全局搜索支持物料资源分组
- 搜索物料编码能同时看到物料资源、相关生产对象、采购仓储、异常线索

不做：

- 不做真实库存扣减
- 不做真实采购创建
- 不做复杂审批
- 不做批量调整优先级
- 不做跨系统真实写回
- 不把物料档案做成维护后台

## 验收标准

1. 从生产对象总览点击物料编码，能打开物料资源总览。
2. 物料资源总览顶部能看到来源单据，并置顶高亮当前来源记录。
3. 同一物料的其他生产单占用仍可见，不能被来源单据过滤掉。
4. 生产对象总览里的物料页签只展示本单切片，不展示完整跨单库存大表。
5. 搜索物料编码时，结果中有 `物料资源` 分组。
6. 搜索 `缺料`、`待领料`、`未到仓` 能命中相关物料资源和生产对象。
7. 页面文案全部中文，状态不能出现英文码。
8. 低分辨率下不出现横向溢出，长表用卡片或内部表格控制。
9. 不引入新依赖，不迁移 React，不做真实后端。
10. 保留现有 `生产对象总览` 和 `生产全局搜索` 的已验证能力，不回退。
