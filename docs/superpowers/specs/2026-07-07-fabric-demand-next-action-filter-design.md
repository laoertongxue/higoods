# 面料需求看板「后续建议工作」筛选设计

## 1. 背景

面料需求看板当前已经支持按面料类型、印花 / 染色需求、异常类型、仓库 / 目的仓筛选。用户在处理异常时，真正关心的往往不是异常名称本身，而是下一步应该由谁做什么。

例如：

- 印花待调拨，本质是中央仓面料仓需要把原料调拨到印花待加工仓。
- 染色待调拨，本质是中央仓面料仓需要把原料调拨到染色待加工仓。
- 直裁待调拨，本质是中央仓面料仓需要把直裁面料调拨到中转仓。
- 缺料异常，本质是采购跟单需要跟进采购、转运或入库。

因此新增一个「后续建议工作」筛选项，用业务动作语言帮助用户快速筛出下一步待处理事项。

## 2. 目标

新增「后续建议工作」筛选项，支持用户按下一步建议动作筛选面料需求列表。

目标包括：

- 让仓储主管快速筛出需要调拨的面料。
- 让采购跟单快速筛出需要采购或入库跟进的面料。
- 保留现有 6 类异常规则，不新增异常类型。
- 不新增宽表列，避免数据列表继续变宽。
- 筛选后统计区、数据列表、分页结果保持一致。

## 3. 非目标

本次不做以下内容：

- 不创建调拨单。
- 不创建采购单。
- 不新增异常类型。
- 不修改现有异常触发和解除规则。
- 不新增宽表列。
- 不新增真实工作流状态。
- 不做权限控制。

## 4. 页面位置

搜索区保持一行布局，新增筛选项放在「异常类型」之后、「仓库 / 目的仓」之前。

调整后的搜索区顺序：

1. 关键词
2. 面料类型
3. 是否需印花
4. 是否需染色
5. 异常类型
6. 后续建议工作
7. 仓库 / 目的仓
8. 重置
9. 筛选

宽屏下尽量同一行展示；小屏或空间不足时允许自然换行。

## 5. 筛选项文案

筛选项名称：

> 后续建议工作

下拉选项：

| 选项 | 业务含义 |
| --- | --- |
| 全部 | 不按后续建议工作过滤 |
| 中央仓面料仓调拨至印花待加工仓 | 筛出需要把印花原料从中央仓调拨到印花待加工仓的面料 |
| 中央仓面料仓调拨至染色待加工仓 | 筛出需要把染色原料从中央仓调拨到染色待加工仓的面料 |
| 中央仓面料仓调拨至中转仓 | 筛出需要把直裁面料从中央仓调拨到中转仓的面料 |
| 采购跟单跟进 | 筛出总量不足，需要采购、转运或入库跟进的面料 |

## 6. 业务映射

「后续建议工作」不独立计算新规则，只映射现有异常。

| 后续建议工作 | 对应异常 |
| --- | --- |
| 中央仓面料仓调拨至印花待加工仓 | 印花待调拨 |
| 中央仓面料仓调拨至染色待加工仓 | 染色待调拨 |
| 中央仓面料仓调拨至中转仓 | 直裁待调拨 |
| 采购跟单跟进 | 缺直裁面料、缺印花原料、缺染色原料 |

如果一条面料同时存在多个异常，只要命中其中一个对应异常，就应该出现在该后续建议工作筛选结果中。

## 7. 联动规则

### 7.1 与异常类型筛选的关系

「异常类型」和「后续建议工作」可以同时生效。

当两个筛选项都不为「全部」时，列表结果需要同时满足两个条件。

示例：

- 异常类型选择「印花待调拨」
- 后续建议工作选择「中央仓面料仓调拨至印花待加工仓」
- 结果展示印花待调拨面料

如果两个筛选条件互相矛盾，列表展示空状态。

示例：

- 异常类型选择「印花待调拨」
- 后续建议工作选择「采购跟单跟进」
- 结果为空

### 7.2 与仓库 / 目的仓筛选的关系

「后续建议工作」不替代仓库筛选。

用户可以同时选择：

- 后续建议工作：中央仓面料仓调拨至印花待加工仓
- 仓库 / 目的仓：印花厂待加工仓

结果展示同时存在印花待调拨异常、且关联印花待加工仓库存记录的面料。

### 7.3 与统计区的关系

统计区只统计筛选后的结果。

当选择「采购跟单跟进」时：

- 总数展示缺料类面料数量。
- 采购中 Yard 展示这些面料对应的采购中数量。
- 库存数量展示这些面料对应的当前多仓库存。

### 7.4 与分页的关系

后续建议工作变化后，数据列表分页回到第一页。

分页总条数、当前展示范围、上一页 / 下一页状态，都按筛选后的结果重新计算。

## 8. 空状态

如果筛选后没有数据，数据列表展示空状态：

> 当前筛选下暂无面料需求

统计区同步显示筛选后的结果，不保留上一次统计。

## 9. 守门检查

需要补充或更新面料需求看板检查脚本，覆盖以下内容：

- 页面包含「后续建议工作」筛选文案。
- 页面包含 4 个后续建议工作选项。
- 采购跟单跟进能覆盖 3 类缺料异常。
- 印花调拨、染色调拨、直裁调拨分别映射到对应待调拨异常。
- 筛选口径不改变现有 6 类异常集合。
- 页面仍保留分页文案。

## 10. 原型审查记录

由于本次改动涉及搜索筛选和业务口径，需要更新面料需求看板原型审查记录。

审查记录中应说明：

- 当前新增筛选项解决的是「用户想按下一步动作查异常」的问题。
- 它不新增业务状态，只是现有异常到后续动作的映射。
- 它不创建真实调拨或采购动作。

## 11. 验收标准

- 搜索区出现「后续建议工作」。
- 下拉选项包含「全部」和 4 类后续建议工作。
- 选择「中央仓面料仓调拨至印花待加工仓」后，只展示印花待调拨相关面料。
- 选择「中央仓面料仓调拨至染色待加工仓」后，只展示染色待调拨相关面料。
- 选择「中央仓面料仓调拨至中转仓」后，只展示直裁待调拨相关面料。
- 选择「采购跟单跟进」后，展示缺直裁面料、缺印花原料、缺染色原料相关面料。
- 与异常类型、仓库 / 目的仓、关键词等筛选条件组合时，结果取交集。
- 筛选后统计区同步变化。
- 筛选后分页回到第一页。
- 宽表不新增列。
- 现有异常规则不改变。

## 12. 实现边界

本次实现应保持最小范围：

- 更新面料需求看板数据筛选口径。
- 更新搜索区展示。
- 更新面料需求看板检查脚本。
- 更新对应原型审查记录。

不应改动：

- 全局菜单结构。
- 其他 WLS 页面。
- 调拨单、采购单、库存流水等未涉及页面。
- 真实后端或接口设计。
