# 生产准备时效调整设计

## 背景

当前 `生产准备时效` 页面已经接入 `FCS -> 生产单管理 -> 生产准备时效`，并已具备准备台账、月度统计、花型师筛选、花型师分配和完成资料上传的原型能力。

本次调整不是新增独立模块，而是在现有页面上补齐更准确的业务口径：

- 进入列表的数据来自预售订单达到做大货要求，当前阈值为大于等于 300。
- 进入列表后先生成生产准备记录，再由系统按商品工艺和品类自动推导商品准备类型。
- 跟单只做确认和修正，不从零手工选择商品类型。
- 商品类型决定默认准备项模板，必做项自动带出，选填项由跟单确认。
- 准备阶段的最终产出必须在所有应完成准备项完成后统一展示。

当前代码已核查：

- 页面文件：`src/pages/production/preparation-timing.ts`
- 数据文件：`src/data/fcs/production-preparation-timing.ts`
- 自检脚本：`scripts/check-production-preparation-timing.ts`
- 菜单配置：`src/data/app-shell-config.ts`
- 路由配置：`src/router/routes-fcs.ts`

## 目标

本次调整目标：

- 让列表入口口径对齐老系统 `首单打版进度列表`。
- 在准备记录上展示选品、买手、跟单、达到做大货要求时间、进入准备时间、达到做大货要求天数。
- 用系统推导和跟单确认共同决定商品准备类型。
- 覆盖四类商品准备类型：
  - `非烫画&非毛织（纯梭织）`
  - `烫画&直喷`
  - `毛织`
  - `毛织&梭织`
- 按商品类型生成默认准备项，明确必做、选填、串行、并行关系。
- 补齐 mock 数据，覆盖四类类型、修正样例、选填样例和最终产出样例。
- 保持月度统计口径：`生产准备记录 + 准备项 = 1`。

## 非目标

本次不做：

- 不新增后端接口、数据库或真实规则引擎。
- 不接真实预售订单数据，不自动计算真实销量。
- 不引入权限系统。
- 不做真实上传服务。
- 不重构 PCS、FCS 生产单或技术包体系。
- 不把商品工艺和品类配置做成独立后台维护页面。

## 入口口径

### 进入条件

一条商品进入 `生产准备时效` 的条件：

- 预售订单达到做大货要求。
- 当前原型阈值固定为 `300`。
- 达到阈值后生成一条 `生产准备记录`。

在原型中不做真实销量计算，只通过 mock 字段表达：

- `largeGoodsThresholdQty`: 做大货要求数量，默认 `300`
- `largeGoodsReachedQty`: 达到时数量
- `largeGoodsReachedAt`: 达到时间
- `largeGoodsReachedDays`: 达到做大货要求天数
- `enteredAt`: 进入准备时间

### 老系统字段映射

老系统 `首单打版进度列表` 字段在新页面中的映射：

| 老系统字段 | 新页面字段 | 说明 |
| --- | --- | --- |
| 商品图片 | 商品图 | 继续展示 |
| SPU | SPU | 继续展示 |
| 选品 | 选品 | 新增 `selectionName` |
| 跟版人员 | 跟单 | 使用 `merchandiserName` |
| 达到首单做货时间 | 达到做大货要求时间 / 进入准备时间 | 两者都展示 |
| 达到 200 单用时天数 | 达到做大货要求天数 | 文案改为 300 口径 |
| 目前进度 | 当前卡点 / 准备状态 | 使用现有状态和卡点字段 |
| 时间 | 准备项时间线 | 下钻到详情展示 |
| 备注 | 备注 / 卡点说明 | 使用 `currentBlockerText` 和 item remark |

## 商品准备类型

新增类型：

```ts
type ProductPrepType =
  | '非烫画&非毛织（纯梭织）'
  | '烫画&直喷'
  | '毛织'
  | '毛织&梭织'
```

## 自动推导规则

系统从商品工艺和品类字段推导商品准备类型。

推导顺序：

1. 如果工艺或品类同时命中毛织和梭织拼接特征，推导为 `毛织&梭织`。
2. 如果工艺或品类命中毛织特征，推导为 `毛织`。
3. 如果工艺命中烫画、直喷、DTF、DTG、数码印，且未命中毛织，推导为 `烫画&直喷`。
4. 其他默认推导为 `非烫画&非毛织（纯梭织）`。

命中特征以 mock 字段表达：

```ts
craftTags: string[]
categoryTags: string[]
derivedProductPrepType: ProductPrepType
```

示例：

- `craftTags: ['数码印', 'DTF']` -> `烫画&直喷`
- `categoryTags: ['毛织']` -> `毛织`
- `categoryTags: ['毛织', '梭织拼接']` -> `毛织&梭织`
- `categoryTags: ['梭织']` -> `非烫画&非毛织（纯梭织）`

## 跟单确认和修正

系统推导后，跟单进入详情确认。

记录字段：

```ts
derivedProductPrepType: ProductPrepType
confirmedProductPrepType: ProductPrepType
prepTypeSource: '系统推导' | '人工修正'
prepTypeConfirmedBy: string
prepTypeConfirmedAt: string
prepTypeOverrideReason: string
```

规则：

- 如果跟单确认系统推导结果，`prepTypeSource = '系统推导'`。
- 如果跟单修改商品准备类型，`prepTypeSource = '人工修正'`。
- 人工修正必须填写 `prepTypeOverrideReason`。
- 页面同时展示系统推导类型和跟单确认类型。
- 后续准备项模板按 `confirmedProductPrepType` 生效。

## 准备项模板

### 非烫画&非毛织（纯梭织）

必做项：

- 梭织基码纸样
- 版衣制作
- 梭织齐码纸样
- 辅料下单

选填项：

- 数码印/DTF/DTG 花型
- 染色调色（面料）

关系：

- `梭织基码纸样 -> 版衣制作 -> 梭织齐码纸样` 串行。
- `辅料下单` 与主线并行。
- 花型和染色调色为选填，由跟单确认后并行执行。

### 烫画&直喷

必做项：

- 数码印/DTF/DTG 花型

选填项：

- 无。

关系：

- 有且仅有花型必做项。
- 不默认生成基码、版衣、齐码、辅料和染色项。

### 毛织

必做项：

- 毛织基码纸样
- 版衣制作
- 毛织齐码纸样
- 辅料下单

选填项：

- 染色调色（面料）

关系：

- `毛织基码纸样 -> 版衣制作 -> 毛织齐码纸样` 串行。
- `辅料下单` 与主线并行。
- 染色调色（面料）由跟单确认后并行执行。

### 毛织&梭织

必做项：

- 毛织基码纸样
- 梭织基码纸样
- 版衣制作
- 毛织齐码纸样
- 梭织齐码纸样
- 辅料下单

选填项：

- 数码印/DTF/DTG 花型
- 染色调色（纱线）
- 染色调色（面料）

关系：

- `毛织基码纸样` 和 `梭织基码纸样` 并行。
- 两个基码纸样都完成后，才能进入 `版衣制作`。
- `版衣制作` 完成后，`毛织齐码纸样` 和 `梭织齐码纸样` 并行。
- `辅料下单` 与主线并行。
- 花型、纱线染色、面料染色为选填，由跟单确认后并行执行。
- 染色调色（纱线）和染色调色（面料）属于同一团队，但页面需要区分对象。

## 准备项字段调整

保留现有准备项字段，新增：

```ts
requiredKind: '必做' | '选填'
selectedByMerchandiser: boolean
selectedAt: string
sequenceGroup: string
dependsOnItemIds: string[]
parallelGroup: string
```

规则：

- 必做项默认 `selectedByMerchandiser = true`。
- 选填项默认 `selectedByMerchandiser = false`，状态为 `待判断`。
- 跟单勾选选填项后，进入待分配或待开始。
- 未选择的选填项不计入完成率，不计入月度完成统计。
- 已选择的选填项和必做项共同决定准备记录是否完成。

## 页面调整

### 准备台账

列表新增或调整列：

- 商品：图片、SPU、商品名
- 商品类型：跟单确认类型，并提示系统推导类型
- 选品 / 买手 / 跟单
- 达到做大货要求：数量、阈值、达到天数
- 进入准备时间
- 准备项确认状态
- 整体状态
- 当前卡点
- 产出状态
- 操作

### 详情抽屉

详情抽屉分为以下区块：

1. 来源信息
   - 商品图、SPU、商品名
   - 选品、买手、跟单
   - 做大货阈值、达到数量、达到时间、达到天数
   - 进入准备时间

2. 商品类型确认
   - 工艺标签
   - 品类标签
   - 系统推导类型
   - 跟单确认类型
   - 确认人、确认时间
   - 修正原因

3. 准备项确认
   - 按必做和选填分组展示。
   - 必做项默认选中，页面不提供取消入口。
   - 选填项提供勾选展示。
   - 已确认后展示确认时间和确认人。

4. 准备项明细
   - 展示每个准备项的负责人、状态、计划时间、实际完成时间和证据。
   - 花型项继续展示花型团队、花型师、完成图片、花型文件、买手确认状态。

5. 准备产出
   - 未全部完成时展示预计产出。
   - 全部完成后展示正式产出。

## 产出展示规则

准备产出包括：

- 正式技术包
- 生产需求单转生产单
- 印花单
- 染色单
- 辅料采购单

展示规则：

- 如果必做项未全部完成，不展示正式产出，只展示预计产出和缺失项。
- 如果已勾选选填项未完成，不展示正式产出。
- 如果所有必做项和已勾选选填项完成，统一展示正式产出。
- 未勾选的选填项显示为 `未选择`，不阻塞产出。

建议字段：

```ts
outputReady: boolean
outputPublishedAt: string
outputs: Array<{
  outputType: '正式技术包' | '生产单' | '印花单' | '染色单' | '辅料采购单'
  outputNo: string
  outputHref: string
  outputStatus: '预计生成' | '已生成'
}>
```

## 月度统计

继续沿用口径：

`生产准备记录 + 准备项 = 1`

完成数量计入条件：

- 准备项 `selectedByMerchandiser = true`
- 准备项 `status = 已完成`
- 准备项 `actualFinishAt` 落入统计月份
- 所属准备记录不是 `已关闭`

不计入条件：

- 未选择的选填项
- `无需`
- `待判断`
- `待分配`
- `待开始`
- `进行中`
- `待确认`
- 未完成的 `已超时`
- 已关闭记录

统计卡片保持：

- 本月完成准备项
- 完成基码
- 完成齐码
- 完成花型
- 完成染色
- 按时完成
- 超时完成
- 平均耗时

统计明细细化：

- 完成基码包含 `梭织基码纸样` 和 `毛织基码纸样`。
- 完成齐码包含 `梭织齐码纸样` 和 `毛织齐码纸样`。
- 完成花型包含 `数码印/DTF/DTG 花型`。
- 完成染色包含 `染色调色（纱线）` 和 `染色调色（面料）`。

## Mock 数据要求

保持至少 12 条生产准备记录，补齐场景覆盖：

| 场景 | 最低数量 | 说明 |

必须包含：

- 1 条选填染色被勾选的记录。
- 1 条毛织&梭织同时勾选纱线染色和面料染色的记录。
- 1 条全部应完成项完成后展示正式产出的记录。
