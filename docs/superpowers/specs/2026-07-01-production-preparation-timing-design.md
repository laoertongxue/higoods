# 生产准备时效产品设计

## 背景

生产准备阶段时效用于跟进商品达到首单做货标准后，到正式进入生产前的准备工作是否按时完成。业务已确认月度统计口径为：`生产准备记录 + 准备项 = 1`。

本设计基于当前 `higoods` 原型仓库核查结果：

- FCS 菜单的 `生产单管理` 分组已存在于 `src/data/app-shell-config.ts`，当前包含生产单任务生成规则、生产需求单、生产单管理、生产单计划、交付仓配置、生产单变更、状态管理和工序工艺字典。
- FCS 生产单路由已集中在 `src/router/routes-fcs.ts`，生产单相关路径统一位于 `/fcs/production/*`。
- 生产单模型位于 `src/data/fcs/production-orders.ts`，已有 `productionOrderNo`、`demandSnapshot`、`techPackSnapshot`、`planEndDate`、`createdAt`、`updatedAt` 等字段，可作为准备记录的关联上下文。
- PCS 花型任务模型位于 `src/data/pcs-pattern-task-types.ts`，已有 `assignedMemberId`、`assignedMemberName`、`completionImageIds`、`patternFileIds`、`buyerReviewStatus`、`updatedAt` 等字段。
- PCS 花型任务字段策略位于 `src/data/pcs-engineering-task-field-policy.ts`，创建时已要求团队和花型师，完成时已要求完成确认图片、花型文件和买手确认通过。
- 当前项目已有 CSV 导出实现模式，例如 `src/pages/deduction-analysis.ts` 使用 `data:text/csv;charset=utf-8` 生成导出链接。
- 当前项目已有印花、染色统计派生层 `src/data/fcs/process-statistics-domain.ts`，其口径是只读统计投影，不作为状态源头。生产准备统计应采用同样原则。

## 目标

新增一个 FCS 页面，用于把生产准备阶段的基码、版衣、齐码、花型、染色、辅料、毛织等准备事项聚合成跟单可查看、可筛选、可导出的台账。

页面目标：

- 跟单能看到每个准备对象当前卡在哪个准备项。
- 业务能按月导出基码、齐码、花型、染色等准备项完成数量。
- 花型任务能明确分配花型师，花型师能筛选自己的任务并上传完成图片。
- 页面是原型演示，不做真实后端、真实权限、真实文件存储、真实审批流。

## 非目标

本次不做以下内容：

- 不新增后端接口、数据库或 API 层。
- 不重构 PCS 花型任务、FCS 生产单或技术包体系。
- 不把页面迁移到 React。
- 不做真实上传服务，上传文件只沿用当前原型的 mock 文件 URL 或 object URL 表达方式。
- 不做权限系统。花型师“我的任务”通过筛选演示，不做登录态权限隔离。
- 不把生产染色加工单完成时间当作准备阶段染色调色完成时间。
- 不用统计结果反写业务状态。

## 菜单与路由

菜单位置：

`FCS 工厂生产协同系统 -> 平台运营系统 -> 生产单管理 -> 生产准备时效`

建议插入位置：

- 位于 `生产单计划` 之后。
- 位于 `交付仓配置` 之前。

原因：

- 生产准备时效是生产单前置准备的聚合台账，和生产单、生产计划、正式技术包、生产需求单强相关。
- 它不属于 PCS 单点工程任务，也不属于 PFOS 印花厂、染厂或毛织厂统计。
- 它发生在任务执行前后衔接处，但主视角是跟单和生产协同，不是工厂执行。

## 业务对象

### 生产准备记录

`ProductionPreparationRecord` 表示一个商品或生产对象进入生产准备阶段后的跟进记录。

一条记录可以关联：

- SPU
- 商品名称
- 买手
- 跟单
- 销量达标时间
- 进入准备时间
- 生产需求单
- 生产单
- 正式技术包版本
- PCS 工程任务
- 准备项明细

准备记录的产生口径：

- 默认来源为商品销量达到首单做货标准后进入首单进度下单列表。
- 老系统语义中，达到 300 件后会进入首单打板进度列表；新系统原型中以 mock 字段表达，不做自动计算。
- 特殊情况，如前置打板、新类目或手动跟进，可以通过 mock 数据标记 `人工加入`。

### 准备项

`ProductionPreparationItem` 表示一条准备记录下的一个准备事项。

统计口径：

- `一条生产准备记录 + 一个准备项 = 1`
- 一个准备项无论上传几张图片、几个文件，都只计 1。
- 一个 SPU 下基码完成计 1，齐码完成再计 1。
- 关闭或标记无需的准备项不计入完成数量。

## 准备项类型与完成口径

### 基码纸样

是否必做：必做。

前置关系：准备记录进入后即可开始。

责任团队：版师团队。

完成口径：基码纸样文件上传完成。

完成时间：基码纸样文件的上传时间。

证据：纸样文件、上传人、上传时间。

超时规则：计划 2 天完成。超过计划完成时间未上传时显示 `已超时`。

### 版衣制作

是否必做：必做。

前置关系：基码纸样完成后才能开始。

责任团队：车板团队。

完成口径：上传模特试版照片。

完成时间：试版照片上传时间。允许重新上传，页面显示最新上传时间，并保留“最新证据”。

证据：试版照片、上传人、上传时间。

超时规则：计划 1 天完成。

### 齐码纸样

是否必做：必做。

前置关系：版衣制作完成后继续推进。

责任团队：版师团队。

完成口径：正式技术包发布时间。

完成时间：正式技术包发布时间。

证据：正式技术包版本、发布时间、发布人。

说明：不以技术包上传纸样时间作为完成时间，避免审核打回导致口径不一致。

### 花型

是否必做：条件做。数码印、烫画、直喷等有花型产出的商品需要。

前置关系：可与基码、染色、辅料并行；部分场景需要等待面料确认。

责任团队：花型团队。

责任人：花型师。

完成口径：花型任务上传完成确认图片和花型文件，并达到买手确认通过。

完成时间：买手确认通过时间；若原型无确认时间，则取花型任务最近更新时间。

证据：完成确认图片、花型文件、买手确认状态、花型师。

代码关联：复用 PCS 花型任务中的 `assignedMemberId`、`assignedMemberName`、`completionImageIds`、`patternFileIds`、`buyerReviewStatus`。

### 染色调色

是否必做：条件做。需要染色调色或色卡确认的商品需要。

前置关系：可与基码、花型、辅料并行；毛织组合场景需在基码整体确认后进入后续。

责任团队：染色团队。

完成口径：上传调色好的样品与潘通色卡照片。

完成时间：调色照片上传时间。允许修改，页面显示最新时间。

证据：调色样照片、潘通色卡照片、上传人、上传时间。

说明：这里是准备阶段的染色调色，不等同于生产染色加工单完成。

### 辅料下单

是否必做：多数需要，允许标记 `无需`。

前置关系：可与基码、花型、染色并行。

责任团队：跟单团队发起，武汉团队或采购团队执行。

完成口径：系统抓取到辅料采购单数据并同步到准备面板。

完成时间：采购单同步时间。

证据：辅料采购单号、供应商、同步时间。

### 毛织纸样

是否必做：条件做。涉及毛织部位或整件毛织时需要。

前置关系：毛织与梭织基码可并行；两者作为整体影响后续纱线染色调色、面料染色调色、数码印等事项。

责任团队：毛织团队。

完成口径：毛织纸样或毛织样确认资料上传。

完成时间：毛织资料上传或确认时间。

证据：毛织纸样文件、毛织样照片、确认记录。

## 状态设计

### 准备记录状态

- `未开始`：记录已进入台账，但所有需要的准备项尚未开始。
- `进行中`：至少一个需要的准备项已开始，且未全部完成。
- `部分超时`：存在需要完成但已超过计划完成时间的准备项。
- `已完成`：所有需要的准备项已完成，并已发布正式技术包。
- `已关闭`：买手决定不下单或业务关闭，不再纳入完成统计。

页面不展示英文状态码。

### 准备项状态

- `无需`：该商品不需要此准备项。
- `待判断`：跟单尚未判断是否需要。
- `待分配`：需要此准备项，但责任团队或责任人未分配。
- `待开始`：已分配但未开始。
- `进行中`：已有动作或证据，但未达完成口径。
- `待确认`：已提交证据，等待买手或跟单确认。
- `已完成`：达到完成口径。
- `已超时`：超过计划完成时间且未完成。

## 页面结构

页面只有一个菜单入口，内部两个页签：

- `准备台账`
- `月度统计`

不单独新增 `报表中心` 菜单。

### 页签一：准备台账

用户：生产跟单、生产协同负责人、业务管理人员。

目标：快速知道每个准备对象卡在哪里，下一步该找谁。

页面布局：

1. 顶部标题区
   - 标题：`生产准备时效`
   - 副标题：`按生产准备记录跟进基码、版衣、齐码、花型、染色、辅料等准备项完成情况。`
   - 右侧动作：`导出当前台账`

2. KPI 区
   - 准备记录总数
   - 进行中
   - 部分超时
   - 今日应完成准备项
   - 本月已完成准备项
   - 待分配花型任务

3. 筛选区
   - 月份：`2026-03`、`2026-04`
   - 跟单：全部、陈静、林晓、周敏
   - 买手：全部、宋雨、何佳、廖敏、许晴
   - 记录状态：全部、未开始、进行中、部分超时、已完成、已关闭
   - 准备项类型：全部、基码纸样、版衣制作、齐码纸样、花型、染色调色、辅料下单、毛织纸样
   - 责任团队：全部、版师团队、车板团队、花型团队、染色团队、采购团队、毛织团队
   - 花型师：全部、冰冰、林小美、Diah、Sari
   - 是否超时：全部、只看超时
   - 关键词：SPU、商品名、生产单号、技术包版本、准备项

4. 列表区
   - 商品：图片、SPU、商品名
   - 买手/跟单
   - 进入准备时间
   - 关联生产单
   - 正式技术包
   - 整体状态
   - 完成进度：例如 `5/7`
   - 当前卡点：例如 `齐码纸样待正式技术包发布`
   - 最早超时项
   - 预计完成时间
   - 操作：查看详情、更新准备项、查看生产单

5. 详情抽屉
   - 基础信息
   - 准备项时间线
   - 准备项明细卡片
   - 关联对象
   - 操作记录

### 页签二：月度统计

用户：业务人员、生产管理人员。

目标：按月导出基码、齐码、花型、染色等完成数量。

页面布局：

1. 顶部说明
   - 显示统计口径：`生产准备记录 + 准备项 = 1`
   - 说明无需项、关闭记录不计入完成数量。

2. 筛选区
   - 统计月份
   - 准备项类型
   - 责任团队
   - 跟单
   - 是否超时

3. 汇总卡片
   - 本月完成准备项
   - 按时完成
   - 超时完成
   - 平均耗时
   - 花型完成
   - 染色完成

4. 统计表
   - 统计月份
   - 准备项
   - 完成数量
   - 按时完成数量
   - 超时完成数量
   - 平均耗时小时
   - 责任团队
   - 最近完成时间
   - 口径说明

5. 明细表
   - 统计月份
   - 准备记录编号
   - SPU
   - 商品名
   - 准备项
   - 责任团队
   - 责任人
   - 计划完成时间
   - 实际完成时间
   - 是否超时
   - 证据摘要

6. 导出
   - `导出月度统计`：导出汇总表。
   - `导出完成明细`：导出明细表。
   - 使用 CSV 即可，不引入 xlsx 依赖。

## 花型师分配与上传

### 分配

花型准备项需要展示：

- 花型团队
- 花型师
- 分配时间
- 花型任务号
- 买手确认状态

分配动作：

- 在准备项卡片中点击 `分配花型师`。
- 弹出轻量弹窗或抽屉，只选择团队和花型师。
- 原型内更新 mock 状态为 `进行中` 或 `待开始`。

不做权限和组织架构扩展。花型师候选来自 mock 配置：

- 中国花型组：冰冰、林小美
- Bandung 花型组：Diah
- Jakarta 花型组：Sari

### 花型师我的任务

在准备台账筛选区提供 `花型师` 筛选。

快捷筛选：

- `我的花型任务`
- `待上传完成图`
- `待买手确认`

原型里“我的任务”可默认映射到 `林小美`，和 PCS 花型任务页面当前原型习惯保持一致。

### 上传完成图片

花型准备项详情中显示：

- `完成确认图片`
- `花型文件`
- `买手确认状态`

上传动作沿用原型轻交互：

- 上传图片后追加到 `completionImageIds`。
- 上传文件后追加到 `patternFileIds`。
- 如果二者都有，状态从 `进行中` 进入 `待确认`。
- 买手确认通过后状态为 `已完成`。

## 统计与导出

### 完成判断

一个准备项计入完成数量必须满足：

- `required = true`
- `status = 已完成`
- `actualFinishAt` 落入筛选月份
- 所属准备记录不是 `已关闭`

不计入完成数量：

- `无需`
- `待判断`
- `待分配`
- `待开始`
- `进行中`
- `待确认`
- `已超时` 且未完成
- 所属准备记录为 `已关闭`

### 汇总指标

月度统计输出：

- 完成数量：符合完成判断的准备项数量。
- 按时完成数量：`actualFinishAt <= plannedFinishAt`。
- 超时完成数量：`actualFinishAt > plannedFinishAt`。
- 平均耗时小时：`actualFinishAt - plannedStartAt` 的平均值。
- 最近完成时间：同准备项类型下最大的 `actualFinishAt`。

### CSV 字段

汇总 CSV 字段：

- 统计月份
- 准备项
- 完成数量
- 按时完成数量
- 超时完成数量
- 平均耗时小时
- 责任团队
- 最近完成时间
- 统计口径说明

明细 CSV 字段：

- 统计月份
- 准备记录编号
- SPU
- 商品名
- 生产单号
- 买手
- 跟单
- 准备项
- 责任团队
- 责任人
- 计划开始时间
- 计划完成时间
- 实际完成时间
- 是否超时
- 证据摘要

导出文件名：

- `生产准备时效月度统计-YYYYMM.csv`
- `生产准备时效完成明细-YYYYMM.csv`

## 交互要求

- 页签切换只更新页面主体区域，不做整页跳转。
- 筛选输入使用本地状态筛选，避免输入时整页明显卡顿。
- 详情抽屉打开时只渲染抽屉，不改变列表滚动位置。
- 花型师分配弹窗只更新当前准备项。
- 上传完成图片和花型文件后，当前准备项卡片局部刷新即可。
- 图标 hydration 仅作用于新插入区域。
