# 三方车缝厂综合评定产品原型设计

## 1. 设计目标

在「工厂池管理」下新增独立二级菜单「三方车缝厂综合评定」，集中展示并维护全部三方车缝厂的能力、产能、时效、品控和综合评级。

本期目标是快速交付可演示的产品原型，让管理人员能够：

- 在一张列表中横向比较全部三方车缝厂的 5 个维度。
- 识别每个子项来自系统数据还是人工填写。
- 筛选每个维度是否已经形成完整结果。
- 维护品类能力、产能和综合评级。
- 查看按现有业务口径计算的时效与品控展示数据。

## 2. 产品边界

### 2.1 与「三方工厂评级」的关系

- 「三方工厂评级」继续表达工厂刚进入业务体系时的准入初评、试产考核及派单、结算风控。
- 「三方车缝厂综合评定」表达工厂进入业务体系后的持续运营综合评价。
- 两个页面独立存在、独立维护，任何一方都不读取、覆盖或修改另一方的评级结果。
- 综合评定中的评级沿用 `S / A / B / C` 四级表达，但属于独立的人工评定结果。

### 2.2 本期包含

- 新增菜单、路由和管理端列表页。
- 覆盖工厂档案中的全部三方车缝厂。
- 五维分组宽表、筛选、联动统计、排序、分页、列设置和固定操作列。
- 「编辑评定」抽屉。
- 数据来源颜色与文字标识。
- 五维完成状态筛选。
- 独立的综合评定展示 Mock 数据。
- 时效、品控数据按现有业务规则计算后展示。

### 2.3 本期不包含

- 不建设真实接口、数据库或数据同步任务。
- 不从其他页面实时读取、跳转或回写数据。
- 不影响派单、竞价、生产、交出、质检或结算。
- 不自动计算综合评级。
- 不允许人工覆盖系统来源字段。
- 不保存历次评定版本和变化历史。
- 不改造现有「三方工厂评级」页面。

## 3. 使用角色和主要场景

### 3.1 主要角色

- 工厂运营人员：维护工厂品类能力、产能和综合评级。
- 业务人员：查看三方车缝厂的综合能力并横向比较。
- 管理人员：筛选未完善工厂，快速掌握工厂池整体情况。

页面属于管理端，可承载高密度表格，但必须保持来源可追溯、字段可比较、操作入口明确。

### 3.2 典型场景

1. 业务人员需要为新纳入的三方车缝厂补齐品类能力、机器台数、工人人数和月产值。
2. 管理人员筛选「产能未完成」或「评级未完成」的工厂，推动业务人员补录。
3. 业务人员比较多家工厂的按时交付表现和品控表现，为线下决策提供参考。
4. 某项数据未来完成真实系统对接后，页面将其来源从蓝色人工数据切换为绿色系统数据，并停止人工编辑。

## 4. 页面信息架构

页面从上到下依次为：

1. 页面标题和数据来源图例。
2. 筛选区。
3. 联动统计区。
4. 五维分组标准列表。
5. 分页。
6. 编辑评定抽屉。

页面路径建议为：

```text
/fcs/factories/third-party-comprehensive-assessment
```

## 5. 筛选设计

### 5.1 筛选项

| 筛选项 | 可选值或说明 |
| --- | --- |
| 工厂名称／编码 | 模糊搜索工厂名称或编码 |
| 品类能力 | 按 11 项业务品类多选 |
| 综合评级 | 全部、S、A、B、C |
| 能力完成状态 | 全部、已完成、未完成 |
| 产能完成状态 | 全部、已完成、未完成 |
| 时效完成状态 | 全部、已完成、未完成 |
| 品控完成状态 | 全部、已完成、未完成 |
| 评级完成状态 | 全部、已完成、未完成 |

多个筛选组之间采用「并且」关系，同一品类能力筛选组内采用「任一匹配」关系。筛选条件变化后回到第 1 页。

### 5.2 联动统计

统计结果随当前筛选结果变化，采用 48 px 单行摘要卡：

- 全部工厂。
- 评定已完善。
- 待完善。
- 评级分布：S、A、B、C。

## 6. 五维字段和数据来源

### 6.1 字段矩阵

| 维度 | 子项 | 展示形式 | 当前来源 | 是否可编辑 |
| --- | --- | --- | --- | --- |
| 能力 | 工艺能力 | 多个工艺标签 | 工厂档案 | 否 |
| 能力 | 品类能力 | 业务品类多选标签 | 人工填写 | 是 |
| 产能 | 机器台数 | 正整数，单位「台」 | 人工填写 | 是 |
| 产能 | 工人人数 | 正整数，单位「人」 | 人工填写 | 是 |
| 产能 | 月产值 | 数值，单位「万印尼盾／月」 | 人工填写 | 是 |
| 时效 | 交付完成 | 按时完成率 | 业务展示数据 | 否 |
| 时效 | 30% 回货 | 按时达标率 | 业务展示数据 | 否 |
| 时效 | 70% 回货 | 按时达标率 | 业务展示数据 | 否 |
| 时效 | 100% 回货 | 按时达标率 | 业务展示数据 | 否 |
| 品控 | 不良品率 | 百分比 | 业务展示数据 | 否 |
| 品控 | 瑕疵率 | 百分比 | 业务展示数据 | 否 |
| 品控 | 返工率 | 百分比 | 业务展示数据 | 否 |
| 评级 | 综合评级 | S、A、B、C | 人工填写 | 是 |

### 6.2 品类能力字典

品类能力只使用一级分类，固定为以下 11 项：

1. 衬衫。
2. T 恤。
3. 马甲。
4. 背心。
5. 连衣裙。
6. 休闲连体裤。
7. 西装连体裤。
8. 休闲套装。
9. 西装套装。
10. 裤子。
11. 半裙。

### 6.3 来源展示规则

- 系统来源使用绿色圆点和绿色文字标签，并显示具体来源，例如「工厂档案」「时效业务数据」「质检业务数据」。
- 人工来源使用蓝色圆点和蓝色文字标签「人工填写」。
- 来源标识落实到每个子项，不使用整个维度共用一个来源标签。
- 未形成结果的字段显示灰色「待完善」或「暂无业务数据」，不得显示为蓝色。
- 五个维度的分组表头可使用克制的浅色背景帮助识别，但不得用整块绿色或蓝色背景冒充数据来源。
- 某字段未来完成真实系统对接后，系统值替换人工值，来源变为绿色，该字段同步变为只读。本期不实现该转换流程。

## 7. 时效计算口径

### 7.1 原型边界

综合评定页面不直接调用时效模块，也不与交出、收货页面串联。原型在自己的 Mock 数据中准备与现有车缝任务口径一致的分配、交出和确认实收事实，再生成各工厂的展示结果。

### 7.2 任务节点规则

沿用现有含车缝任务交付与回货时效规则：

| 任务类型 | 30% 回货节点 | 70% 回货节点 | 100% 回货节点 |
| --- | --- | --- | --- |
| 独立车缝任务 | 第 4 天 | 第 8 天 | 第 9 天 |
| 车缝到包装连续任务 | 第 5 天 | 第 9 天 | 第 10 天 |
| 裁片到包装连续任务 | 第 6 天 | 第 9 天 | 第 12 天 |

所有节点从业务分配时间起算，每 1 天按完整 24 小时计算。

### 7.3 单任务判定

- 交付完成：工厂累计交出数量在最终截止时间前达到实际分配数量。
- 30%／70%／100% 回货：接收方累计确认实收数量在对应截止时间前达到目标数量。
- 节点目标数量等于实际分配数量乘以节点比例，出现小数时向上取整。
- 截止时间后首次达到目标数量属于「逾期达标」，不得改写为「按时达标」。
- 截止时间已过但尚未达到目标数量属于「逾期未达标」。
- 尚未到截止时间且未达到目标数量的任务不计入该节点按时率的分母。

### 7.4 工厂汇总

```text
交付完成按时率 = 按时完成交付的任务数 ÷ 已到最终截止时间或已经完成交付的任务数

节点按时率 = 按时达到该节点的任务数 ÷ 已到该节点截止时间或已经达到该节点的任务数
```

没有可计入分母的任务时显示「暂无业务数据」，不得显示 `0%`。

## 8. 品控计算口径

### 8.1 原型边界

综合评定页面不直接调用质检模块，也不跳转到质检单。原型使用独立的工厂质检 Mock 事实，计算方式与现有三方车缝工厂评级中的质检口径一致。

### 8.2 数量口径

```text
工厂责任瑕疵数量 = 工厂责任瑕疵原因明细数量合计

不良数量 = 返工数量 + 工厂责任瑕疵数量
```

工厂责任瑕疵原因范围沿用当前质检责任字典，不在本页面另建原因分类。

### 8.3 工厂汇总比例

先汇总同一工厂的数量，再计算比例：

```text
不良品率 = (返工数量合计 + 工厂责任瑕疵数量合计) ÷ 质检数量合计

瑕疵率 = 工厂责任瑕疵数量合计 ÷ 质检数量合计

返工率 = 返工数量合计 ÷ 质检数量合计
```

不得直接平均多张质检单的百分比。质检数量合计为 0 时显示「暂无业务数据」，不得显示 `0%`。

## 9. 五维完成状态

| 维度 | 已完成条件 |
| --- | --- |
| 能力 | 工艺能力和品类能力均有值 |
| 产能 | 机器台数、工人人数、月产值均有合法值 |
| 时效 | 交付完成、30%、70%、100% 四项均形成可展示结果 |
| 品控 | 不良品率、瑕疵率、返工率三项均形成可展示结果 |
| 评级 | 已选择 S、A、B、C 中的一项 |

五维均完成时，工厂显示「评定已完善」；否则显示「待完善 N 项」。

## 10. 列表设计

### 10.1 分组表头

列表采用两级表头：

- 固定列：工厂信息。
- 能力：工艺能力、品类能力。
- 产能：机器台数、工人人数、月产值。
- 时效：交付完成、30% 回货、70% 回货、100% 回货。
- 品控：不良品率、瑕疵率、返工率。
- 评级：综合评级。
- 右侧固定列：操作。

工厂信息列显示工厂名称、编码、整体完善状态、最后更新人和最后更新时间。

### 10.2 宽表规则

- 默认展示全部 5 个维度和全部子项。
- 宽表只在表格容器内部横向滚动，页面主体不得横向溢出。
- 工厂信息列固定在左侧，操作列固定在右侧。
- 支持列排序、显示、隐藏、冻结和顺序调整。
- 工厂信息、综合评级、操作列为必需列，不允许隐藏。
- 列偏好和每页条数按路由持久化；当前页和排序不持久化。
- 列表必须分页，默认每页 10 条。

## 11. 编辑评定抽屉

### 11.1 可编辑内容

点击「编辑评定」打开右侧抽屉，只编辑：

- 品类能力。
- 机器台数。
- 工人人数。
- 月产值。
- 综合评级。

工艺能力、时效和品控以只读区块展示，并标明绿色系统来源。

### 11.2 校验

- 品类能力至少选择 1 项。
- 机器台数和工人人数只允许正整数。
- 月产值必须大于 0，允许小数，单位固定为「万印尼盾／月」。
- 综合评级只能选择 S、A、B、C。
- 保存成功后只刷新当前行、统计摘要和完成状态，不触发整页重绘。
- 关闭抽屉前若存在未保存修改，显示放弃修改确认。

### 11.3 更新记录

每家工厂只保存一份当前综合评定，记录最后更新人和最后更新时间，不保存历史版本。

## 12. 空态和异常展示

- 工厂档案没有工艺能力：显示「工厂档案待补充」，并在抽屉中说明该字段不能在当前页面修改。
- 时效没有可计算任务：显示「暂无业务数据」。
- 品控没有有效质检数量：显示「暂无业务数据」。
- 人工字段缺失：显示「待完善」。
- 筛选无结果：显示「暂无符合条件的三方车缝厂」，保留清空筛选入口。
- 数值输入不合法：在字段下方说明错误原因和正确单位，不使用笼统的「输入错误」。

## 13. 原型数据场景

Mock 数据至少覆盖：

1. 五维全部完成、评级为 S 的工厂。
2. 五维全部完成、评级为 A 的工厂。
3. 产能缺失的工厂。
4. 品类能力缺失的工厂。
5. 没有时效业务数据的工厂。
6. 没有有效质检数据的工厂。
7. 时效表现较差但品控较好的工厂。
8. 品控表现较差但时效较好的工厂。
9. 评级尚未填写的工厂。
10. 工厂档案缺少工艺能力的工厂。

所有工厂身份必须来自当前工厂档案中的三方车缝厂，不另造无法与工厂主档对应的展示身份。

## 14. 页面组件与实现边界

- 页面继续使用 Vanilla TypeScript 字符串模板渲染。
- 列表复用标准列表页、标准列表表格和分页组件。
- 抽屉、按钮、徽章、筛选等优先复用现有 `src/components/ui/` 组件。
- 新增独立的综合评定页面数据模块，保存人工字段和独立展示事实。
- 计算函数负责把展示事实聚合为时效和品控指标，页面只消费结果，不在模板中散落计算规则。
- 不新增后端、接口层、状态管理框架或真实跨模块事件链。

## 15. 验收标准

### 15.1 页面与导航

- 「工厂池管理」下出现「三方车缝厂综合评定」菜单。
- 新页面与「三方工厂评级」同时存在，互不覆盖。
- 列表覆盖全部三方车缝厂。

### 15.2 数据与展示

- 表头按能力、产能、时效、品控、评级正确分组。
- 全部子项默认在列表中可见。
- 系统来源显示绿色并写明具体来源，人工来源显示蓝色「人工填写」。
- 时效与品控展示值符合本设计中的计算公式。
- 无数据场景不显示虚假的 `0%`。
- 每个维度均可按已完成、未完成筛选。

### 15.3 编辑与交互

- 每家工厂均可打开编辑抽屉。
- 只有品类能力、产能和综合评级可以编辑。
- 工艺能力、时效、品控不可编辑。
- 保存后当前行、统计和完成状态同步更新。
- 抽屉开关、筛选、排序和分页不产生明显整页闪烁，目标响应时间不高于 200 ms。

### 15.4 列表治理与分辨率

- 页面声明标准列表页模式并通过列表页治理检查。
- 支持分页、排序、列显示、列冻结、列顺序和固定操作列。
- 1366 × 768 下完整可用，1280 × 720 下主要操作可见。
- 宽表横向滚动限制在表格容器内部。

### 15.5 原型治理

- 新增对应的原型审查记录。
- 通过 `npm run check:prototype-design-governance`。
- 通过最小业务检查和 `npm run build`。
- 在真实浏览器中验证菜单、页面、筛选、编辑抽屉、横向滚动和分页。

## 16. 最终设计结论

采用「五维分组标准列表＋编辑抽屉」方案。页面以快速展示和维护为核心：能力部分混合工厂档案与人工数据，产能和综合评级由人工维护，时效和品控按现有业务口径从独立 Mock 事实计算展示。页面不与现有生产、交出、质检、派单、结算及初评模块串联，避免为展示原型引入生产级复杂度。
