# 工序工艺字典完整顺序可视化设计

## 背景

工序工艺字典当前列表已经展示“基础路线顺序”，但用户只能逐页查看，无法一次性理解完整工序工艺的前后关系。技术包工艺路线会以该基础顺序作为默认参考，因此字典页需要一个只读入口，让生产计划、跟单、工艺配置人员能快速查看完整的基础工序顺序。

## 目标

- 在工序工艺字典页提供 `查看完整工序顺序` 入口。
- 用可视化方式展示完整基础路线顺序。
- 清楚表达同一步内多个工序工艺是“同基础顺序”，不表示严格串行。
- 保持只读，不支持调整、拖拽、保存。

## 非目标

- 不新增菜单或独立页面。
- 不修改工序工艺字典数据结构。
- 不修改技术包路线、生产单任务生成、连续工序判断逻辑。
- 不在本入口中提供排序编辑能力。

## 页面入口

在 `工序工艺字典` 标题区右侧增加按钮：

- 按钮文案：`查看完整工序顺序`
- 点击行为：打开大弹窗
- 关闭行为：关闭弹窗后保留当前列表筛选、分页状态

## 弹窗内容

弹窗标题：`完整工序工艺顺序`

顶部说明：

> 基础路线顺序仅作为技术包路线默认参考，最终以款式级技术包确认路线为准。

主体使用横向时间轴展示：

- 按基础路线顺序从小到大排列。
- 每个步骤是一列，例如 `第 4 步`、`第 5 步`。
- 同一步内多个工序工艺纵向堆叠，表示同基础顺序。
- 未配置基础顺序的工序工艺放在最后的 `未配置顺序` 区域。

每张工序工艺卡片展示：

- 工序 / 工艺名称
- 作用对象
- 阶段
- 是否出任务

## 数据口径

- 复用现有工序工艺字典数据。
- 基础路线顺序复用当前列表中的基础路线顺序口径。
- 弹窗展示全部工序工艺，不受当前搜索、阶段、粒度、状态筛选影响。
- 页面文案必须使用中文业务语义，不展示代码字段。

## 验收标准

- 工序工艺字典页可见 `查看完整工序顺序` 按钮。
- 点击按钮后打开只读大弹窗。
- 弹窗展示完整 37 条工序工艺。
- 工序工艺按基础路线顺序展示。
- 同一步多个工序工艺以并列/堆叠方式展示，不误导为串行。
- 未配置基础顺序的项集中展示在 `未配置顺序` 区域。
- 弹窗内不出现英文代码字段。
- 关闭弹窗后当前列表筛选和分页不丢失。
