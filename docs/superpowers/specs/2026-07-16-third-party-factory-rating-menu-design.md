# 三方工厂评级独立菜单产品设计

日期：2026-07-16

## 1. 背景

当前三方车缝工厂评级信息展示在工厂档案详情中，容易造成两个问题：

- 工厂档案承担了主数据、账号、产能、结算、评级风控等过多信息，评级不容易被 PPIC、跟单、结算人员快速找到。
- 评级实际影响车缝派单和结算拦截，不只是工厂档案的附属说明，应独立成为工厂管理下的风控对象。

本期将工厂档案中的“评级与派单风控”信息移出，新增菜单“三方工厂评级”，集中展示全部三方车缝工厂评级快照，并保持车缝分配、对账单等模块继续读取同一份评级口径。

## 2. 产品目标

1. 工厂管理菜单下新增“三方工厂评级”，作为三方车缝工厂风险查看入口。
2. 工厂档案不再展示完整评级详情，回归工厂主档、能力、PDA、结算资料等主数据职责。
3. 新页面覆盖全部三方车缝工厂评级快照，不能遗漏工厂档案主档中的三方车缝工厂。
4. 评级页面与车缝分配、结算拦截保持同一数据口径，避免页面显示可合作但派单或结算判断不一致。
5. 页面首屏按“先筛选、再看统计、再看列表”的顺序组织，统计卡片必须跟随筛选结果联动。

## 3. 设计范围

### 3.1 本期纳入

- 新增 FCS 工厂管理菜单项：三方工厂评级。
- 新增三方工厂评级列表页。
- 从工厂档案详情中移除完整“评级与派单风控”面板。
- 列表页展示全部三方车缝工厂评级快照。
- 详情抽屉展示单个工厂的评级原因、派单策略、结算策略、扣分和履约记录。
- 保持车缝分配工作台、对账单 / 结算模块继续使用同一评级数据。
- 更新对抗式核查，覆盖数据完整性、页面可见性、交互可达性、业务闭环。

### 3.2 本期不纳入

- 不建设真实评分引擎。
- 不新增真实后端、数据库、接口或权限体系。
- 不做评级编辑、人工扣分录入、审批流。
- 不改派单和结算已有拦截规则的业务口径。
- 不扩展到所有第三方工艺工厂，本期仍聚焦三方车缝工厂。
- 不把工厂档案改造成评级入口页或风险看板。

## 4. 菜单与路由

新增菜单放在 FCS 的“工厂管理”分组中，位置紧跟“工厂档案”之后。

建议路由：

`/fcs/factories/third-party-rating`

菜单展示：

```text
工厂管理
- 工厂入驻管理
- 工厂档案
- 三方工厂评级
- 工厂产能档案
- 能力标签
- 结算信息
- 工厂状态
- 工厂绩效
```

菜单命名使用“三方工厂评级”，不使用“评级与派单风控”。原因是菜单应表达业务对象和主入口，派单风控、结算拦截属于页面内展示和下游联动结果。

## 5. 页面定位

“三方工厂评级”是管理端列表页，面向中国管理人员、PPIC、跟单、财务和管理层。

页面回答四个问题：

1. 当前三方车缝工厂池里有哪些工厂？
2. 每个工厂当前评级和合作状态是什么？
3. 这个工厂派单时能不能选、适合什么单？
4. 这个工厂结算时能不能发起新对账或新结算？

页面不是一线员工执行端，不需要扫码、大按钮或低密度卡片。它应使用管理端标准列表模式，保留筛选、统计、分页、详情抽屉和固定操作列。

## 6. 首屏信息架构

首屏顺序必须固定为：

1. 标题：三方工厂评级。
2. 筛选区。
3. 联动统计卡片。
4. 标准列表。

### 6.1 筛选区

筛选项：

- 工厂名称 / 编码。
- 评级：全部、S、A、B、C。
- 合作状态：全部、正常合作、考核中、黑名单。
- 工厂规模：全部、大型工厂、小型工厂。
- 是否允许派单：全部、允许派单、限制派单、禁止派单。
- 是否允许结算：全部、允许结算、禁止新结算。

筛选区是统计和列表的共同输入。用户改变任何筛选条件后，统计卡片和列表必须同时基于同一筛选结果刷新。

### 6.2 联动统计卡片

统计卡片必须位于筛选区下方，且必须基于当前筛选结果计算，不能展示固定全量值。

统计项：

- 当前结果总数。
- 正常合作。
- 考核中。
- 黑名单。
- S 级。
- A 级。
- B 级。
- C 级。

示例：

当用户筛选“小型工厂 + 允许派单”时，统计卡片只统计这个结果集中的正常合作、考核中、黑名单和 S/A/B/C 分布。

如果筛选结果为空，统计卡片显示 0，列表显示空态。不能回退展示全量统计。

### 6.3 标准列表

列表必须使用现有标准列表页组件契约，并满足列表页治理要求：

- 页面声明标准列表页模式。
- 使用标准列表页外壳。
- 使用标准列表表格。
- 使用标准分页。
- 支持列排序、列显示、列冻结和右侧固定操作列。
- 即使当前只有 11 条 mock 数据，也必须展示分页口径。

建议默认每页 10 条，第二页展示剩余工厂，以证明分页真实存在。

## 7. 列表字段

列表列建议如下：

| 列 | 说明 | 是否必需 |
| --- | --- | --- |
| 工厂 | 展示工厂名称、编码 | 是 |
| 评级 | S / A / B / C，使用中文状态徽章 | 是 |
| 合作状态 | 正常合作、考核中、黑名单 | 是 |
| 车位与规模 | 例如 48 个车位 / 大型工厂 | 是 |
| 试单上限 | 1000 件、300 件或未设置 | 否 |
| 派单策略 | 优先派单、正常可选、黄牌提示、仅试产、禁止派单 | 是 |
| 结算策略 | 可按账本发起结算、禁止发起新结算等 | 是 |
| 近 90 天表现 | 派单数、平均延期、准时率、异常单数 | 否 |
| 操作 | 查看评级 | 是 |

操作列固定在右侧，始终可见。

## 8. 详情抽屉

点击“查看评级”打开详情抽屉。

抽屉内容：

1. 顶部评级摘要：工厂名称、编码、评级、合作状态。
2. 评级原因：展示最近评级原因。
3. 风控提示：
   - 黑名单：不能派单，不能发起新结算，历史账款需主管处理。
   - 考核中：只能试产，首单按规模限制。
   - B 级：黄牌提示，建议小单和非急单。
4. 车位与规模、试单上限。
5. 派单策略和结算策略。
6. 指标卡：当前总分、交期扣分、质量扣分、人工扣分。
7. 时效说明：近 90 天或首单考核摘要。
8. 履约记录表：生产单、单据类型、延期天数、发料 / 合格数量、结果。

抽屉中的内容可以沿用现有工厂档案评级面板的信息结构，但页面标题应从“评级与派单风控”调整为更适合独立页面的“评级详情”或“派单与结算风控”。

## 9. 从工厂档案移出规则

工厂档案详情中必须移除完整“评级与派单风控”大卡片，包括：

- 评级徽章。
- 派单策略。
- 结算策略。
- 扣分指标。
- 近 90 天说明。
- 履约记录表。

工厂档案仍保留自身主数据职责：

- 工厂基础信息。
- 工厂类型和组织层级。
- 接单能力。
- 生产流程开始条件。
- PDA 账号和角色。
- 结算资料主数据。

本期不建议在工厂档案里保留评级摘要卡，避免评级信息重新回流到工厂档案。若后续确需关联入口，只允许保留轻量跳转动作，例如“查看三方工厂评级”，但不展示评分和风控详情。

## 10. 数据口径

新页面继续读取现有三方工厂评级快照数据。

关键要求：

- 评级快照必须覆盖工厂档案主档中的全部三方车缝工厂。
- 页面列表、车缝分配候选、结算候选必须使用可互通的工厂 ID / 工厂编码。
- 黑名单工厂在评级页面显示禁止派单、禁止新结算，同时在车缝分配和结算生成中被拦截。
- 考核中工厂在评级页面显示只允许试产，同时在车缝分配中不能承接常规单。
- B 级工厂在评级页面显示黄牌提示，同时在车缝分配中保留确认提示而不是硬拦截。

评级页面是展示入口，不是新的评级真相源。真相源仍是共享 mock 数据。

## 11. 业务闭环

### 11.1 工厂档案到评级

工厂档案维护工厂是否为三方车缝工厂、工厂编码、车位能力等基础事实。评级页面必须覆盖这些三方车缝工厂，不能只展示部分样例。

### 11.2 评级到车缝分配

车缝分配选择工厂时，应继续读取评级快照：

- 黑名单不能派单。
- 考核中只能试产。
- B 级给黄牌确认。
- S / A 正常可选。

评级页面展示的策略文案必须与派单工作台实际提示一致。

### 11.3 评级到对账结算

对账单和结算候选应继续读取评级快照：

- 黑名单禁止发起新结算。
- 历史账本保留查看，不删除历史事实。
- 非黑名单工厂按原结算规则生成候选或对账单。

评级页面展示的结算策略必须与结算拦截一致。

## 12. 异常与空态

- 没有筛选结果：列表显示“暂无符合条件的三方车缝工厂”，统计全部为 0。
- 工厂有评级快照但无履约记录：详情抽屉显示“暂无履约记录”，评级仍可展示。
- 工厂档案中存在三方车缝工厂但缺少评级快照：对抗式检查必须失败，不能在页面静默隐藏。
- 评级快照存在但工厂主档不存在：对抗式检查必须失败，避免孤儿 mock 数据。
- 工厂 ID 和工厂编码无法互通：对抗式检查必须失败，避免页面可见但派单或结算无法串联。

## 13. 交互要求

- 筛选改变后，统计和列表基于同一结果集刷新。
- “重置”恢复全部筛选，并刷新统计和列表。
- “查看评级”打开抽屉，不应触发整页不可控闪烁。
- 列表分页、排序、列显示和冻结遵循标准列表页行为。
- 详情抽屉关闭后保留当前筛选和分页。

## 14. 原型审查要求

本次实现会涉及 FCS 管理端页面、菜单、路由、mock 数据展示和跨模块业务闭环，因此实施时必须更新原型审查记录。

审查重点：

- 角色匹配：页面是管理端，不按员工执行端设计。
- 信息架构：菜单位于工厂管理，符合业务对象边界。
- 页面模式：使用标准列表页，不自造列表模板。
- 文案：状态均为中文，不展示英文状态码。
- 数量与状态：车位、试单上限、派单数、延期天数均带单位。
- 协作关系：工厂档案、车缝分配、结算读取同一评级事实。
- 追溯：详情能看到扣分来源和履约记录。

## 15. 验收标准

1. 工厂管理菜单中出现“三方工厂评级”，位置紧跟“工厂档案”。
2. 访问 `/fcs/factories/third-party-rating` 可看到三方工厂评级页面。
3. 页面首屏顺序为标题、筛选区、联动统计卡片、标准列表。
4. 统计卡片随筛选结果联动；筛选为空时统计为 0。
5. 列表覆盖全部三方车缝工厂评级快照。
6. 列表可打开评级详情抽屉。
7. 工厂档案中不再出现完整“评级与派单风控”面板。
8. 黑名单、考核中、B 级工厂在评级页、车缝分配、结算模块中的风控口径一致。
9. 对抗式检查覆盖数据完整性、页面可见性、交互可达性、业务闭环四个方向。
10. 标准列表页治理、原型设计治理和构建检查通过。

## 16. 自检结论

- 没有保留未决事项。
- 菜单位置已明确为 FCS 工厂管理分组，紧跟工厂档案。
- 统计卡片位置和联动规则已明确为硬要求。
- 范围聚焦独立菜单和页面迁移，不引入真实后端、权限、评分引擎或编辑流程。
- 工厂档案移出边界、派单联动、结算联动均已明确。
