# PDA 结算工厂端边界设计

日期：2026-07-03
状态：已确认设计，待书面规格审查
适用项目：HiGood FCS 原型仓库

## 背景

当前 PDA 端结算页把平台内部结算对象和工厂端可理解对象混在了一起。

代码现状中，PDA 结算详情页仍按 `总览 / 质检扣款 / 正式流水 / 对账与预付款` 组织。边界文档也把工厂端结算描述为查看周期、质检扣款、正式流水、对账与预付款。

新的业务边界已经明确：PDA 是工厂端结算工作台，只负责让工厂查看、核对、确认、申诉和查看付款结果，不负责平台内部的预结算流水管理、预付款批次管理、飞书付款申请或打款回写。

## 目标

本次设计目标是把 PDA 结算整理成工厂能理解的结算入口。

必须满足：

- PDA 主对象是对账单、质检记录和结算资料。
- 付款只作为对账单的只读付款状态展示。
- 不在 PDA 暴露 `预结算流水` 这个平台内部概念。
- 不在 PDA 暴露预付款批次、飞书付款审批、打款回写操作。
- 质检记录展示该工厂全部质检事实，并标记是否关联对账单及关联哪张对账单。
- 对账单详情必须保留工厂可理解的结算明细，说明金额从哪些收入和扣款组成。
- 当前金额统一使用 IDR。

## 非目标

- 不重做真实后端、权限系统、数据库或 API。
- 不改工厂档案完整结算资料模型。
- 不新增审核流程。
- 不把平台 Web 的对账生成、预付款批次、飞书付款申请搬到 PDA。
- 不做独立付款 Tab。
- 不做独立预结算流水 Tab。
- 不重新设计平台端预付款批次页面。

## 当前问题

### 1. PDA 暴露了平台内部对象

`预结算流水` 和 `预付款批次` 是平台端财务和业务人员使用的内部对象。工厂端如果直接看到这些词，会误以为自己需要理解或处理平台内部流程。

### 2. 付款边界容易越界

付款申请、飞书审批和打款回写由平台业务人员或财务处理，跟工厂没有操作关系。PDA 只能展示对账单是否付款，不应该给工厂付款操作入口。

### 3. 质检记录和质检扣款被混在一起

工厂需要看到质检事实本身：生产单、SKU、质检数量、合格数量、返工数量、返工接收对象、瑕疵原因数量。扣款和对账只是附属标记，不能把 `质检记录` 做成只处理扣款的页面。

### 4. 全量质检记录必须可筛选

PDA 要看该工厂全部质检记录，但手机屏幕很小。没有快捷筛选会让现场人员看不动，也难以找到未进对账或有争议的记录。

## 设计原则

### 1. 工厂语言优先

PDA 页面不使用平台内部对象名作为主入口。`预结算流水` 在 PDA 上表达为对账单详情中的 `结算明细`。

### 2. 对账单是金额主对象

工厂确认和申诉围绕对账单发生。付款状态也挂在对账单上，而不是独立成为工厂端付款模块。

### 3. 质检记录是事实来源

PDA 的质检记录必须先展示事实，再弱化展示扣款和对账引用。质检记录不编辑扣款金额。

### 4. 平台操作不上 PDA

预付款批次管理、飞书付款申请、飞书审批状态同步、银行回写登记都属于平台内部流程。PDA 只消费结果。

## PDA 页面设计

### 信息架构

PDA 结算采用以下结构：

```text
结算首页 / 对账单 / 质检记录 / 结算资料
```

`结算首页` 是默认页，用于摘要和待办，不是一个新的业务对象。

### 结算首页

首页展示本厂结算摘要：

- 待确认对账单数量。
- 异议中对账单数量。
- 未付款对账单数量。
- 已付款对账单数量。
- 待关注质检记录数量。
- 当前结算资料版本。

首页只给入口，不展示复杂流水。

### 对账单列表

对账单列表是 PDA 结算的核心。

每张对账单卡片展示：

- 对账单号。
- 对账时间段。
- 结算对象。
- 应付金额。
- 扣款金额。
- 应付净额。
- 对账单状态。
- 工厂确认状态。
- 付款状态：未付款、付款处理中、已付款。
- 已付款时展示付款时间和付款金额。

不展示付款申请、飞书审批、打款回写操作。

### 对账单详情

对账单详情展示：

- 金额汇总。
- 对账单状态。
- 工厂确认或申诉入口。
- 付款状态。
- 结算明细。
- 关联质检记录。
- 本单使用的结算资料版本。

`结算明细` 用工厂能理解的语言展示，不叫预结算流水：

- 任务收入。
- 返工扣款。
- 瑕疵扣款。
- 延误扣款。

每条明细需要展示来源生产单、SKU 或质检记录，便于工厂核对金额。

### 质检记录

质检记录展示该工厂全部质检记录。

列表必须有快捷筛选：

- 全部。
- 未进对账。
- 已进对账。
- 有返工。
- 有扣款。

每条质检记录展示：

- 质检单号。
- 生产单。
- SKU 概览。
- 质检时间。
- 质检数量。
- 合格数量。
- 返工数量。
- 瑕疵数量。
- 关联状态：未进对账 / 已进对账。
- 已进对账时展示对账单号。

详情以质检事实为主：

- SKU 明细。
- 质检数量、合格数量、返工数量。
- 返工接收对象。
- 瑕疵原因和数量。
- 来源后道质检单。

扣款和对账信息只弱展示：

- 关联对账单号。
- 关联扣款金额。
- 关联结算明细。

### 结算资料

结算资料只展示：

- 当前收款账号。
- 当前生效版本。
- 对账单使用版本。
- 资料变更状态。

是否允许工厂在 PDA 发起资料变更，沿用现有权限设计；本次不扩大。

## Web 页面设计

Web 端仍是平台业务和财务操作台。

Web 端继续负责：

- 生成对账单。
- 消费预结算流水。
- 填写瑕疵扣款和延误扣款。
- 管理预付款批次。
- 申请飞书付款。
- 登记打款回写。

这些动作不下放到 PDA。

## 数据边界

PDA 可以读取平台内部对象生成的结果，但不直接把内部对象名暴露给工厂。

对应关系：

| 平台内部对象 | PDA 表达 |
| --- | --- |
| 预结算流水 | 对账单详情里的结算明细 |
| 预付款批次 | 对账单付款状态 |
| 飞书付款审批 | 不展示流程，只折算成付款状态 |
| 打款回写 | 不展示操作，只展示已付款结果 |
| 质检记录 | 质检记录 |
| 扣款记录 | 结算明细中的扣款来源 |

## 验证要求

最小验证需要覆盖：

1. PDA 不出现独立 `预结算流水` Tab。
2. PDA 不出现独立 `付款` Tab。
3. PDA 不展示付款申请、飞书审批申请、打款回写入口。
4. 对账单列表展示付款状态。
5. 对账单详情展示结算明细，但不使用 `预结算流水` 作为工厂端主标题。
6. 质检记录列表展示全部工厂质检记录。
7. 质检记录列表可筛选未进对账、已进对账、有返工、有扣款。
8. 已进对账的质检记录展示对账单号。
9. 质检记录详情以质检事实为主，扣款和对账信息弱展示。
10. Web 端平台内部对象和操作不受影响。
11. Web 和 PDA 金额统一显示 IDR。

## 实施边界

推荐小步实现：

1. 先调整 PDA 结算二级结构和文案边界。
2. 再把对账单列表改为展示付款状态。
3. 再把 `正式流水` 表达收敛到对账单详情的 `结算明细`。
4. 再补 PDA 质检记录列表和关联对账单标记。
5. 最后补最小检查脚本。

不做全局领域模型重构，不改平台 Web 预付款批次生命周期。
