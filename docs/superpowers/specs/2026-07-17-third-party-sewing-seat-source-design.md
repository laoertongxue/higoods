# 三方车缝工厂车位数与评级快照同步规格

日期：2026-07-17

## 1. 背景

三方工厂评级页已经集中展示评级、合作状态、派单策略、结算策略和首单上限。前一版原型中，三方车缝工厂的车位数只维护在评级快照里，导致 3 个问题：

- 工厂档案里看不到三方车缝工厂实际可用于派单的车位数。
- 工厂产能资料无法解释评级页中的大型/小型工厂判断。
- 评级快照可以显示车位数，但这个数字没有和派单、结算、产能资料形成同一业务来源。

本规格把车位数口径收回到工厂主档和产能资料，评级页只读取同步后的结果，不再成为车位数的维护入口。

## 2. 产品目标

本期目标是形成一个可演示、可核查的车位数闭环：

1. 在工厂档案维护三方车缝工厂的车缝车位数。
2. 在工厂产能资料展示同一车位数，便于解释产能来源。
3. 三方工厂评级快照读取时从工厂主档/产能资料同步车位数。
4. 评级页的工厂规模、首单上限和派单规则都使用同步后的车位数派生。
5. 入驻资料转正式工厂时，能给车缝车位数一个初始来源，后续再由工厂档案维护。

## 3. 范围

### 3.1 本期纳入

- 三方车缝工厂主档新增车缝车位数口径。
- 工厂档案编辑抽屉展示并维护“车缝车位数”。
- 工厂产能档案列表和详情展示“车缝车位数”。
- 三方工厂评级读取主档/产能资料中的车位数。
- 按车位数自动派生大型/小型工厂。
- 按大型/小型工厂自动派生首单上限。
- 入驻转正式时，对有车缝能力的工厂初始化车缝车位数。
- 专项检查脚本校验所有三方车缝工厂主档、评级快照、规模和首单上限一致。

### 3.2 本期不纳入

- 不在入驻申请页新增独立“车缝车位数”录入字段。
- 不做真实后端同步任务。
- 不做真实产能计算引擎。
- 不做真实审批流或版本化变更记录。
- 不改造全量工厂产能资料为标准列表页。
- 不扩展到非车缝三方工厂。

## 4. 角色与页面

| 角色 | 页面 | 任务 |
| --- | --- | --- |
| 工厂管理人员 | 工厂档案 | 维护三方车缝工厂的车缝车位数 |
| 产能管理人员 | 工厂产能档案 | 查看车位数、机器数量、有效工人数量和能力资料 |
| PPIC | 三方工厂评级、车缝分配工作台 | 根据评级、车位规模和首单上限判断能否派单 |
| 业务主管 | 三方工厂评级 | 判断转正、拉黑或延长考核 |

## 5. 核心业务口径

### 5.1 车缝车位数

车缝车位数表示三方车缝工厂可用于承接车缝任务的有效车位规模。

它不是机器数量，也不是有效工人数量：

- 机器数量用于说明设备资产。
- 有效工人数量用于说明可参与生产的人力。
- 车缝车位数用于评级规模、试产首单上限和派单风控。

当工厂是三方车缝工厂时，工厂档案必须维护车缝车位数。评级页不得把这个字段作为独立维护来源。

### 5.2 工厂规模

| 车缝车位数 | 工厂规模 |
| --- | --- |
| 大于等于 30 个 | 大型工厂 |
| 小于 30 个 | 小型工厂 |

如果车缝车位数缺失，三方车缝评级治理对象应视为资料不完整，不能把评级页的旧数字当作长期可信来源。

### 5.3 首单上限

| 工厂规模 | 试产首单上限 |
| --- | --- |
| 大型工厂 | 1000 件 |
| 小型工厂 | 300 件 |

首单上限由同步后的车缝车位数派生，不由评级页手工维护。

### 5.4 入驻资料初始口径

入驻申请转正式工厂时，如果申请能力包含车缝，系统用有效工人数量初始化车缝车位数。

这个初始化只解决“转正式后的初始值”，不是长期维护来源。正式合作后，车缝车位数以工厂档案维护值为准，并在产能档案中展示。

## 6. 数据来源优先级

三方工厂评级读取车位数时按以下优先级处理：

1. 优先读取工厂主档的车缝车位数。
2. 如主档没有，则读取工厂产能资料中的车缝车位数。
3. 如仍无有效值，评级快照的原有车位数只作为原型兜底展示，不作为长期业务来源。

在当前原型中，11 个三方车缝工厂必须全部在工厂主档维护车缝车位数，专项检查不允许依赖兜底值通过。

## 7. 页面设计

### 7.1 工厂档案

当工厂满足以下条件时，编辑抽屉展示“车缝车位数”：

- 工厂层级为“三方工厂”。
- 工厂类型为“三方车缝工厂”。

字段要求：

- 字段名称：车缝车位数。
- 输入类型：正整数。
- 帮助文案：用于三方工厂评级规模、首单上限和派单风控同步。
- 保存后进入工厂主档。

列表侧也需要能看到三方车缝工厂的车位数摘要，避免用户必须进入评级页才知道车位规模。

### 7.2 工厂产能档案

产能档案展示同一车位数，和以下字段并列：

- 有效工人数量。
- 机器数量。
- 车缝车位数。
- 工序工艺能力。
- 机器明细。

产能档案本期只展示同步结果，不单独开放编辑入口。编辑入口仍在工厂档案。

### 7.3 三方工厂评级

评级页的“规模”列展示：

- 工厂规模。
- 车缝车位数。
- 来源：工厂档案 / 产能资料。

评级页不得把车位数作为独立维护入口。它只展示已同步后的业务结果。

## 8. 评级与派单规则联动

车位数同步后，以下规则必须使用同步后的值：

| 规则 | 使用车位数的方式 |
| --- | --- |
| 工厂规模 | 大于等于 30 为大型工厂，否则为小型工厂 |
| 首单上限 | 大型工厂 1000 件，小型工厂 300 件 |
| 考核中派单 | 只允许试产单，且本次派单数量不能超过首单上限 |
| 评级列表筛选 | 工厂规模筛选按同步后的规模判断 |
| 统计卡片 | S/A/B/C、正常合作、考核中、黑名单统计按筛选后的同步快照计算 |

派单和结算可执行规则仍沿用既有统一评估函数：

- 派单使用 `evaluateThirdPartyFactoryDispatchPolicy()`。
- 结算使用 `evaluateThirdPartyFactorySettlementPolicy()`。

## 9. 延长考核期逻辑

延长考核期不是“全部禁止”，也不是一个新的黑名单状态。

本期业务含义是：

- 工厂仍处于“考核中”。
- 不转为“正常合作”。
- 不进入“黑名单”。
- 下一次仍只能接试产单。
- 下一次试产仍受当前车位规模派生的首单上限控制。
- 完成交出后，再由主管判断转正、继续延长考核或拉黑。

本期结构化字段：

| 字段 | 含义 |
| --- | --- |
| `assessmentDecision` | 考核结论：转正、拉黑、延长考核 |
| `assessmentRound` | 当前是第几轮试产考核 |
| `assessmentReason` | 延长原因，例如质量波动、交期不稳定、资料不完整、主管要求补单验证 |
| `nextAllowedDocumentType` | 延长考核后的下一单仍为试产单 |
| `nextTrialLimitQty` | 延长考核下一单上限，跟随车位规模派生的试产上限 |

当前原型用“考核中 + `TRIAL_ONLY` + 试产单 + 下一单上限”表达延长考核的可执行结果，并在评级详情中展示“延长考核中”、考核轮次、延长原因和下一单规则。独立的延长审批动作不在本期范围内。

## 11. 对抗式核查要求

### 11.1 数据完整性

- 每一个三方车缝工厂都必须有车缝车位数。
- 每一个三方车缝工厂都必须有评级快照。
- 评级快照中的车位数必须等于主档/产能资料同步结果。
- 规模和首单上限必须由车位数派生，不能手工写死后与车位数冲突。

### 11.2 页面可见性

- 工厂档案能看到并维护车缝车位数。
- 工厂产能档案能看到车缝车位数。
- 三方工厂评级页能看到车位数来源。
- 评级页统计和筛选使用同步后的快照。

### 11.3 交互可达性

- 工厂档案编辑抽屉可修改车缝车位数。
- 切换为非三方车缝工厂时，不显示车缝车位数字段。
- 评级页列设置、分页和详情入口不受车位数展示影响。

### 11.4 业务闭环

- 考核中工厂仍只能接试产单。
- 试产单数量不能超过同步车位规模派生的首单上限。
- 黑名单仍禁止派单和新结算。
- 历史账本仍可查看。

## 12. 验收标准

| 编号 | 验收点 | 通过标准 |
| --- | --- | --- |
| AC-01 | 主档覆盖 | 11 个三方车缝工厂均有车缝车位数 |
| AC-02 | 评级同步 | 评级快照车位数等于工厂主档/产能资料同步结果 |
| AC-03 | 规模派生 | 30 个车位及以上为大型工厂，否则为小型工厂 |
| AC-04 | 首单上限 | 大型工厂 1000 件，小型工厂 300 件 |
| AC-05 | 工厂档案可维护 | 三方车缝工厂编辑抽屉展示“车缝车位数” |
| AC-06 | 产能资料可见 | 工厂产能档案详情展示“车缝车位数” |
| AC-07 | 评级页可追溯 | 评级页展示“来源：工厂档案 / 产能资料” |
| AC-08 | 派单规则联动 | 考核中工厂按同步后的首单上限阻断超量试产单 |
| AC-09 | 结算规则不回退 | 黑名单禁止新结算，历史账本仍可查看 |

## 14. 自检

| 检查项 | 结论 | 说明 |
| --- | --- | --- |
| 是否回答“在哪里添加三方车缝工厂车位数” | 通过 | 在工厂档案维护，产能资料展示，入驻转正式初始化 |
| 是否避免评级快照孤立维护车位数 | 通过 | 评级读取时同步主档/产能资料，专项检查校验一致性 |
| 是否覆盖全部三方车缝工厂 | 通过 | 规格要求 11 个全部覆盖，专项检查断言主档和快照数量一致 |
| 是否说明延长考核期逻辑 | 通过 | 延长考核等同继续保持考核中，仍只允许试产单并受首单上限控制 |
| 是否和派单规则联动 | 通过 | 首单上限由同步后的车位规模派生，派单评估继续消费该结果 |
| 是否和结算规则联动 | 通过 | 黑名单结算拦截保持不变，不因车位数调整回退 |
| 是否符合原型边界 | 通过 | 不引入后端同步任务、审批流、真实产能计算引擎 |
| 是否保留页面可见性 | 通过 | 工厂档案、产能档案、评级页都有对应展示或维护入口 |

## 15. 剩余例外

- 入驻申请页暂不新增“车缝车位数”字段。原因是当前入驻表单已有机器数量、有效工人数量和机器明细，本期只要求转正式后能初始化车位数，并由正式工厂档案维护。
- 当前原型没有独立的“延长考核”审批动作。延长考核通过“继续保持考核中”表达业务结果，下一单仍走试产单和首单上限规则。
- 当前产能资料只展示车位数，不在产能资料页单独编辑。编辑入口统一放在工厂档案，避免同一字段多处维护。
