# 水溶工序及加工单产品设计方案

## 1. 文档信息

- 设计范围：工序工艺字典、技术包 BOM、技术包工序、生产单拆解、染色需求单、染色加工单、独立水溶加工单、FCS、PFOS、PDA
- 设计结论：已完成业务确认
- 适用阶段：生产准备阶段
- 归属工厂：染厂
- 核心约束：同一物料同时需要水溶和染色时，固定先水溶、后染色，并由同一家染厂连续完成

## 2. 背景与问题

花边等物料在进入染色前，可能需要先做水溶处理。当前技术包 BOM 只能表达印花、染色、预缩、水洗等工序，无法正式记录水溶要求，也无法从正式技术包版本生成对应的加工任务。

本次需要解决两类业务：

1. 物料只需要水溶：直接生成独立水溶加工单，并交给染厂执行。
2. 同一物料同时需要水溶和染色：不额外生成水溶单据，而是在染色加工单中增加水溶步骤，由同一家染厂连续完成。

水溶与生产准备时效无关，不进入生产准备时效的统计、催办和阻塞判断。

## 3. 产品目标

1. 在工序工艺字典中形成正式的水溶工序。
2. 技术包 BOM 可以按具体物料勾选水溶。
3. 技术包工序页能够展示水溶及其关联物料。
4. 正式技术包版本进入生产单后，按具体 BOM 物料稳定生成正确的加工单据。
5. 独立水溶任务在 FCS、PFOS、PDA 三端形成完整执行闭环。
6. 水溶加染色场景保持一张染色加工单、一个工厂、一条连续执行链路。
7. 防止不同工艺要求的染色需求被错误合并。

## 4. 明确不在本次范围

- 不新增水溶需求单。
- 不新增连续工序任务这一业务对象。
- 不支持水溶与染色分厂加工。
- 不把水溶接入生产准备时效。
- 不新增水溶专用仓库菜单。
- 不新增水溶专用计价、结算或对账规则。
- 不新增批次管理体系。
- 不允许按备货创建染色加工单时人工附加水溶。
- 不重做 PDA 全局导航和通用任务架构。
- 不改变印花、预缩、水洗及其他既有工序的生成规则。

## 5. 核心业务对象

### 5.1 工序字典项

水溶作为准备阶段的正式工序，归属染厂，排序位于染色之前。

### 5.2 BOM 物料水溶要求

水溶要求记录在具体 BOM 物料行上，而不是只记录在技术包或工序名称层级。所有 BOM 物料类型都允许选择水溶，不预设物料分类限制。

### 5.3 独立水溶加工单

当某 BOM 物料只选择水溶、未选择染色时生成。它是可独立派厂、执行、交接和完成的加工单。

### 5.4 含水溶的染色加工单

当同一 BOM 物料同时选择水溶和染色时生成。它仍然是染色加工单，只增加需先水溶标识和水溶执行步骤，不生成独立水溶加工单。

## 7. 工序工艺字典设计

### 7.1 新增字典项

| 字段 | 设计值 |
|---|---|
| 工序名称 | 水溶 |
| 所属阶段 | 准备阶段 |
| 归属工厂类型 | 染厂 |
| 排序 | 位于染色之前 |
| 启用状态 | 启用 |
| 适用对象 | BOM 物料 |

### 7.2 字典页展示

水溶作为结果型字典项展示，重点表达所属阶段、归属工厂和引用关系。本次不扩展复杂的新增、批量维护或版本管理能力。

### 7.3 工序顺序

同一 BOM 物料同时存在水溶和染色时，系统固定顺序为：

~~~text
水溶 → 染色
~~~

用户不能在技术包中交换该顺序。

## 8. 技术包 BOM 设计

### 8.1 BOM 字段

在每一条 BOM 物料行的工序选择区域新增水溶选项，与染色同级。勾选结果必须绑定当前 BOM 物料标识。

### 8.2 可选范围

所有 BOM 物料均可勾选水溶。系统不根据花边、面料、辅料等分类进行硬性限制。

### 8.3 页面反馈

- 仅勾选水溶：显示水溶标签。
- 仅勾选染色：保持现有染色标签。
- 同时勾选：显示水溶、染色两个标签，并提示固定顺序先水溶、后染色。
- 草稿阶段仅更新技术包内的工序预览，不生成任何加工单据。

### 8.4 变更边界

正式版本冻结后，后续编辑草稿或发布新版本，不得直接改变已按旧版本生成的加工单。新版本只影响后续采用该版本的新生产单或明确重新拆解的业务。

## 9. 技术包工序页设计

技术包工序页新增水溶工序卡片或行，展示：

- 工序名称：水溶。
- 所属阶段：准备阶段。
- 归属：染厂。
- 关联 BOM 物料：展示具体物料名称、编码及用量单位。
- 与染色的关系：同一物料同时存在两项时，展示先水溶、后染色。

工序页必须由 BOM 勾选结果汇总，但仍保留具体关联物料，不得只汇总为技术包级的一个布尔值。

## 10. 正式版本、快照与生成时点

### 10.1 生成时点

单据不是在 BOM 勾选时生成。正确时点为：

1. 技术包草稿维护 BOM。
2. 发布正式技术包版本。
3. 生产单引用并冻结正式技术包快照。
4. 生产单执行拆解。
5. 系统按快照中的具体 BOM 物料和工序组合生成单据。

### 10.2 快照内容

正式技术包快照至少需要保留：

- BOM 物料标识。
- 物料名称、编码、规格和单位。
- 单耗和损耗率。
- 是否需要水溶。
- 是否需要染色。
- 水溶与染色的固定顺序。
- 工序与 BOM 物料的关联关系。

### 10.3 防重复

以生产单、正式技术包快照和 BOM 物料为生成唯一性边界。对同一次拆解重复执行时，不得重复生成独立水溶加工单或染色需求。

## 11. 水溶计划数量

水溶计划数量按 BOM 物料实际适用的生产数量计算：

~~~text
水溶计划数量
= 适用颜色尺码的生产数量
× BOM 单耗
×（1 + BOM 损耗率）
~~~

规则：

- 继承 BOM 物料单位。
- 不直接使用成衣件数作为水溶数量。
- 若物料只适用于部分颜色或尺码，仅计算对应生产数量。
- 计算结果作为加工单计划数量和 PDA 数量校验基准。

## 12. 单据生成规则

| 同一 BOM 物料工序组合 | 染色需求 | 独立水溶加工单 | 染色加工单 | 执行方式 |
|---|---:|---:|---:|---|
| 仅水溶 | 不生成 | 自动生成 | 不生成 | 独立派厂、独立执行、独立交接 |
| 仅染色 | 生成普通需求 | 不生成 | 人工创建 | 保持现有染色流程 |
| 水溶和染色 | 生成需先水溶需求 | 不生成 | 人工创建含水溶染色单 | 同厂先水溶后染色，一次交接 |
| 均未选择 | 不生成 | 不生成 | 不生成 | 无加工任务 |

组合判断只发生在同一 BOM 物料行。同一技术包中物料甲选择水溶、物料乙选择染色，不得合并为含水溶的染色加工单。

## 13. 染色需求单设计

### 13.1 需求标识

染色需求新增工艺前置属性：

- 普通染色。
- 需先水溶。

列表和详情展示需先水溶标签，并在工艺要求中显示水溶 → 染色。

### 13.2 来源

需先水溶标识只能来自正式技术包快照中的同一 BOM 物料，不允许在染色需求单上临时手工添加。

### 13.3 数量

染色需求数量沿用对应 BOM 物料的物料需求口径。水溶完成数量将成为后续开始染色时的可投入上限。

## 14. 染色加工单创建设计

### 14.1 创建方式

保持现有方式：

- 按染色需求创建。
- 按备货创建。

本次不把染色加工单改为自动创建。

### 14.2 多需求合单约束

同一张染色加工单可以关联多个染色需求，但所有被选需求的水溶属性必须一致：

- 普通染色需求只能与普通染色需求合单。
- 需先水溶需求只能与需先水溶需求合单。
- 两类需求不能混合。

交互规则：

1. 用户选择第一条需求后，系统据此过滤或禁用不兼容需求。
2. 若通过其他入口形成混合选择，提交时必须拦截。
3. 提示文案：所选染色需求的水溶要求不一致，请分别创建加工单。

### 14.3 按备货创建

按备货创建缺少正式技术包 BOM 来源，因此本次只能创建普通染色加工单，不提供人工勾选或附加水溶能力。

### 14.4 加工单展示

含水溶的染色加工单显示：

- 需先水溶标签。
- 工艺路线：水溶 → 染色 → 既有后处理。
- 水溶计划数量、完成数量和差异。
- 当前步骤。
- 同一染厂连续加工提示。

它不会出现在独立水溶加工单列表中。

## 15. 染厂分配与能力校验

### 15.1 独立水溶加工单

可以人工选择具备水溶能力的染厂。

### 15.2 含水溶的染色加工单

在分配染色加工单时一次确定染厂。系统只允许选择同时具备水溶能力和染色能力的染厂。

不会出现先分配水溶加工单、再让染色加工单跟随的操作，因为该场景没有独立水溶加工单。

### 15.3 不支持分厂

含水溶的染色加工单从水溶开始到染色及既有后处理结束，始终由同一家染厂执行。本次不提供步骤拆分、转厂或中间交接入口。

## 16. FCS 独立水溶加工单

### 16.1 菜单

在任务编排与执行准备下新增水溶加工单菜单。

### 16.2 列表范围

只展示独立水溶加工单，不展示含水溶的染色加工单。

### 16.3 列表字段

- 水溶加工单号。
- 生产单号。
- 款号或款式。
- BOM 物料名称、编码和规格。
- 计划数量和单位。
- 已完成数量。
- 染厂。
- 当前状态。
- 计划交期。
- 异常标识。
- 来源技术包版本。

### 16.4 筛选

- 加工单号、生产单号、物料关键词。
- 状态。
- 染厂。
- 计划交期。
- 是否异常。

### 16.5 详情

- 来源信息。
- 物料与数量。
- 工艺要求。
- 派厂信息。
- PDA 执行记录。
- 数量差异。
- 交接和收货结果。
- 异常及主管处理记录。

### 16.6 关键动作

- 分配染厂。
- 查看任务。
- 查看执行进度。
- 查看异常。
- 查看交接。

## 17. PFOS 独立水溶加工单

### 17.1 菜单

在染厂管理下新增水溶加工单菜单。

### 17.2 列表范围

只展示分配给当前染厂的独立水溶加工单。

### 17.3 页面重点

- 待处理任务。
- 当前物料和计划数量。
- 当前状态。
- PDA 执行人及最近操作。
- 异常和主管处理入口。
- 待交出任务。

### 17.4 含水溶染色单

仍在染色加工单页面管理，通过需先水溶标签和当前步骤展示，不复制到水溶加工单菜单。

## 18. 独立水溶状态设计

~~~mermaid
stateDiagram-v2
  [*] --> 待分配染厂
  待分配染厂 --> 待原料: 分配染厂
  待原料 --> 待水溶: 原料就绪
  待水溶 --> 水溶中: 开始水溶
  水溶中 --> 待交出: 完成水溶
  水溶中 --> 生产暂停: 数量不足或异常
  生产暂停 --> 水溶中: 主管确认继续
  生产暂停 --> 待水溶: 主管退回重做
  待交出 --> 交出待收货: 完成交出
  交出待收货 --> 已完成: 数量一致并确认收货
  交出待收货 --> 收货差异: 收货数量存在差异
  收货差异 --> 已完成: 差异处理完成
~~~

状态文案在 FCS、PFOS、PDA 保持一致。生产暂停和收货差异是需要明确处理的状态，不得自动跨越。

## 19. 含水溶染色单执行设计

### 19.1 执行链路

保持既有染色加工单的准备、样办、染缸计划等环节，在真正开始染色前插入水溶执行步骤：

~~~text
既有染前准备 → 待水溶 → 水溶中 → 待染色 → 染色中 → 既有后处理 → 待交出
~~~

不要求为了水溶重排所有既有染色前置节点。

### 19.2 硬性前置

- 未完成水溶，不允许开始染色。
- 水溶完成数量为染色可投入数量上限。
- 若水溶完成数量少于计划数量，必须进入差异处理。
- 主管确认部分继续后，只允许在已完成水溶数量范围内开始染色。

### 19.3 交接

水溶完成后不生成中间交接。完成染色及既有后处理后，按原染色加工单完成一次最终交出和收货。

## 20. PDA 产品设计

### 20.1 入口

不新增底部导航。独立水溶任务和含水溶染色任务都进入现有执行列表。

### 20.2 任务分类

- 独立水溶任务：显示水溶加工单标签。
- 含水溶染色任务：仍显示染色加工单标签，并增加需先水溶标签。

### 20.3 搜索

支持按加工单号、生产单号、物料编码、物料名称搜索。含水溶染色任务继续支持现有目标颜色搜索。

### 20.4 列表卡片

首屏只保留现场执行必需信息：

- 当前任务类型。
- 物料名称和编码。
- 计划数量和单位。
- 当前步骤。
- 主动作。
- 异常提示。

管理信息和完整节点历史放入详情或折叠区域。

### 20.5 当前动作优先

任务详情顶部增加当前要做什么区域，只展示一个明确主动作：

| 当前步骤 | 主动作 |
|---|---|
| 待水溶 | 开始水溶 |
| 水溶中 | 完成水溶 |
| 生产暂停 | 查看主管处理 |
| 待染色 | 开始染色 |
| 染色中 | 完成染色 |
| 待交出 | 去交出 |

完整工艺节点保留，但默认折叠，避免现场人员在多个按钮中选错。

### 20.6 扫码与防错

- 支持扫描加工单、物料或现场标签进入任务。
- 扫码结果必须校验任务归属、物料和当前步骤。
- 扫到其他工厂、其他任务或错误物料时，明确提示并禁止操作。
- 已完成水溶后重复扫描进入，不得重复开始或重复完成。

### 20.7 数量录入

完成水溶时录入实际完成数量：

- 默认带出计划数量，允许按现场实际调整。
- 实际完成数量不得小于零。
- 超过计划数量时需二次确认并记录原因。
- 少于计划数量时必须填写原因，并进入生产暂停或差异处理。
- 含水溶染色任务的后续染色投入不得超过水溶完成数量。

### 20.8 弱网与重复提交

- 主动作提交后立即进入处理中状态并禁用重复点击。
- 网络超时保留现场录入内容，允许重试。
- 使用业务动作唯一性避免重复开始、重复完成和重复交出。
- 重试成功后明确显示当前状态，不要求操作员判断是否已提交。

### 20.9 角色

| 角色 | 水溶相关权限 |
|---|---|
| 现场操作员 | 扫码、开始水溶、完成水溶、查看当前任务 |
| 生产主管 | 处理数量不足、生产暂停、确认部分继续或退回 |
| 交接人员 | 对独立水溶加工单执行交出 |
| 管理员 | 查看全链路，并作为异常兜底 |

## 21. 数量不足和异常处理

### 21.1 独立水溶

水溶完成数量不足时：

1. 操作员填写实际数量和原因。
2. 加工单进入生产暂停。
3. 主管选择继续补做、按实际数量继续交出，或退回重做。
4. 处理结果和责任人写入记录。

### 21.2 含水溶染色

水溶完成数量不足时：

1. 不允许直接按原计划数量开始染色。
2. 主管确认是否补做或按实际完成数量继续。
3. 按实际数量继续时，染色投入上限同步调整。
4. 整张加工单仍保持一个工厂和一条执行链路。

### 21.3 路由绕过

即使通过历史链接或直接地址进入后续节点，也必须校验水溶完成状态和数量，不得只依赖前端按钮隐藏。

## 22. 跨端一致性

| 业务信息 | FCS | PFOS | PDA |
|---|---|---|---|
| 独立水溶加工单 | 独立菜单，可派厂和跟踪 | 独立菜单，可管理执行 | 统一执行列表 |
| 含水溶染色单 | 染色加工单内展示 | 染色加工单内展示 | 染色任务内展示当前水溶步骤 |
| 当前步骤 | 展示 | 展示 | 突出主动作 |
| 水溶完成数量 | 展示与追踪 | 展示与异常处理 | 现场录入 |
| 生产暂停 | 查看处理结果 | 主管处理 | 操作员查看状态 |
| 交接 | 查看 | 管理 | 交接人员执行 |

各端共享同一业务状态和数量口径，只允许表现层级不同，不允许产生三套相互矛盾的状态。

## 23. Mock 数据覆盖

原型至少覆盖以下场景：

1. 技术包 BOM 单物料仅水溶。
2. 技术包 BOM 单物料仅染色。
3. 同一物料同时水溶和染色。
4. 不同物料分别水溶和染色，验证不得误组合。
5. 所有物料类型均可选择水溶。
6. 草稿勾选后未生成单据。
7. 正式版本拆解后生成独立水溶加工单。
8. 正式版本拆解后生成需先水溶的染色需求。
9. 重复拆解未生成重复单据。
10. 普通染色需求与需先水溶需求混选被拦截。
11. 按备货创建染色单时无水溶入口。
12. 独立水溶单分配给有水溶能力的染厂。
13. 含水溶染色单选择缺少任一能力的染厂时被拦截。
14. 独立水溶正常开始、完成、交出和收货。
15. 独立水溶数量不足进入生产暂停。
16. 含水溶染色单未完成水溶时无法开始染色。
17. 含水溶染色单按水溶实际完成数量继续染色。
18. 水溶完成后没有中间交接。
19. PDA 扫描错误物料被拦截。
20. PDA 重复点击未产生重复动作。
21. PDA 弱网重试后状态正确。
22. 新技术包版本未修改既有加工单。

## 24. 验收标准

### 24.1 技术包

- 工序字典可看到准备阶段的水溶。
- BOM 每条物料都可选择水溶。
- 技术包工序页能看到水溶及具体关联物料。
- 同物料同时选择时固定显示先水溶、后染色。

### 24.2 单据

- 仅水溶自动生成一张独立水溶加工单。
- 水溶加染色只生成需先水溶的染色需求，后续创建一张含水溶的染色加工单。
- 不生成水溶需求单。
- 不生成连续工序任务。
- 不重复生成。

### 24.3 派厂

- 独立水溶只能分配给具备水溶能力的染厂。
- 含水溶染色单只能分配给同时具备两项能力的染厂。
- 全流程不存在分厂入口。

### 24.4 执行

- 独立水溶在 FCS、PFOS、PDA 可完成全闭环。
- 含水溶染色任务未完成水溶时无法开始染色。
- 水溶完成数量限制染色投入数量。
- 含水溶染色任务只有最终一次交接。

### 24.5 PDA

- 不新增底部导航。
- 当前步骤只突出一个主动作。
- 扫码、数量、角色、弱网和重复提交均有明确防错。
- 状态和数量与 FCS、PFOS 一致。

### 24.6 范围

- 水溶不进入生产准备时效。
- 不新增水溶仓库、结算、批次等范围外能力。
- 不影响其他既有工序的业务规则。

## 25. 原型设计治理要求

后续原型实现应遵循印尼工厂现场协同设计规范，重点检查：

- 角色边界：操作员、主管、交接人员和管理员动作清晰。
- 当前任务：PDA 首屏明确当前物料、数量、步骤和主动作。
- 防错：错误工厂、错误物料、错误步骤和重复提交均能拦截。
- 主管兜底：数量不足和生产暂停有明确处理路径。
- 跨端一致：FCS、PFOS、PDA 共用同一状态与数量口径。
- 中文表达：页面不直接显示英文状态码。
- 性能：弹窗、页签、列表动作和输入不触发不必要的整页重绘。
- 列表：FCS、PFOS 的独立水溶加工单列表必须带分页。

涉及页面、数据、路由或交互实现时，需要新增对应原型审查记录，并运行项目的原型设计治理检查。

## 26. 设计自检结论

本方案已完成以下对抗式检查：

- 对象边界清晰：独立水溶单与含水溶染色单不重复建模。
- 生成边界清晰：由正式技术包快照和生产单拆解触发，不由草稿勾选触发。
- 物料边界清晰：组合判断以同一 BOM 物料为准。
- 合单边界清晰：不同水溶属性的染色需求不能合单。
- 来源边界清晰：按备货创建不能手工附加水溶。
- 工厂边界清晰：含水溶染色全程同厂。
- 数量边界清晰：按 BOM 口径计算，水溶产出限制染色投入。
- 状态边界清晰：独立水溶与含水溶染色分别定义，跨端口径一致。
- PDA 边界清晰：复用统一执行入口，仅增强水溶相关任务表现和动作。
- 范围边界清晰：未扩展到时效、仓库、结算、批次和分厂。
