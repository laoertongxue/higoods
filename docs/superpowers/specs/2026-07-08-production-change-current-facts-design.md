# 生产单变更当前事实详情化设计

## 1. 背景

生产单变更新增页第 1 步当前只展示生产进度摘要，例如“配料：已配 60%”“领料：已领 40%”。这类摘要不能支撑变更判断。

业务人员在发起变更前需要知道：

- 哪些物料需要配，分别需要多少。
- 每个物料已配多少、已领多少、还有多少可以改。
- 生产单原数量、当前数量、色码数量和已生成单据数量是否一致。
- 已生成哪些下游单据，生成时间、当前状态是什么。
- 单据是否已经配料、领料、开工、交出、结算。
- 哪些范围可改，哪些需要退换/追回，哪些已经不可追回只能追溯或补差。

因此第 1 步应从“进度摘要”升级为“生产单当前事实”。

## 2. 设计目标

新增页第 1 步采用：

> 模块分组 + 行级明细 + 数量差异。

默认展示变更判断必需事实；每个模块提供“展开全部”，查看完整单据和执行记录。

本设计不把第 1 步做成完整生产单台账。工厂报价、完整库存流水、完整质检扣款、完整产能日历等不进入默认展示。

## 3. 展示范围

第 1 步只覆盖生产单变更会直接用到的 7 类事实：

1. 生产数量 / 色码数量事实
2. 物料配料 / 领料事实
3. 裁剪 / 铺布 / 裁片事实
4. 印花 / 染色 / 特殊工艺事实
5. 车缝 / 后道 / 交出事实
6. 结算 / 成本已发生事实
7. 历史变更 / 补丁 / 审核锁定事实

## 4. 页面结构

### 4.1 顶部事实摘要

展示生产单、款式、冻结技术包、最新正式版、交期、数量概览、当前事实摘要。

示例：

| 字段 | 示例 |
| --- | --- |
| 生产单 | PO-202603-0004 |
| 款式 / SPU | Celana Jogger Pria / SPU-2024-010 |
| 冻结技术包 | 正式版 v1.0 |
| 最新正式版 | 正式版 v1.2 |
| 交期 | 2026-04-20 |
| 生产数量 | 原 1200 件 / 当前 1200 件 |
| 当前事实摘要 | 已配料、已领料、已铺布、有历史变更 |

### 4.2 生产数量 / 色码数量事实

覆盖“已经生成生产单后，需要修改生产数量”的场景。这里展示数量事实，不在第 1 步直接选择增减单据处理方式。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 色码范围 | 颜色、尺码、批次 |
| 原生产数量 | 生产单创建时的数量 |
| 当前生产数量 | 当前生产单记录的数量 |
| 本次拟变更数量 | 第 2 步填写变更内容后用于第 3 步测算的增减数量 |
| 已生成单据数量 | 配料、裁剪、印花、染色、车缝等已生成单据覆盖的数量 |
| 已执行数量 | 已配、已领、已裁、已加工、已交出的数量 |
| 未执行数量 | 尚未进入下游执行、可直接按新数量生成或调整的数量 |
| 数量差异说明 | 增加、减少、色码间调拨、取消部分尺码等事实说明 |

数量变更需要支持以下情况：

- 总数增加：已生成生产单后追加生产数量。
- 总数减少：客户取消部分数量或计划缩量。
- 色码结构调整：总数不变，但颜色 / 尺码数量重新分配。
- 批次调整：总数不变，但拆批或合批影响下游单据。
- 已执行后减量：已配、已领、已裁或已加工的部分不能简单删除，需要进入第 3 步判断退料、作废、补差或追溯。

### 4.3 物料配料 / 领料事实

按物料行展示，不只展示百分比。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 物料 | 物料编码、名称、颜色、规格 |
| 应配 | 当前生产单需要配料数量 |
| 已配 | 已确认配料数量 |
| 已领 | 已领料数量 |
| 剩余可改 | 尚未领用或尚未使用、可按新口径处理的数量 |
| 单据来源 | 配料单、领料单、配料记录 |
| 事实说明 | 是否存在缺料、超配、已领未用、已用不可追回 |

### 4.4 裁剪 / 铺布 / 裁片事实

按批次 / 颜色 / 尺码 / 裁剪对象展示。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 批次 / 色码 | 受影响生产范围 |
| 计划数量 | 计划裁剪数量 |
| 已铺布 | 已铺布张数或层数 |
| 已裁 | 已裁数量 |
| 可改数量 | 未裁或可追回数量 |
| 不可追回数量 | 已裁、已交出或无法改做数量 |
| 事实说明 | 已铺布、已裁、已交出、可改数量的事实说明 |

### 4.5 印花 / 染色 / 特殊工艺事实

按工单展示。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 工单号 | 印花单、染色单、特殊工艺单 |
| 生成时间 | 单据生成时间 |
| 当前状态 | 未开始、加工中、已完成、已取消等 |
| 计划数量 | 工单计划处理数量 |
| 已加工数量 | 已完成或已投入加工数量 |
| 是否可取消重开 | 系统根据状态给出判断 |
| 事实说明 | 当前工单是否已投入、已完成、已取消或待重开 |

### 4.6 车缝 / 后道 / 交出事实

按交出对象或任务对象展示。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 任务 / 交出单 | 车缝任务、后道任务、交出单 |
| 生成时间 | 单据生成时间 |
| 当前状态 | 未交出、部分交出、已交出、已回写 |
| 计划数量 | 应交或应处理数量 |
| 已交出 | 已交出数量 |
| 待交出 | 尚未交出数量 |
| 事实说明 | 未交出、部分交出、已交出、是否已回写结算依据 |

### 4.7 结算 / 成本已发生事实

展示是否已有结算或成本事实，不展示完整财务台账。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 成本类型 | 料、工、费 |
| 已发生金额 | 已产生或已进入结算口径的金额 |
| 关联单据 | 结算单、对账单、扣款单、补差单 |
| 当前状态 | 未结算、部分结算、已结算、暂缓结算 |
| 事实说明 | 是否需要重算、追加差异、扣减差异或补差 |

### 4.8 历史变更 / 补丁 / 审核锁定事实

展示会影响本次变更判断的历史记录。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 变更单号 | 历史变更单 |
| 变更结果 | 版本关系变更、生产单层补丁、版本 + 补丁、仅成本、仅记录 |
| 状态 | 草稿、审核中、已通过、已驳回、已完成 |
| 影响范围 | 色码、批次、工序、单据范围 |
| 锁定状态 | 无锁定、影响范围锁定、整单暂停、已释放 |
| 事实说明 | 是否与本次变更冲突 |

## 5. 展开规则

默认展示每个模块最关键的 3-5 行。

当数量较多时：

- 数量行按本次数量差异优先排序：有增减 / 已生成单据 / 已执行 / 未执行。
- 物料行按变更相关性排序：有缺口 / 已领 / 已用 / 可改。
- 单据按生产链路排序：配料、领料、裁剪、印花、染色、特殊工艺、车缝、交出、结算。
- 历史变更按未完成优先，再按时间倒序。

“展开全部”后展示完整行，但仍保持在第 1 步内，不跳转到其他页面。

## 6. 与后续步骤的关系

第 1 步只负责展示事实，不让业务人员选择处理方式。

第 3 步根据这些事实生成：

- 影响范围
- 数量增减后的色码 / 批次 / 单据差异
- 可改做 / 不可追回数量
- 受影响单据
- 系统建议处理方式

第 4 步再汇总：

- 单据处理后结果
- 成本影响
- 时效影响
- 执行方式

## 7. 验收标准

1. 第 1 步不再只展示“已配 60%”这类百分比摘要。
2. 生产数量变更场景至少能看到原数量、当前数量、拟变更数量、已生成单据数量、已执行数量、未执行数量。
3. 物料配料 / 领料至少能看到应配、已配、已领、剩余可改。
4. 已生成单据至少能看到单据号、生成时间、当前状态、数量事实和事实说明。
5. 裁剪、印染、车缝、交出、结算模块都有行级事实入口。
6. 页面不展示未定义清楚的风险标签；第 1 步只展示事实，第 3 步再给处理建议。
7. 默认视图信息可控，完整事实通过“展开全部”查看。
8. 不新增快捷路径，不改变生产单变更 4 步主流程。
