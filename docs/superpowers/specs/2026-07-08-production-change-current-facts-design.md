# 生产单变更当前事实详情化设计

## 1. 背景

生产单变更新增页第 1 步当前只展示生产进度摘要，例如“配料：已配 60%”“领料：已领 40%”。这类摘要不能支撑变更判断。

业务人员在发起变更前需要知道：

- 哪些物料需要配，分别需要多少。
- 每个物料已配多少、已领多少、还有多少可以改。
- 已生成哪些下游单据，生成时间、当前状态是什么。
- 单据是否已经配料、领料、开工、交出、结算。
- 哪些范围可改，哪些需要退换/追回，哪些已经不可追回只能追溯或补差。

因此第 1 步应从“进度摘要”升级为“生产单当前事实”。

## 2. 设计目标

新增页第 1 步采用：

> 模块分组 + 风险标识 + 行级明细。

默认展示变更判断必需事实；每个模块提供“展开全部”，查看完整单据和执行记录。

本设计不把第 1 步做成完整生产单台账。工厂报价、完整库存流水、完整质检扣款、完整产能日历等不进入默认展示。

## 3. 展示范围

第 1 步只覆盖生产单变更会直接用到的 6 类事实：

1. 物料配料 / 领料事实
2. 裁剪 / 铺布 / 裁片事实
3. 印花 / 染色 / 特殊工艺事实
4. 车缝 / 后道 / 交出事实
5. 结算 / 成本已发生事实
6. 历史变更 / 补丁 / 审核锁定事实

## 4. 页面结构

### 4.1 顶部事实摘要

展示生产单、款式、冻结技术包、最新正式版、交期、关键风险。

示例：

| 字段 | 示例 |
| --- | --- |
| 生产单 | PO-202603-0004 |
| 款式 / SPU | Celana Jogger Pria / SPU-2024-010 |
| 冻结技术包 | 正式版 v1.0 |
| 最新正式版 | 正式版 v1.2 |
| 交期 | 2026-04-20 |
| 当前关键风险 | 已配料、已领料、已铺布、有历史变更 |

### 4.2 物料配料 / 领料事实

按物料行展示，不只展示百分比。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 物料 | 物料编码、名称、颜色、规格 |
| 应配 | 当前生产单需要配料数量 |
| 已配 | 已确认配料数量 |
| 已领 | 已领料数量 |
| 剩余可改 | 尚未领用或尚未使用、可按新口径处理的数量 |
| 单据来源 | 配料单、领料单、配料记录 |
| 当前判断 | 未领可改、已领需退换、已用不可追回、缺料待补 |

### 4.3 裁剪 / 铺布 / 裁片事实

按批次 / 颜色 / 尺码 / 裁剪对象展示。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 批次 / 色码 | 受影响生产范围 |
| 计划数量 | 计划裁剪数量 |
| 已铺布 | 已铺布张数或层数 |
| 已裁 | 已裁数量 |
| 可改数量 | 未裁或可追回数量 |
| 不可追回数量 | 已裁、已交出或无法改做数量 |
| 当前判断 | 可改做、需追回、不可追回仅追溯 |

### 4.4 印花 / 染色 / 特殊工艺事实

按工单展示。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 工单号 | 印花单、染色单、特殊工艺单 |
| 生成时间 | 单据生成时间 |
| 当前状态 | 未开始、加工中、已完成、已取消等 |
| 计划数量 | 工单计划处理数量 |
| 已加工数量 | 已完成或已投入加工数量 |
| 是否可取消重开 | 系统根据状态给出判断 |
| 当前判断 | 可取消重开、当前单改做、下批生效、不处理 |

### 4.5 车缝 / 后道 / 交出事实

按交出对象或任务对象展示。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 任务 / 交出单 | 车缝任务、后道任务、交出单 |
| 生成时间 | 单据生成时间 |
| 当前状态 | 未交出、部分交出、已交出、已回写 |
| 计划数量 | 应交或应处理数量 |
| 已交出 | 已交出数量 |
| 待交出 | 尚未交出数量 |
| 当前判断 | 未交出可改、已交出需追溯、已结算需补差 |

### 4.6 结算 / 成本已发生事实

展示是否已有结算或成本事实，不展示完整财务台账。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 成本类型 | 料、工、费 |
| 已发生金额 | 已产生或已进入结算口径的金额 |
| 关联单据 | 结算单、对账单、扣款单、补差单 |
| 当前状态 | 未结算、部分结算、已结算、暂缓结算 |
| 当前判断 | 可重算、追加差异、扣减差异、只能补差 |

### 4.7 历史变更 / 补丁 / 审核锁定事实

展示会影响本次变更判断的历史记录。

必备字段：

| 字段 | 说明 |
| --- | --- |
| 变更单号 | 历史变更单 |
| 变更结果 | 版本关系变更、生产单层补丁、版本 + 补丁、仅成本、仅记录 |
| 状态 | 草稿、审核中、已通过、已驳回、已完成 |
| 影响范围 | 色码、批次、工序、单据范围 |
| 锁定状态 | 无锁定、影响范围锁定、整单暂停、已释放 |
| 当前判断 | 是否与本次变更冲突 |

## 5. 风险标识

每一行事实必须有一个面向业务的判断标签。

推荐标签：

- 未领可改
- 未开工可改
- 已领需退换
- 已铺布需确认
- 已加工需追回
- 已交出仅追溯
- 已结算需补差
- 有锁定不可直接改

这些标签不替代第 3 步处理建议，只帮助业务人员在第 1 步理解事实。

## 6. 展开规则

默认展示每个模块最关键的 3-5 行。

当数量较多时：

- 物料行按风险优先排序：有缺口 / 已领 / 已用 / 可改。
- 单据按生产链路排序：配料、领料、裁剪、印花、染色、特殊工艺、车缝、交出、结算。
- 历史变更按未完成优先，再按时间倒序。

“展开全部”后展示完整行，但仍保持在第 1 步内，不跳转到其他页面。

## 7. 与后续步骤的关系

第 1 步只负责展示事实，不让业务人员选择处理方式。

第 3 步根据这些事实生成：

- 影响范围
- 可改做 / 不可追回数量
- 受影响单据
- 系统建议处理方式

第 4 步再汇总：

- 单据处理后结果
- 成本影响
- 时效影响
- 执行方式

## 8. 验收标准

1. 第 1 步不再只展示“已配 60%”这类百分比摘要。
2. 物料配料 / 领料至少能看到应配、已配、已领、剩余可改。
3. 已生成单据至少能看到单据号、生成时间、当前状态、数量事实和变更判断。
4. 裁剪、印染、车缝、交出、结算模块都有行级事实入口。
5. 每行事实都有业务判断标签。
6. 默认视图信息可控，完整事实通过“展开全部”查看。
7. 不新增快捷路径，不改变生产单变更 4 步主流程。
