# HiGood 印尼工厂现场协同系统原型审查清单

## 1. 使用说明

本文档用于审查和校验已经完成的 HiGood 产品原型。审查对象包括 FCS、PFOS、WLS、PCS 中会被工厂人员使用的 Web 和 PDA 页面。

审查目标不是判断页面是否「好看」，而是判断页面是否能在印尼工厂真实推进：

- 一线员工能不能看懂。
- 员工会不会算错。
- 员工会不会选错。
- 主管能不能兜底。
- 管理端能不能追溯事实。
- Web、PDA、仓储、结算之间是否协作一致。

每次审查应记录页面路径、角色、端类型、问题标签和结论。

## 3. 角色匹配检查

### 检查项

- 页面是否明确服务某一类角色，而不是把所有人放在同一页面里？
- 页面是否区分管理端、主管端、员工执行端？
- 员工执行端是否隐藏了不该由员工判断的管理信息？
- 主管端是否能看到异常、差异、待处理和兜底入口？
- 管理端是否能看到全链路、来源和追溯记录？

### 不合格信号

- 一线员工页面出现大量统计、日志、流水、配置字段。
- 员工必须理解完整业务流程才能操作。
- 主管无法介入异常。
- 管理端只能看到结果，不能追溯事实来源。

### 问题标签

`字段过载`、`缺主管兜底`、`协作断裂`

## 4. 任务清晰度检查

### 检查项

- 用户进入页面后，是否 3 秒内知道现在要做什么？
- 页面是否明确展示当前任务、当前对象、当前动作、当前结果？
- 员工执行端首屏是否只有一个主动作？
- 复杂动作是否拆成短步骤？
- 页面是否避免让员工先找菜单、选模块、理解状态机？

### 不合格信号

- 主按钮不明显。
- 同一屏出现多个同级主动作。
- 用户必须读一段说明后才知道下一步。
- 列表里有很多相似任务，需要靠文字判断。

### 问题标签

`读不懂`、`点错风险`、`选不对`

## 5. 信息架构与导航检查

### 检查项

- 员工执行端是否优先进入待办或扫码，而不是功能菜单？
- 管理端菜单是否反映业务对象和流程边界？
- 页面跳转是否和上游 / 下游业务关系一致？
- 从 Web 打开的移动端任务，是否能在 PDA 或现场执行列表中找到同一事实？
- 返回、取消、继续处理路径是否清楚？

### 不合格信号

- 员工需要知道「接单 / 执行 / 交接 / 仓管 / 结算」之间的系统关系。
- 页面入口多，但任务入口不明显。
- Web 和 PDA 看到的任务口径不一致。

### 问题标签

`读不懂`、`协作断裂`

## 6. 页面模式检查

### 检查项

- 工作台是否按角色突出最重要的待处理事项？
- 列表页是否匹配角色信息密度？
- 员工执行端是否优先用卡片、步骤、扫码结果，而不是复杂表格？
- 详情页是否按角色分层展示？
- 仓储页是否围绕「物在哪里」组织？
- 交接页是否围绕「谁交给谁」组织？
- 质检页是否围绕质量事实组织？
- 技术资料页是否能帮助工厂避免做错？

### 不合格信号

- 员工执行端直接照搬管理端表格。
- 仓储页面只像库存表，不表达收发存责任。
- 交接页面只像单据详情，不表达交出方、接收方和差异。
- 技术资料页只展示编码和表格，缺少图片、颜色、尺码、工艺重点。

### 问题标签

`字段过载`、`选不对`、`协作断裂`

## 7. 信息负荷检查

### 检查项

- 员工执行端字段是否足够少？
- 是否存在过多 KPI 卡片、Tab、筛选、表格列、行操作？
- 首屏是否展示了当前动作，而不是展示统计和说明？
- 弹窗是否过长？
- 是否存在横向滚动或低分辨率下按钮不可见？

### 不合格信号

- 员工端页面需要上下左右找信息。
- 一张卡片内有超过现场识别所需的信息。
- 低分辨率下无法完成提交。
- 筛选条件多到像管理后台。

### 问题标签

`字段过载`、`读不懂`、`视觉干扰`

## 8. 文案检查

### 检查项

- 文案是否清晰、简洁、准确、一致、友好、尊重？
- 按钮是否写具体动作？
- 是否避免系统抽象词？
- 错误文案是否说明哪里错、为什么错、怎么改？
- 同一动作在 FCS、PFOS、WLS、PCS 中是否同名？
- 是否避免责备用户？

### 不合格信号

- 使用「提交」「处理」「执行流程」但上下文不清。
- 员工端出现「写回」「投影」「来源记录」「执行对象」「状态流转」「链路」。
- 错误只写「操作失败」「输入错误」「参数异常」。
- 同一动作混用「交出」「发起交出」「提交交接」「登记确认」。

### 问题标签

`读不懂`、`状态抽象`

## 9. 数量与状态检查

### 检查项

- 数量是否带单位？
- 系统是否自动计算已领、应领、剩余、已交、应交、差异？
- 页面是否避免让员工心算？
- 差异是否直接显示多 / 少多少？
- 状态是否翻译成现场动作？
- 页面是否避免展示英文状态码？

### 不合格信号

- 只显示数字不显示单位。
- 员工要自己算剩余数量。
- 只写「存在差异」，不写差多少。
- 员工端展示 `PENDING`、`DONE`、`IN_PROGRESS`。
- 员工端只展示「待回写」「状态变更」等抽象状态。

### 问题标签

`算不准`、`状态抽象`

## 10. 扫码与识别检查

### 检查项

- 能扫码的地方是否优先扫码？
- 扫码后是否立刻展示系统识别结果？
- 是否用图片、颜色、尺码、袋号、库位辅助识别？
- 列表选择是否只是兜底？
- 扫错对象时是否阻断并告诉用户扫什么？

### 不合格信号

- 员工必须从多个相似对象里手选。
- 只靠长编号识别任务或物料。
- 扫码失败只提示失败，没有下一步。
- 扫错对象仍允许继续提交。

### 问题标签

`缺扫码识别`、`选不对`、`点错风险`

## 11. 防错检查

### 检查项

- 是否阻断超量、扫错、重复提交、漏填、状态不允许操作？
- 提交前是否有可理解的确认信息？
- 危险动作是否二次确认？
- 是否有主管处理入口？
- 是否有异常上报入口？

### 不合格信号

- 只提醒不阻断。
- 超量仍可提交。
- 危险动作和主动作摆在一起。
- 异常只能写备注，没有结构化入口。
- 员工不知道出错后找谁。

### 问题标签

`点错风险`、`缺主管兜底`

## 12. UI 样式检查

### 检查项

- 页面风格是否清晰、稳定、克制？
- 员工执行端字号是否足够大？
- 主按钮是否唯一且明显？
- 危险按钮是否隔离？
- 状态色是否克制？
- 异常是否突出？
- 图标是否有文字说明？
- 图片是否能帮助识别真实对象？
- 低分辨率下主要内容和按钮是否可见？

### 不合格信号

- 满屏红黄绿状态色。
- 主按钮不明显。
- 图标按钮没有文字，员工不知道含义。
- 装饰卡片过多，真实动作不突出。
- 低分辨率下弹窗提交按钮不可见。

### 问题标签

`视觉干扰`、`点错风险`、`组件误用`

## 13. 组件交互检查

### 检查项

- 筛选是否适合当前角色？
- Tabs 是否过多？
- 弹窗和抽屉是否过长？
- 表单是否让员工手填过多？
- 上传失败是否可恢复？
- 扫码失败是否有明确下一步？
- 分页是否会让员工找不到任务？
- 状态徽章是否可理解？
- 步骤条是否简短？

### 不合格信号

- 员工执行页有复杂筛选。
- 表单大量手填。
- 上传失败后用户不知道是否保存。
- 分页隐藏了当前任务。
- 状态徽章只是后台术语。

### 问题标签

`组件误用`、`字段过载`、`读不懂`

## 14. 协作关系检查

### 检查项

- 页面是否清楚表达谁交给谁？
- 谁确认了什么？
- 谁对数量、质量、时间负责？
- 差异发生后谁处理？
- Web、PDA、仓储、结算是否读取同一事实？
- 操作记录是否能证明责任转移？

### 不合格信号

- 页面只展示状态，不展示责任方。
- 交接页面没有交出方和接收方。
- 差异页面没有处理人和处理路径。
- 管理端与现场端看到的数量或状态口径不同。

### 问题标签

`协作断裂`、`追溯不足`

## 15. 异常与追溯检查

### 检查项

- 异常是否结构化？
- 是否记录操作人、时间、原因、照片、差异数量？
- 是否能追溯到任务、交接单、仓储记录、质检记录、结算记录？
- 异议、代确认、差异重交等流程是否说明清楚？
- 异常是否能回到管理端处理？

### 不合格信号

- 异常只能写自由备注。
- 没有照片或扫码证据。
- 只有结果，没有过程。
- 代确认没有说明不是工厂本人确认。
- 差异进入死胡同，无法继续处理。

### 问题标签

`追溯不足`、`协作断裂`

## 16. 现场设备可用性检查

### 检查项

- PDA 小屏是否可用？
- 1024 x 768 是否可用？
- 工厂电脑低分辨率是否可用？
- 扫码枪输入是否可用？
- 弱网、上传慢、重复点击是否有处理？
- 多人共用设备时是否能识别当前账号和角色？

### 不合格信号

- 弹窗过宽。
- 表格横向滚动后看不到操作按钮。
- 扫码枪输入后页面无响应。
- 上传失败后只能重新进入页面。
- 账号切换和角色身份不清。

### 问题标签

`组件误用`、`点错风险`、`缺主管兜底`

## 17. 审查结论

审查结论分 3 类：

| 结论 | 标准 |
| --- | --- |
| 通过 | 页面符合角色、任务、文案、视觉、防错和协作要求，可继续作为原型演示。 |
| 有条件通过 | 页面主流程可演示，但存在局部问题，需要记录问题并排期优化。 |
| 不通过 | 页面会导致员工读不懂、算不准、选不对、点错，或无法支撑协作事实。 |

## 18. 固定问题标签

审查时使用以下固定标签：

- `读不懂`：页面表达、文案或结构让用户难以理解。
- `算不准`：需要用户心算或数量表达不清。
- `选不对`：用户容易选错任务、物料、袋子、库位或对象。
- `点错风险`：按钮、布局、确认机制导致误操作。
- `状态抽象`：状态名偏系统内部，不能转成现场动作。
- `字段过载`：字段、表格、筛选、Tab、日志过多。
- `缺主管兜底`：异常或不确定场景没有主管介入路径。
- `缺扫码识别`：应扫码或图像识别的场景仍依赖手选。
- `协作断裂`：页面没有表达责任转移或跨端事实不一致。
- `追溯不足`：缺少操作人、时间、原因、照片、差异、来源记录。
- `视觉干扰`：颜色、卡片、图标、密度影响识别重点。
- `组件误用`：组件形态不适合角色和现场设备。
