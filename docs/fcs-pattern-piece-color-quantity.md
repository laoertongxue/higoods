# DXF/RUL 裁片明细颜色片数收口

## 范围

本次只调整纸样管理第二步“版师技术信息”里的裁片明细结构。DXF/RUL 原有解析能力保留，解析结果落表时不再生成可编辑“片数”列，而是生成“解析参考片数”，再由用户按适用颜色维护每种颜色片数。

## 规则

1. 裁片明细主表不再直接维护片数。
2. 每行按“适用颜色与颜色片数”维护。
3. 当前部位总片数 = 当前行所有已启用颜色的颜色片数合计。
4. 纸样当前总片数 = 所有裁片明细行的当前部位总片数合计。
5. DXF Quantity 进入“解析参考片数”，只作为参考，不参与最终总片数计算。
6. 保存纸样和技术包版本快照时，同时保留 `colorPieceQuantities`、`totalPieceQty`、`patternTotalPieceQty` 和 `parsedQuantity`。

## DXF/RUL 落表

DXF/RUL 解析后生成裁片明细行：

- 部位名称来自 DXF 部位名称。
- 尺码来自 RUL 尺码。
- 备注来自 DXF Annotation。
- 解析参考片数来自 DXF Quantity。
- 颜色片数根据技术包颜色初始化，最终以用户维护值为准。

## 不做内容

本次不做逐片特殊工艺，不做菲票逐片生成，不改捆条维护，不改重复纸样校验，不改特殊工艺任务生成。

## 流程图

上传DXF和RUL
-> 解析部位名称尺码备注和解析参考片数
-> 生成裁片明细行
-> 初始化适用颜色和颜色片数
-> 用户维护每种颜色片数
-> 系统计算当前部位总片数
-> 系统计算纸样当前总片数
-> 保存纸样和技术包快照

## 状态机

待解析
-> 已解析裁片明细
-> 待维护颜色片数
-> 已维护颜色片数
-> 当前总片数已计算
-> 已保存

异常状态：

颜色片数无效
-> 停留当前行
-> 显示中文错误提示
-> 修改后重新计算
