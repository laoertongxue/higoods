# 印花 / 染色 Web 操作弹窗与派单口径

## 业务说明

印花加工单和染色加工单在创建时已经指定工厂，不参与竞价、报价、待定标或接单流程。分配方式固定为派单，不允许在加工单详情中编辑。派单价格必须带币种和计价单位，当前演示口径为 `IDR/Yard`，例如 `1200 IDR/Yard`。

印花 / 染色加工单转为工厂端移动应用任务后，直接进入执行模块的待加工、加工中、待交出、待回写、待审核、异常或已完成口径，不进入工厂端移动应用接单模块。平台侧也按执行链路读取状态，不展示竞价、报价、待定标、待接单作为印花 / 染色加工单状态。

## Web 操作要求

工艺工厂 Web 端所有印花 / 染色状态操作都必须先弹出操作弹窗，填写本次操作信息后才能执行。弹窗按动作动态展示必填字段，例如操作人、操作时间、打印机编号、染缸号、完成面料米数、完成裁片数量、交出卷数等。

弹窗确认后调用 `executeProcessWebAction`，再由统一写回服务 `executeProcessAction` 写回事实源、生成操作记录、触发仓和交出联动。按钮不再通过 `data-nav + webAction` 作为主交互直接写回。

## 操作记录

详情页统一展示为“操作记录”，不再使用“Web 端操作记录”。操作记录合并 Web 端与移动端：

- Web 端操作
- 工厂端移动应用操作

合并规则为按加工单 `sourceId` 和移动端 `taskId` 查询统一操作记录，使用 `operationRecordId` 去重，并按 `operatedAt` 倒序展示。来源列保留“Web 端”和“移动端”。

## 中文流程图

创建印花或染色加工单
-> 指定工厂
-> 分配方式固定为派单
-> 填写派单价格
-> 生成移动端执行任务
-> 进入执行待加工
-> Web端或移动端执行操作
-> 弹窗填写操作信息
-> 统一写回
-> 生成操作记录
-> 平台侧同步结果

## 中文状态机

已创建
-> 已派单
-> 待加工
-> 加工中
-> 待交出
-> 待回写
-> 待审核
-> 已完成

异常流转：

待回写
-> 数量差异
-> 异常
-> 处理差异
-> 重新交出或确认完成
