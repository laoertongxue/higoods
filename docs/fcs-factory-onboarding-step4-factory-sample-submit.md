# 工厂入驻第 4 步：工厂端样衣确认与样衣提交

## 本次定位

本步骤建立在平台已完成“登记并发放样衣”之后，只补齐工厂端移动应用里的两个动作：

- 工厂确认收到样衣。
- 工厂提交样衣审核资料。

本步骤不实现平台样衣审核、不实现样衣审核通过转正式合作、不生成工厂档案、不转正式管理员账号，也不开放执行、交接、仓管、结算。

## 工厂确认收到样衣规则

当入驻申请和样衣验证对象处于 `待工厂确认收样` 时，工厂端展示样衣验证信息，并提供 `确认收到样衣` 操作。

确认收样需要填写：

- 确认收样时间：必填。
- 确认收样人：必填，默认当前登录账号管理员姓名。
- 收样备注：非必填。

确认后：

- `FactorySampleVerification.status` 变为 `待工厂提交样衣审核`。
- `FactorySampleVerification.currentNode` 变为 `工厂提交样衣审核`。
- 写入 `factoryReceivedAt`、`factoryReceivedBy`、`factoryReceiveRemark`、`receiveActionCount`。
- 写入样衣验证 `actionLogs`，动作名为 `工厂确认收到样衣`。
- 入驻申请状态同步变为 `待工厂提交样衣审核`，当前节点仍为 `样衣验证`。

## 工厂提交样衣审核资料规则

当状态为 `待工厂提交样衣审核` 时，工厂端展示提交样衣审核资料表单。

必填字段：

- 上传样衣照片：支持 jpg、jpeg、png、webp，允许多张。
- 上传样衣视频：支持 mp4、mov，允许多个。
- 工艺说明：说明样衣制作过程、使用工艺和关键注意点。

非必填字段：

- 问题说明。
- 备注。
- 补充文件：支持 pdf、jpg、jpeg、png、webp、mp4、mov。

提交后：

- `FactorySampleVerification.status` 变为 `待平台审核样衣`。
- `FactorySampleVerification.currentNode` 变为 `平台审核样衣`。
- 写入 `factorySubmittedAt`、`factorySubmittedBy`、`factorySamplePhotos`、`factorySampleVideos`、`factoryCraftDescription`、`factoryProblemDescription`、`factorySubmitRemark`、`factorySubmissionFiles`、`factorySubmissionRoundNo`、`submissionActionCount`。
- 写入样衣验证 `actionLogs`，动作名为 `工厂提交样衣审核`。
- 入驻申请状态同步变为 `待平台审核样衣`，当前节点变为 `样衣审核`。

## 状态流转

中文流程图：

```text
平台发放样衣
-> 工厂端展示样衣验证信息
-> 工厂确认收到样衣
-> 状态变为待工厂提交样衣审核
-> 工厂上传样衣照片和视频
-> 工厂填写工艺说明
-> 提交样衣审核
-> 状态变为待平台审核样衣
-> 平台等待下一步审核
```

中文状态图：

```text
待工厂确认收样
-> 待工厂提交样衣审核
-> 待平台审核样衣

预留后续状态：

待平台审核样衣
-> 样衣审核退回
-> 待工厂提交样衣审核

待平台审核样衣
-> 样衣审核拒绝

待平台审核样衣
-> 样衣审核通过待转正式
```

## 平台端展示

平台“工厂入驻管理”的“样衣验证”页签本次只做只读展示：

- `待工厂提交样衣审核`：展示工厂已确认收样、确认收样时间、确认收样人、收样备注。
- `待平台审核样衣`：展示工厂已提交样衣审核资料、提交时间、提交人、样衣照片、样衣视频、工艺说明、问题说明、备注、补充文件。

平台样衣审核按钮和审核动作放到第 5 步实现。

## 业务准入限制

以下状态仍不能进入执行、交接、仓管、结算：

- 待工厂确认收样
- 待工厂提交样衣审核
- 待平台审核样衣
- 样衣审核退回
- 样衣审核拒绝
- 样衣审核通过待转正式

只有 `已转正式合作` 才能进入业务页面。
