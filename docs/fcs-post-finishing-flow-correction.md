# 后道工厂流程纠偏说明

本文档记录后道工厂与车缝工厂协同后的页面原型口径。所有节点均为前端原型 mock 串联，不引入后端、数据库或接口服务。

## 流程图一：专门后道工厂完整流程

生产单
-> 后道单
-> 接收领料
-> 质检
-> 后道
-> 复检
-> 后道交出仓
-> 交出
-> 接收方回写
-> 完成

## 流程图二：车缝厂已做后道后的后道工厂流程

车缝任务
-> 车缝完成
-> 车缝厂完成后道
-> 交给后道工厂
-> 后道工厂接收领料
-> 后道工厂质检
-> 后道工厂复检
-> 后道交出仓
-> 交出
-> 接收方回写
-> 完成

## 流程图三：车缝工厂同时做车缝和后道

车缝工厂接单
-> 执行车缝
-> 车缝完成
-> 执行后道
-> 后道完成
-> 交给后道工厂
-> 后道工厂接收领料

## 流程图四：Web 端与移动端数据串联

Web 端后道单
-> 工厂端移动应用接收领料
-> 工厂端移动应用质检
-> 工厂端移动应用后道或跳过后道
-> 工厂端移动应用复检
-> Web 端后道交出仓
-> 工厂端移动应用交出
-> Web 端交出记录和统计同步

## 状态机一：专门后道工厂完整流程

待接收领料
-> 接收中
-> 已接收
-> 待质检
-> 质检中
-> 质检完成
-> 待后道
-> 后道中
-> 后道完成
-> 待复检
-> 复检中
-> 复检完成
-> 待交出
-> 已交出
-> 已回写
-> 已完成

异常流转：

接收中
-> 接收差异
-> 平台处理中
-> 已接收或重新接收

质检中
-> 质检异常
-> 平台处理中
-> 待后道或关闭

复检中
-> 复检差异
-> 平台处理中
-> 待交出或重新复检

## 状态机二：车缝厂已做后道后的后道工厂流程

待接收领料
-> 接收中
-> 已接收
-> 待质检
-> 质检中
-> 质检完成
-> 待复检
-> 复检中
-> 复检完成
-> 待交出
-> 已交出
-> 已回写
-> 已完成

说明：此流程中，后道工厂不再执行后道动作，页面展示“后道已由车缝厂完成”。

## 状态机三：车缝工厂同时做车缝和后道

待车缝
-> 车缝中
-> 车缝完成
-> 待后道
-> 后道中
-> 后道完成
-> 待交后道工厂
-> 已交后道工厂
-> 后道工厂待接收领料

说明：质检和复检由后道工厂执行；车缝工厂只完成车缝和本厂后道。
