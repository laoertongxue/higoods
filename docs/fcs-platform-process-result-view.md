# FCS 第 9 步：平台侧同步看结果

## 流程图

Web端或移动端操作
-> 写回统一事实源
-> 生成仓记录交出记录审核记录差异记录
-> 平台结果视图读取统一事实源
-> 派生平台聚合状态
-> 派生风险提示
-> 派生下一步跟单动作
-> 工厂生产协同系统展示结果

## 状态机

待同步
-> 读取统一事实源
-> 派生平台聚合状态
-> 派生风险提示
-> 派生跟单动作
-> 平台侧已展示

异常流转：

读取统一事实源
-> 发现差异记录
-> 平台状态变为异常
-> 生成处理差异动作

## 三端分层

1. 工艺工厂运营系统展示细状态，例如印花打印、转印、染色排缸、包装等现场节点。
2. 工厂端移动应用展示现场执行状态和可执行动作。
3. 工厂生产协同系统展示平台聚合状态、风险提示和下一步跟单动作。
4. 平台侧不展示工厂内部全部细节点，细状态只作为“工厂内部状态”辅助字段。
5. 平台侧通过 `platform-process-result-view` 读取统一事实源，能看到最新待加工仓、待交出仓、交出记录、回写记录、审核记录、差异记录。
6. 平台侧结果必须能追溯到工艺详情和移动端任务，并保留查看待交出仓、查看交出记录、查看差异记录等跟单入口。

## 跟单动作口径

- 待送货：跟进工厂交出。
- 待回写：跟进接收方回写。
- 待审核：处理审核。
- 异常：处理差异或要求重新交出。
- 已完成：查看详情或查看交出记录。

平台结果视图不新增真实后端、不新增数据库，只在原型内从统一 mock 事实源派生展示结果。
