# 工厂入驻口径收口第 2 步：审核结果统一为已通过 / 未通过

## 背景

本次收口把平台初审和平台样衣审核的结果统一为两个业务口径：`已通过`、`未通过`。原型页面不再向用户展示“允许再次申请”“不允许再次申请”“允许再次提交”“不允许再次提交”等分支，避免审核语义和后续补充动作混在一起。

## 平台初审规则

平台初审结果只有：

- 已通过：入驻申请进入 `待样衣验证`，等待平台登记并发放样衣。
- 未通过：入驻申请进入 `平台审核退回`，工厂可登录、查看审核意见、补充资料并重新提交入驻申请。

平台初审未通过不锁定账号，不禁止同名工厂再次发起入驻。旧数据里的拒绝状态仅作为历史记录展示，新流程不再产生 `平台审核拒绝`。

## 样衣审核规则

平台样衣审核结果只有：

- 已通过：入驻申请进入 `样衣审核通过待转正式`，等待平台执行正式转档。
- 未通过：入驻申请进入 `样衣审核退回`，工厂可登录、查看样衣审核意见和需重新提交内容，并重新提交样衣审核资料。

样衣审核未通过不锁定账号，不禁止同名工厂再次发起入驻。旧数据里的拒绝状态仅作为历史记录展示，新流程不再产生 `样衣审核拒绝`。

## 旧结果兼容

系统保留旧结果归一化能力：

- `通过` 归一为 `已通过`
- `不通过且允许再次申请` 归一为 `未通过`
- `不通过且不允许再次申请` 归一为 `未通过`
- `不通过且允许再次提交` 归一为 `未通过`
- `不通过且不允许再次提交` 归一为 `未通过`

页面、弹窗、筛选、详情、检查脚本和 Playwright 均使用 `已通过 / 未通过`。

## 不变范围

本次不改样衣提交字段，不改 PPIC 逻辑，不改正式转档逻辑，不恢复 `/fcs/pda/login`，也不新增兼容跳转。
