1. 业务背景与设计目标

1.1 背景

印尼供应链团队（含生产团队）中分出一个专门的小组——配料小组，统一负责所有物料的配料操作。 配料小组按照生产单 BOM（物料清单），从各个仓库中挑出物料、拣货、放入暂存区、确认配料完成，然后通知下游来领料或配送。

配料完成后，系统再执行任务分配（即确定由哪个工厂承接该工序），这与当前"先分配再配料"的代码逻辑完全相反。

1.2 设计目标

  1. 在 FCS 下建立独立的配料管理菜单，取代当前 WLS 下的入口
  2. 按配料对象拆分为 5 个独立视图 + 1 个综合看板：染色配料、印花配料、裁片配料、车缝配料、其他配料、配料列表
  3. 实现「先配料、再分配」的核心业务顺序
  4. 引入完整的配料生命周期：创建 → 拣货 → 入暂存区 → 确认完成 → 通知下游
  5. 车缝配料受裁片放行数量约束

3. 五大配料类型与分类规则

3.1 分类定义

一个生产单的所有物料，按物料类型（面料/辅料/纱线/包材）和上游工序（是否需要印花/染色）自动路由到对应的配料类型。 分类规则以当前代码 buildDefaultTaskLinks() 为基准。

配料类型物料筛选条件配货仓库下一步动作工厂确定性
染色配料 面料 + upstreamSourceType='染色' 染色厂待加工仓 加工领料 工艺绑定
印花配料 面料 + upstreamSourceType='印花' 印花厂待加工仓 加工领料 工艺绑定
裁片配料 面料 + upstreamSourceType='中转仓库存' 中转仓 裁床领料 工艺绑定
车缝配料 辅料 / 纱线 辅料仓 / 纱线仓 配送至车缝厂 分配决定
其他配料 包材 包材仓 配送至承接厂 分配决定
工厂确定性说明： 染色/印花/裁片是工艺绑定的，面料只能到对应类型的工厂加工，所以配料时就能确定目标工厂类型。 车缝/其他配料的目标工厂依赖于任务分配结果，配料时需要标记为"待分配"，不预先确定配送目的地。

3.2 关于三方裁片仓

在实际业务中，一些工厂具有复合生产能力。例如：

  • 仅做裁片：工厂只承接裁片工序，裁好的片交给其他工厂做车缝
  • 裁片 + 车缝：工厂既做裁片也做车缝，裁完直接缝，无需跨厂移交
  • 裁片 + 车缝 + 后道：工厂做全流程，从裁片到包装都在同一厂完成

这对于裁片配料的影响是：裁片配料在中转仓完成并确认后，下游的领料工厂可能是一个纯裁片厂， 也可能是一个裁片+车缝一体厂。但无论该工厂后续是否兼做车缝，裁片配料阶段的操作不发生变化—— 配料小组只管把成品面料从中转仓配好，交给裁片工厂领走。其余的辅料（车缝配料）仍然由配料小组在辅料仓/纱线仓独立操作。

对车缝配料的影响是：车缝配料完成后配送的目的地，既可能是独立车缝厂，也可能是裁片+车缝一体厂（即同一工厂既领了裁片面料，又接收辅料）。这在任务分配阶段确定。

4. 核心业务流程

4.1 总体流程

4.2 染色配料详细流程

  1. 面料调拨到待加工仓：采购入库 → 面料仓 → 调拨 → 染色厂待加工仓（WAIT_PROCESS）
  2. 配料小组创建配料记录：按生产单 BOM 计算面料需求量，创建 DRAFT 记录
  3. 仓库拣货：仓库操作员从染色厂待加工仓货架取出面料实物，状态变为 PICKED
  4. 跟单入暂存区：跟单核实后放入染色厂暂存区，状态变为 STAGED
  5. 配料确认完成：配料小组确认 CONFIRMED → 触发染色厂加工领料通知
  6. 染色厂加工领料：染色工厂操作员领取面料，进行染色生产
  7. 加工完成回中转仓：染色完成的成品面料回转中转仓，等待进入裁片配料环节
印花配料流程与染色配料完全对称，只是目标仓库为印花厂待加工仓。染色/印花完成后成品面料均回转中转仓。

4.3 裁片配料详细流程

  1. 面料已在中转仓就位：印花/染色加工完成的成品面料已经回转到中转仓；不需要印染的面料由面料仓直接调拨到中转仓
  2. 配料小组创建配料记录：按生产单 BOM 计算裁片所需面料量，创建 DRAFT 记录
  3. 仓库拣货：仓库操作员从中转仓货架取出成品面料，PICKED
  4. 跟单入暂存区：放入中转仓暂存区，STAGED
  5. 配料确认完成：CONFIRMED → 通知裁片工厂可领料
  6. 裁片工厂领料：裁片工厂操作员到中转仓领料，入裁片工厂待加工仓

4.4 车缝配料详细流程

  1. 裁床做裁片放行判断：裁床主管通过裁片放行管理判断能做多少件
  2. 计算配料约束：车缝配料最大数量 = min(生产单需求, 放行件数 × BOM单件用量, 库存可用量)
  3. 配料小组创建配料记录：按约束值创建 DRAFT 记录（标记 taskLinks 为"未分配"）
  4. 仓库拣货 + 入暂存区：PICKED → STAGED → CONFIRMED
  5. 等待任务分配：配料完成后等待车缝任务分配确定具体工厂
  6. 配送：任务分配完成后，按工厂分别配送到对应车缝厂
其他配料（包材）流程与车缝配料类似。包材的目标工厂可能是后道厂，也可能是车缝厂（如果该车缝厂有包装产线），需等任务分配确定。

7. 配料数量计算规则

7.1 核心公式

染色 / 印花 / 裁片配料

最大可配 = min(生产单BOM需求量, 仓库可用库存)

默认配料 = 最大可配

实际配料 ≤ 最大可配（由配料操作员填写）

不受裁片放行约束

车缝配料（辅料配料）

最大可配 = min(生产单BOM需求量, 裁片放行件数 × BOM单件用量, 仓库可用库存)

默认配料 = 最大可配

实际配料 ≤ 最大可配

受裁片放行数量 + 库存数量双重约束

其他配料（包材配料）

最大可配 = min(生产单BOM需求量, 仓库可用库存)

默认配料 = 最大可配

不受裁片放行约束

7.2 车缝配料计算示例

字段数值说明
生产单需求量500 条拉链500件 × 1条/件
裁片放行件数320 件裁床主管判断"部分可以做 320件"
仓库可用库存300 条辅料仓当前库存
最大可配min(500, 320×1, 300) = 300 条受库存限制
默认配料300 条等于最大可配
备注剩余 20 件缺辅料（放行320但库存只够300），等拉链到货再补配

字段数值说明

裁片放行件数320 件
仓库可用库存800 条充足

8. 暂存区与配料完成确认

8.1 暂存区概念

暂存区是仓库内的一个指定物理区域。每个配料仓库都有一个或多个暂存区。 配料完成后，货被放在暂存区等待下游来取或配送。

中转仓的特殊性：中转仓接收的是印花/染色加工完成的成品面料。 裁片配料的物料正是从这些成品面料中选取。暂存区位于中转仓内，方便裁片工厂来领。

配料类型配货仓库暂存区位置通知对象
染色配料染色厂待加工仓染色厂暂存区染色厂（加工领料）
印花配料印花厂待加工仓印花厂暂存区印花厂（加工领料）
裁片配料中转仓（印染成品面料存放处）中转仓暂存区裁片工厂
车缝配料辅料仓 / 纱线仓辅料仓暂存区 / 纱线仓暂存区待分配后确定
其他配料包材仓包材仓暂存区待分配后确定

8.3 确认规则

  • 确认对象是整条配料记录（非逐行确认）
  • 只有 STAGED 状态可确认
  • 确认后该配料记录进入 CONFIRMED，下游立即可见（领料列表）
  • 确认时生成 "配料完成通知" 运行时事件
  • 确认后配料记录不可修改，只能由下游打回后重新确认

10. 数据模型设计

10.0 配料单与配料记录的关系 —— 操作核心是配料记录

在业务层面，配料单（MaterialPrepOrder）是一张生产单的配料容器， 而真正的操作对象是 配料记录（MaterialPrepRecord）。

  • 一个生产单 → 按物料路由到各配料类型，每个类型产生一张配料单
  • 一张配料单 → 包含多个物料行（MaterialPrepLine）
  • 一个物料行 → 可以有多条配料记录（分批次配料）
  • 一条配料记录 → 可以覆盖多个物料明细（记录内统一拣货、入暂存区、确认）

举例：裁片配料单有 3 条面料物料行（主身面料、里布、拼接面料）。配料小组分两批操作：

批次配料记录覆盖物料状态
第一批配料记录 #1主身面料 200 yard + 里布 150 yardDRAFT → PICKED → STAGED → CONFIRMED
第二批配料记录 #2拼接面料 80 yard（等印花完才可配）DRAFT → PICKED → STAGED → CONFIRMED

每一条配料记录独立走完自己的生命周期（DRAFT → PICKED → STAGED → CONFIRMED）。 配料单的状态由其下属所有配料记录和物料行的完成度派生。

配料列表（综合看板）

聚合展示所有配料单，按配料类型和配料单状态多维度 Tab 筛选。供管理者全局掌握配料进度。

  • Tab：全部 / 待配料 / 配料中 / 已配齐 / 已关闭
  • 筛选：配料类型、生产单号、交期范围
  • 每行展示：生产单号、款式、配料类型、配料进度、状态
  • 点击行进入对应类型的配料详情

染色配料 / 印花配料

面料仓 → 染色/印花厂待加工仓 的配料操作。下游下一步是加工领料。

  • 配料单列表（按状态 Tab）
  • 配料详情（生产需求信息 / 库存与调拨进度 / 配料记录 / 拣货状态 / 暂存区）
  • 新增配料记录弹窗
  • 拣货确认 / 入暂存区确认 / 配料完成确认
  • 加工领料通知

裁片配料

面料仓 → 中转仓 的配料操作。下游下一步是裁床领料。与当前 WLS 配料管理逻辑最接近。

  • 配料单列表（按状态 Tab）
  • 配料详情（5 个 Tab：需求 / 库存 / 任务 / 记录 / 领料）
  • 新增配料记录弹窗
  • 确认配料记录（DRAFT → PICKED → STAGED → CONFIRMED）
  • 裁床领料通知

车缝配料

辅料仓 / 纱线仓 → 车缝厂 的配料操作。受裁片放行数量约束，配料完成时工厂待分配。

  • 配料单列表（按状态 Tab）
  • 裁片放行约束卡片：展示裁床判断结果、放行件数、最大可配
  • 配料详情（需求 / 库存 / 放行约束 / 记录 / 暂存区）
  • 新增配料记录（maxPrepQty = min(需求, 放行×用量, 库存)）
  • 确认配料完成（CONFIRMED 后进入"待分配"）
  • 任务分配完成后回写工厂，按工厂拆分配送

其他配料

包材仓 → 后道厂或车缝厂 的配料操作。同车缝配料，工厂待分配。

  • 配料单列表
  • 配料详情 + 新增记录 + 确认流程
  • 确认后进入"待分配"，分配后回写工厂

12. 设计决策与差异分析

12.1 关键设计决策

序号决策原因
1 配料管理从 WLS 移至 FCS 配料小组统一在工厂生产协同系统内操作，不再分属仓储物流系统
2 先配料再分配 配料只跟 BOM 走，不绑定工厂；分配时才确定工厂并触发配送
3 按配料对象拆分为 5 个独立视图 不同配料类型的仓库、约束、下游动作不同，独立视图减少认知负担
4 新增 PICKED 和 STAGED 状态 区分"创建"与"真正可领"之间的拣货、核实、入暂存区步骤
5 车缝配料引入裁片放行约束 辅料用量直接跟"能做多少件"正相关，裁床没放行的件数配了也是浪费
6 面料仓作为独立的物料入口仓库；中转仓是印染加工后的成品面料存放地 面料先进面料仓，再调拨到印染待加工仓；印染完成后成品面料回中转仓，裁片配料从中转仓取料
7 三方裁片仓可兼做多工序 裁片工厂可能是纯裁片、裁片+车缝、或裁片+车缝+后道，配料操作不受影响，但配送目的地由分配确定
