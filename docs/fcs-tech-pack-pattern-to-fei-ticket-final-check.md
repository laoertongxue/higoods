# 技术包纸样到菲票全链路最终验收

## 验收范围

本次验收覆盖第 1-7 步的最终链路：工序工艺字典、BOM 缩水/洗水联动、纸样两步维护、捆条维护、重复纸样校验、DXF/RUL 裁片明细颜色片数、裁片实例、逐片特殊工艺、逐片菲票和特殊工艺任务消费菲票。

流程图：

维护工序工艺字典
-> BOM选择缩水洗水需求
-> 自动生成准备阶段工序
-> 纸样两步维护
-> 版师上传PRJ唛架图片DXF和RUL
-> 维护捆条
-> 校验重复纸样
-> 解析裁片明细
-> 维护颜色片数
-> 生成裁片实例
-> 逐片配置特殊工艺和位置
-> 生成逐片菲票
-> 菲票携带特殊工艺
-> 特殊工艺任务读取菲票
-> 全链路检查脚本和Playwright验证

状态机：

待验证
-> 前置脚本校验
-> 字典校验
-> BOM联动校验
-> 纸样维护校验

-> 菲票校验

异常流转：

任一校验失败
-> 定位失败模块
-> 修复遗漏
-> 重新执行全部校验
-> 最终通过

## 字典验收

工序工艺字典必须包含准备阶段“缩水”“洗水”。特殊工艺分为“辅助工艺”和“特种工艺”，其中捆条适用对象为面料，橡筋定长切割适用对象为辅料。逐片特殊工艺选择器只能读取裁片部位工艺，不允许出现捆条、橡筋定长切割、缩水、洗水。

## BOM 联动验收

物料清单保留“缩水需求”“洗水需求”，取值为“是 / 否”。任一物料行为“是”时自动生成准备阶段工序；全部取消时只自动移除无人工维护内容的自动工序，有人工内容时必须等待用户确认。

## 纸样维护验收

纸样管理按“跟单基础信息 / 版师技术信息”两步维护。跟单维护纸样名称、纸样类型、是否针织、门幅和关联物料；版师维护排料长度、纸样 PRJ 文件、唛架图片、DXF、RUL、捆条和裁片明细。旧图片字段统一在页面上展示为“唛架图片”。

## 捆条与重复校验

捆条跟随纸样维护，一个纸样可维护多个捆条，每个捆条必须有名称、长度（cm）和宽度（cm）。重复纸样校验区分明确重复与疑似重复：同名、PRJ、DXF、RUL 重复阻止保存；关联物料、唛架图片、解析结构相似进入疑似重复确认。

## 裁片明细验收

DXF/RUL 解析结果落到裁片明细后，主表不再维护可编辑片数列。裁片片数按适用颜色和颜色片数维护，单行总片数由启用颜色片数合计，纸样当前总片数由全部裁片明细行合计。

## 裁片实例与逐片工艺验收

系统根据每行裁片明细的适用颜色和颜色片数生成裁片实例。特殊工艺挂到裁片实例上，而不是只挂到部位行。每个特殊工艺必须选择一个位置：左、右、底、面。同一裁片实例不允许重复添加同一特殊工艺。

## 菲票验收

菲票按裁片实例逐片生成，一个裁片实例对应一个有效菲票。菲票必须包含 sourcePieceInstanceId、originalCutPieceOrderId、特殊工艺和工艺位置。菲票归属原始裁片单，合并裁剪批次只作为执行上下文。

## 特殊工艺任务消费验收

特殊工艺任务生成或流转读取菲票上的结构化 specialCrafts。系统可以按 craftCode 聚合菲票，并展示菲票数量和工艺位置。差异记录通过 relatedFeiTicketIds 追溯菲票。

## 版本快照验收

技术包版本快照必须保留 BOM 缩水/洗水需求、准备阶段自动工序、纸样两步字段、PRJ、唛架图片、捆条、重复确认信息、颜色片数、裁片实例和逐片特殊工艺配置。

## 裁片冻结口径

菲票永远回落原始裁片单。合并裁剪批次只记录为执行上下文，不作为菲票归属主体。一个菲票不对应多个裁片实例，一个裁片实例不生成多个有效菲票。

## 检查脚本

最终总检查脚本为：

- `scripts/check-tech-pack-pattern-to-fei-ticket-final.ts`

该脚本会先执行第 1-7 步检查脚本，再对字典、BOM、纸样、捆条、重复校验、裁片明细、裁片实例、菲票、特殊工艺任务消费和版本快照做关系校验。

## Playwright 覆盖

最终 Playwright 用例为：

- `tests/tech-pack-pattern-to-fei-ticket-final.spec.ts`

覆盖路径包括字典与 BOM 联动、纸样两步维护、捆条、重复纸样校验、颜色片数、逐片特殊工艺、逐片菲票、二维码追溯、打印预览和特殊工艺任务消费菲票。
