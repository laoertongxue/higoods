# 工厂入驻 Step5：平台样衣审核闭环

## 业务定位

平台样衣审核发生在工厂提交样衣照片、视频、工艺说明之后，用来判断工厂是否具备进入正式合作前的样衣验证条件。它仍属于入驻验证阶段，不等于正式合作。

## 样衣审核三种结果

1. 通过：入驻申请进入 `样衣审核通过待转正式`，样衣验证对象进入 `样衣审核通过`。
2. 不通过且允许再次提交：入驻申请进入 `样衣审核退回`，工厂端可重新上传样衣照片、样衣视频、工艺说明、问题说明和补充文件。
3. 不通过且不允许再次提交：入驻申请进入 `样衣审核拒绝`，账号锁定，同名工厂/公司名称不能再次发起入驻。

## 为什么通过后只是待转正式

样衣审核通过只说明样衣验证完成，还需要后续正式转档动作才能生成工厂档案、转正式管理员账号并开放业务权限。因此本步骤只进入 `样衣审核通过待转正式`，不直接进入 `已转正式合作`。

## 退回后重新提交规则

平台选择 `不通过且允许再次提交` 时必须填写样衣审核意见，并选择需重新提交内容。工厂端展示上次样衣审核结果、审核意见和需重新提交内容，重新提交后状态回到 `待平台审核样衣`，`factorySubmissionRoundNo` 自动递增。

## 拒绝后锁定规则

平台选择 `不通过且不允许再次提交` 时，系统将：

- `accountLocked = true`
- `accountLockedReason = 样衣审核意见`
- `factoryNameLocked = true`
- 登录提示：`该账号已被锁定，不能继续登录。`
- 同名工厂/公司名称再次入驻提示：`该工厂入驻申请已被拒绝，不能再次发起入驻。`

## 样衣审核记录字段

`FactorySampleReviewRecord` 记录：

- 样衣审核记录 ID：`sampleReviewId`
- 样衣审核轮次：`sampleReviewRoundNo`
- 审核结果：`sampleReviewResult`
- 样衣审核意见：`sampleReviewOpinion`
- 是否允许再次提交：`allowResubmit`
- 需重新提交内容：`requiredResubmitItems`
- 审核人：`reviewer`
- 审核时间：`reviewedAt`
- 状态和节点流转：`fromStatus`、`toStatus`、`fromNode`、`toNode`
- 对应工厂提交轮次：`relatedSubmissionRoundNo`
- 样衣质量结论：`sampleQualityConclusion`
- 能力验证结论：`capacityConclusion`
- 备注：`remark`

## 本次明确不做

本次不正式转档，不生成工厂档案，不转正式管理员账号，不开放执行、交接、仓管、结算。业务页仍只有 `已转正式合作` 可进入。

## 中文流程图

```text
工厂提交样衣审核资料
-> 平台样衣审核
-> 审核通过
-> 样衣审核通过待转正式
-> 等待下一步正式转档

工厂提交样衣审核资料
-> 平台样衣审核
-> 不通过且允许再次提交
-> 样衣审核退回
-> 工厂重新提交样衣审核资料
-> 待平台审核样衣

工厂提交样衣审核资料
-> 平台样衣审核
-> 不通过且不允许再次提交
-> 样衣审核拒绝
-> 锁定账号
-> 禁止同名工厂再次入驻
```
