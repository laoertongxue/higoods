| 任务清晰度 | 通过 | 烫画、直喷统一展示“已裁部位、成衣”，不再使用“成衣半成品”。 |
| 信息架构与导航 | 通过 | 不新增页面和菜单，在现有工序工艺维护中收口。 |

| 信息负荷 | 通过 | 仅收口对象名称和必要的顺序阻断提示。 |
| 文案 | 通过 | 阻断提示明确说明必须先染色、后印花及调整方法。 |
| 数量与状态 | 通过 | 裁片按生产数量乘每件片数生成，单位为片；成衣仅汇总成衣 BOM 适用 SKU 的生产数量，单位为件。 |
| 扫码与识别 | 通过 | 已裁部位任务必须绑定并扫描真实菲票；成衣任务不进入菲票链路。 |
| 防错 | 通过 | 拖动、保存和确认时均阻断同一 BOM 物料先印后染；成衣工艺缺少关联成衣 BOM 时阻断任务生成。 |
| UI 样式 | 通过 | 未改变页面布局和视觉层级。 |
| 组件交互 | 通过 | 沿用现有操作入口，不引入新交互框架。 |
| 协作关系 | 通过 | 已确认路线不会向下游传递先印后染的错误顺序；直喷、烫画按各自工艺和本次作用对象生成独立任务数据。 |
| 异常与追溯 | 通过 | 成衣明细保存来源 BOM 行，只生成适用 SKU，不生成纸样、裁片或菲票占位；裁片保留真实纸样、部位和菲票链路。 |
| 现场设备可用性 | 不适用 | 本次未修改员工执行端。 |

## 5. 主要问题与处理

| 问题 | 标签 | 影响角色 | 处理方式 | 是否仍有风险 |
| --- | --- | --- | --- | --- |
| 直喷、烫画作用对象口径不一致 | 选不对 | 技术包维护人 | 两项工艺均限定为“已裁部位、成衣” | 否 |
| 人工排序可形成先印后染 | 点错风险 | 工艺负责人 | 同物料拖动、保存、确认均阻断并提示调整 | 否 |
| 单据定义清理后生产来源印染加工单被漏生成 | 协作断裂 | 生产计划员 | 印花、染色加工单生成器读取 `TASK` 定义，并以统一加工单回归检查覆盖 | 否 |
| 旧快照的“成衣半成品”向下游泄漏 | 选不对 | 技术包维护人、特殊工艺工厂 | 仅在读取边界兼容旧值，生成、保存及仓储流转统一输出“成衣” | 否 |
| 成衣辅助工艺沿用裁片数量、纸样或菲票 | 算不准 | 生产计划员、辅助工艺仓管 | 成衣只取关联 BOM 适用 SKU 数量，单位固定为件，纸样、裁片和菲票字段保持为空 | 否 |
| 成衣任务无法回溯冻结 BOM | 追溯不足 | 生产计划员、工艺负责人 | 明细保存 `sourceBomItemId`；缺少关联成衣 BOM 时按统一错误模型阻断 | 否 |
| 纸样包与物料关联记录重复携带同一裁片明细 | 算不准 | 生产计划员、辅助工艺仓管 | 快照存在物理纸样层时只消费该层；旧快照仅有物料关联记录时保留原数据，不按名称、部位或颜色模糊合并 | 否 |

## 6. 最终结论

结论：通过

说明：本次收口技术包对象规则和辅助工艺任务生成，不增加加工单前置锁、依赖 ID 或自动解锁。直喷、烫画均覆盖已裁部位与成衣；成衣采用件数且不依赖纸样、裁片或菲票，已裁部位必须保留真实纸样、部位和菲票链路。纸样包与其物料关联记录不得重复放大计划数量，不同纸样包仍保留独立身份。本任务未提前实现成衣仓至后道工厂的完整交接页面，该链路由后续任务单独验收。
