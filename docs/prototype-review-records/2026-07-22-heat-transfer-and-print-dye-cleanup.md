## 6. 最终结论

结论：有条件通过

说明：

- 烫画与直喷的菜单、动态路由、标题、加工单业务类型、加工单号和筛选状态已隔离。
- 两类加工单均保留裁片部位与成衣演示记录。
- 成衣链路已按“成衣仓通过 Web / PDA 逐 SKU 出库 → 辅助工艺待收货 → 逐 SKU 收货入待加工仓 → 加工 → 逐 SKU 待交出 → 我方后道逐 SKU 收货”实现；未出库和仅出库未收货均不会形成待加工库存。
- 成衣仓出库、辅助工艺收货、完工和后道收货均走统一动作或领域服务；仓联动失败不会先改工单状态，已收货状态不能重复执行。
- 本次样例将计划 500 件的一个 SKU 从成衣仓实出 498 件，PDA 应收和默认实收均为 498；加工环节对指定 SKU 报废 1 件、后道环节对另一个指定 SKU 少收 1 件，差异均落到实际 SKU，不从总数顺序猜测。
- 契约通过真实 PDA 页面渲染和按钮 handler 验证公开出库入口、无权限拒绝、重复操作拒绝、部分出库默认收货成功，以及烫画 / 直喷同 SKU 草稿隔离。
- 加工执行逐 SKU 保存已收、完工、报废和货损；后道使用用户提交的逐 SKU 实收映射生成一条与生产单、烫画加工单关联的既有后道任务。
- 成衣链路不生成菲票、不展示部位、不回裁床；裁片链路仍按片流转并保留已有菲票事实。
- 例外仅为既有公共动态加工单列表尚未迁移标准列表组件；本次不复制页面、不修改基线或治理脚本来规避检查。
