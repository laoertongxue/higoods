## 5. 主要问题与处理

| 问题 | 标签 | 影响角色 | 处理方式 | 是否仍有风险 |
| --- | --- | --- | --- | --- |
| 原台账为手写宽表，缺少列治理 | 字段过载、组件误用 | 跟单、管理人员 | 接入标准列表表格、分页和列设置；允许隐藏非必需列。 | 否 |
| 宽表操作列可能随横向滚动离开视口 | 点错风险 | 全部管理端角色 | 操作列声明为固定右侧必需列。 | 否 |
| 标准化可能删减原业务入口 | 追溯不足 | 跟单、责任团队 | 保留查看详情、确认工作项、独立准备项操作、产出和物料维护入口。 | 否 |
| 列设置可能触发整页重绘 | 组件误用 | 全部管理端角色 | 表格、分页和列设置使用局部 DOM 区域更新。 | Task 6 继续补完整响应耗时矩阵 |
| 页码和排序跨刷新或重新进入保留 | 点错风险 | 跟单、管理人员 | 页码和排序仅保留在当前局部交互；刷新、筛选或 SPA 重新进入时统一回到第 1 页且未排序。 | 否 |
| 同路由业务弹窗导致台账上下文丢失 | 点错风险 | 跟单、责任团队 | 路由进入生命周期与页面渲染分离；查看详情、准备项操作和上传后的重渲染保留当前页与排序。 | 否 |
| 列设置局部渲染后图标不可见 | 组件误用 | 全部管理端角色 | 每次局部更新后只对新插入区域执行图标 hydration，不扫描整页。 | 否 |
| 列设置抽屉在 SPA 重进时自动重开 | 点错风险 | 跟单、管理人员 | 路由重新进入时统一清理页码、排序、抽屉和拖拽瞬时状态。 | 否 |
| 月度统计和明细统计为手写表格，列偏好互相影响 | 字段过载、组件误用 | 跟单、管理人员 | 两个 Tab 分别使用标准表格、标准分页和独立存储键；各自持久化列显示、顺序、冻结和每页条数。 | 否 |
| 统计分页和排序写入 URL 后会在刷新时错误恢复 | 点错风险 | 跟单、管理人员 | 页码和排序只保留在当前 Tab 的内存状态；刷新、Tab 切换和 SPA 重进均重置。 | 否 |
| 月份进入明细时可能丢失多选筛选或传播废弃参数 | 追溯不足 | 管理人员 | 明细链接保留合法重复筛选值，只覆盖 `tab=detail` 和 `detailPage=1`，不传播废弃参数及准备项进度。 | 否 |
| 统计宽表可能造成页面级横向溢出 | 字段过载 | 管理人员 | 页面主体保持无横向溢出，月度和明细宽表只在标准表格容器内滚动。 | 否 |
| 统计处理器响应无关输入时重复聚合 | 组件误用 | 跟单、管理人员 | 事件处理先按路由和统计列表前缀快速返回；性能验收通过单次 Event 完成探针等待异步分发、宏任务和下一帧，关键词输入和台账无关事件不构建统计上下文、不刷新列表且响应小于 200ms。 | 否 |
| 台账处理器吞掉统计列拖拽结束事件 | 组件误用 | 管理人员 | 台账和统计处理器分别校验所属路由及列前缀后再清理拖拽状态。 | 否 |
| CSV 仅验证下载入口，未验证完整筛选结果和标准转义 | 追溯不足 | 管理人员 | 浏览器解码实际导出 URI，核对表头及全量筛选行数；专项纯函数覆盖逗号、引号和换行转义。 | 否 |
