## 6. 对抗式反例验收

| 方向 | 构造反例 | 结果与防错 |
| --- | --- | --- |
| 数据完整性 | 未确认工作项；普通项状态已完成但无完整上传；辅料项无采购单号、任一采购单缺下单时间或完成时间不等于最晚下单时间；依赖项无有效凭证；Mock 状态与确认逻辑冲突 | 均被专项检查拦截。完成项统一通过数据层有效凭证函数；普通项要求完整上传，辅料项不要求上传但要求每张采购单号及逐单下单时间完整，实际完成时间取最大下单时间；每条 Mock 均校验确认、依赖、状态和产出一致性。 |
| 页面可见性 | 准备项与责任团队选择不兼容；筛选无结果；旧筛选字段残留；标题被包入列表卡片 | 已选值即使不兼容仍保持勾选且可见，并展示标准空态；筛选区不出现责任人、是否超时、买手；页面文案为中文；标题位于标准列表页头部而非列表卡片。 |
| 交互可达性 | 台账五类多选缺项；统计四类多选被单值关键词混淆；联动错误取交集；分页、排序、列设置或业务弹窗丢失上下文 | 台账跟单/记录状态/准备项/准备项进度/责任团队均为多选；统计跟单/记录状态/准备项/责任团队为多选，关键词按批准口径保留单值输入且无准备项进度；双向联动按并集；三个列表分页、三态排序、列显示/顺序/冻结/恢复可达；台账操作列固定右侧；业务弹窗保持当前页和排序。 |
| 业务闭环 | 月份进入明细丢重复参数；筛选后 CSV 只导出当前页或转义损坏；准备项/团队/进度跨项命中；无上传已完成进入统计 | 月份和 Tab 链接只保留合法白名单并保留重复多选参数；CSV 在每页 5 条时仍导出准确的 6 条筛选明细，专项覆盖逗号、引号、换行；`PREP-202603-002` 的版衣制作不满足开始条件、但其他准备项已完成，筛选“版衣制作 + 车板团队 + 已完成”时明确排除该记录，证明三类条件在同一准备项匹配；无有效上传的普通完成项不进入月度与明细统计。 |
| 运行态边界 | 无凭证已完成进入 KPI；重新勾选仍为无需；下载重算记录状态；错误记录号与准备项编号串写；已关闭记录生成正式产出；localStorage 暂时不可写 | KPI 只认有效完成凭证；重新勾选恢复待开始；纯下载只记录审计事实；上传下载联合校验记录号和准备项编号；已关闭产出字段保持不变；存储恢复后自动补写会话降级状态并清除缓存。 |
| 对抗篡改 | 同时替换确认项、普通上传和辅料表单的记录号/准备项编号；篡改染色表单上下文；下载按钮同时替换记录号、准备项编号和上传编号 | 所有表单以浏览器当前路由为非表单事实，三方不一致即拒绝写入；下载三元组必须归属于当前路由记录/准备项，不能跨记录追加审计。 |
| 真实产物 | `.txt` 伪装 `image/png`、`.jpg` 伪装 `text/plain`、图片扩展名和 MIME 都正确但文件头为文本；花型只有完成图或只有源文件；Mock 已完成花型缺任一产物 | MIME/扩展名不一致和常见文件签名不符均拒绝；花型完成图与源文件不能互相充当，只有双产物齐备才完成；已完成花型 Mock 同时保留真实图片和 `.ai` 上传记录。 |
| 下载闭环 | 非基码上传记录点击下载；基码下载按钮同时篡改为另一记录上传三元组 | 非基码文件触发真实浏览器下载但不增加基码审计；篡改三元组不会下载或写入，合法已完成基码仍可下载并记录每次下载时间。 |
| 性能终点 | 只改 URL 或仅保留原分页容器，统计表延迟或完全不更新；每页 5 改 8 时仍等待原本就存在的“1 / 2” | 筛选必须等“毛织基码纸样完成数量由 2 变 0”；pageSize 必须等局部分页状态由 5 变 8，均在浏览器真实事件后首个满足条件的动画帧冻结耗时。 |
