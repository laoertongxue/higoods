## 3. 自查结论

| 检查项 | 结论 | 说明 |
| --- | --- | --- |
| 角色匹配 | 通过 | 页面定位为 FCS 管理端，服务跟单、生产计划、生产主管、财务和工艺负责人做变更判断、影响确认和追溯，不下放给一线员工判断技术包版本或补丁概念。 |
| 任务清晰度 | 通过 | 列表页负责查找和发起，新增 / 编辑页负责填写变更意图并承接系统计算，详情页负责查看影响、单据、料工费、时效和审核记录，关系诊断页负责从生产单视角发起变更。 |
| 信息架构与导航 | 通过 | `/changes`、`/changes/new`、`/changes/:changeOrderId`、`/changes/:changeOrderId/edit`、`/changes/orders/:productionOrderId` 分别承接不同对象，避免列表页继续混入详情和关系判断。 |
| 页面模式 | 通过 | 管理端页面使用表格、Tab、摘要卡和处理记录承接较高信息密度；新增 / 编辑按步骤拆分，先让业务说清变了什么，再由系统反推变更类型和影响范围。 |
| 信息负荷 | 通过 | 列表页只保留变更单列表与待处理生产单来源池；详情页按 Tab 展示影响明细，避免首屏堆叠全部单据、成本和时效信息。 |
| 文案 | 通过 | 页面文案使用中文业务表达，操作保持“新增变更”“继续处理”“发起变更”“查看关系”等动作化说法，不展示英文状态码。 |
| 数量与状态 | 通过 | 变更单、生产影响、单据处理、料工费和时效影响均用中文状态、数量单位、金额差异和延误天数表达，支持多次变更和多次补丁的演示数据。 |
| 防错 | 通过 | 旧版本关系变更和生产补丁入口提交后进入变更单详情；生产补丁草稿只能落到草稿等可继续处理状态，避免原型直接制造完成、驳回等终态。 |
| UI 样式 | 通过 | 保持现有 Tailwind 字符串模板后台风格，状态色克制，主要动作与辅助动作层级清楚。 |
| 组件交互 | 通过 | Tab、表格行操作、继续处理、发起变更和查看关系均沿用现有轻交互模式；详情 Tab 限定在固定范围，避免非法状态写入页面。 |
| 协作关系 | 通过 | 页面把跟单发起、工艺判断、生产计划处理、生产主管确认和财务查看成本影响放在同一变更单闭环里，处理记录可证明责任和时间。 |
| 异常与追溯 | 通过 | 未找到对象时提供返回入口；变更单详情展示创建人、创建时间、审核人、影响明细和最后处理记录，支持从变更单回看生产单关系。 |
| 现场设备可用性 | 通过 | 本次页面为管理端，不改变 PDA 或一线执行端；关系诊断和变更详情保留管理端密度，未向现场低分辨率执行页扩散复杂表格。 |

## 4. 问题标签

- `状态抽象`
- `字段过载`
- `点错风险`
- `协作断裂`
- `追溯不足`

## 5. 主要问题与处理

| 问题 | 标签 | 影响角色 | 处理方式 | 是否仍有风险 |
| --- | --- | --- | --- | --- |
| 列表页混入详情和关系诊断，用户进入 `/fcs/production/changes` 后无法区分是在看变更单、生产单关系还是影响台账。 | 字段过载、状态抽象 | 跟单、生产计划、工艺负责人 | 将列表、新增 / 编辑、变更单详情、生产单关系诊断拆到独立路由；列表页只保留变更单列表和待处理来源池。 | 否 |
| 旧动作停留在生产单关系页提示，不进入变更单列表和变更单详情，结果难以追踪。 | 协作断裂、追溯不足 | 跟单、生产计划、财务 | 版本关系变更和生产补丁提交后生成可查询的变更单，并跳转到变更单详情，列表页可继续查看。 | 否 |
| Mock 数据缺少同一生产单多次变更、多次补丁，无法证明页面能表达重复发生的业务场景。 | 追溯不足 | 生产计划、生产主管、财务 | 补充多次变更、多次补丁、草稿、待处理和已闭环等样例，让列表、详情和关系诊断都能展示历史与当前影响。 | 否 |
| 旧弹窗入口只在局部页面临时处理，容易绕开新变更单闭环。 | 点错风险、协作断裂 | 跟单、工艺负责人 | 列表页和变更单详情不再渲染旧弹窗入口；旧弹窗仅保留在生产单关系诊断兼容入口，并在提交后进入新变更单详情。 | 否 |
| 生产补丁草稿状态过宽，原型可能直接创建完成、驳回等不应由草稿入口产生的终态。 | 状态抽象、点错风险 | 跟单、生产计划 | 将补丁草稿收窄为可继续处理的草稿状态，后续审核、驳回、完成必须通过变更单流程表达。 | 否 |
| 新增 / 编辑页保存草稿和提交审核混用新建逻辑，编辑当前变更单时可能生成新单；生产补丁查看通知跳到变更单详情而非生产单关系诊断。 | 点错风险、协作断裂、追溯不足 | 跟单、生产计划、工艺负责人 | 保存草稿显式写入草稿状态；编辑页只允许更新草稿 / 退回修改状态的当前变更单；查看通知固定进入生产单关系诊断 notice Tab。 | 否 |

## 6. 最终结论

结论：通过

说明：

- 本轮页面分层、数据闭环和 Mock 场景已满足原型演示要求，变更意图、影响计算、单据处理、料工费、时效和处理记录能在变更单内追溯。
- 生产补丁定义为生产单层补丁，只覆盖具体生产单的局部例外和生效范围；不修改生产单创建时冻结的技术包快照，也不污染正式技术包版本。
- 本记录只审查 FCS 管理端生产单变更管理页面，不扩展 PDA、一线执行端或真实后端状态机。
