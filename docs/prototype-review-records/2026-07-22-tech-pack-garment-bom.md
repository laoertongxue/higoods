## 3. 自查结论

| 检查项 | 结论 | 说明 |
| --- | --- | --- |
| 角色匹配 | 通过 | 页面继续服务技术资料管理与生产跟单，不新增一线员工判断。 |
| 任务清晰度 | 通过 | 选择“成衣”后只保留适用 SKU、固定数量口径、使用工序和备注。 |
| 信息架构与导航 | 通过 | 沿用技术包原有页签和弹窗，不新增入口。 |
| 页面模式 | 通过 | 沿用管理端编辑弹窗；成衣与普通物料共用 BOM 列表。 |
| 信息负荷 | 通过 | 成衣模式隐藏编码、规格、损耗和印染准备字段。 |
| 文案 | 通过 | 使用“成衣”“件”“单件用量”“适用 SKU”等中文业务口径。 |
| 数量与状态 | 通过 | 单位固定为件、单件用量固定为 1、损耗率固定为 0。 |
| 扫码与识别 | 通过 | 本次为管理端技术资料维护，不涉及现场扫码。 |
| 防错 | 通过 | 未选择至少一个适用 SKU 时按钮不可用，保存时再次阻断；切换类型即清空面料字段。 |
| UI 样式 | 通过 | 复用既有技术包弹窗、表单和按钮样式。 |
| 组件交互 | 通过 | 类型和 SKU 切换只替换 BOM 弹窗，不触发整页重绘。 |
| 协作关系 | 通过 | 成衣仅进入成衣辅助工艺关系；采购、纸样、裁片和印染入口显式排除。 |
| 异常与追溯 | 通过 | 保存后的成衣 BOM 保留适用 SKU、使用工序和备注，技术包版本可追溯。 |
| 现场设备可用性 | 通过 | 未改变现场端页面；管理端弹窗继续支持低分辨率滚动。 |

## 4. 问题标签

- `选不对`
- `算不准`
- `字段过载`
- `协作断裂`

## 5. 主要问题与处理

| 问题 | 标签 | 影响角色 | 处理方式 | 是否仍有风险 |
| --- | --- | --- | --- | --- |
| 成衣沿用面料字段会被误送采购、纸样和印染链路 | `协作断裂`、`字段过载` | 技术资料维护人员、生产跟单 | 成衣类型切换时清空并隐藏面料专属字段；下游入口按 BOM 类型隔离 | 否 |
| 成衣 SKU 为空会造成加工件数范围不确定 | `选不对`、`算不准` | 技术资料维护人员 | 按钮态和保存事件双重校验至少一个适用 SKU | 否 |
| 正式版本发布与旧成衣工艺关联可能绕过校验 | `选不对`、`协作断裂` | 技术资料维护人员、生产跟单 | 所有发布入口统一校验；编辑时清理错误关联，保存前校验全部关联均为成衣 BOM | 否 |
| 成衣排在 BOM 首位时可能被纸样或生产制品兜底误取 | `算不准`、`协作断裂` | 技术资料维护人员、生产跟单 | 纸样、自动款色映射、水溶、印花和染色演示上下文统一只读取非成衣 BOM | 否 |
| 印花、染色真实加工单及准备页仍可能读取首条成衣 | `算不准`、`协作断裂` | 生产跟单、印花厂、染色厂 | 下沉统一非成衣选择器，印花、染色加工单、准备页和技术包快照兜底共同复用 | 否 |

## 6. 最终结论

结论：通过
