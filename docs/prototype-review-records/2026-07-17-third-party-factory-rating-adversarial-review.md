| 任务清晰度 | 通过 | 评级列表、定标候选、改派弹窗均直接展示当前可执行规则结果。 |
| 信息架构与导航 | 通过 | 评级信息集中到三方工厂评级菜单，工厂档案只维护车位数等主档资料。 |

| 数量与状态 | 通过 | 试产质检数量闭合为合格数量加不良数量；不良由返工数量和工厂责任瑕疵原因派生。 |

## 4. 问题标签

- `选不对`

- `协作断裂`

## 5. 主要问题与处理

| 问题 | 标签 | 影响角色 | 处理方式 | 是否仍有风险 |
| --- | --- | --- | --- | --- |
| 部分试产记录的合格数量与不良数量不能闭合到质检数量。 | 算不准 | 工厂运营、财务 | 修正 Mock 数据，并在专项检查中校验 `合格数量 + 不良数量 = 质检数量`。 | 无 |
| 部分评级快照和试产记录的工厂名称与工厂主档不一致。 | 追溯不足 | 工厂运营 | 统一为工厂主档名称，并校验快照、试产记录必须匹配主档编码和名称。 | 无 |
| 旧列表列偏好可能隐藏新增的试产单、试产结论、试产轮次等列。 | 选不对 | 工厂运营 | 升级列偏好版本，并增加旧偏好反例渲染检查。 | 无 |
| 竞价定标候选只展示策略文案，规则确认没有进入定标域函数。 | 点错风险 | 定标员 | 增加运行时定标规则评估函数，候选工厂展示规则结果；黄牌和主管指定必须确认后才能定标。 | 无 |
| 车缝改派可以绕过三方工厂派单规则。 | 协作断裂 | 跟单、工厂运营 | 在改派域函数执行同一套派单规则，并在改派弹窗展示阻断或确认项。 | 无 |
| 车缝分配候选工厂兜底展示仍读取自由文本派单策略。 | 选不对 | 跟单 | 移除自由文本兜底，候选规则结果仅由结构化评估产生。 | 无 |
