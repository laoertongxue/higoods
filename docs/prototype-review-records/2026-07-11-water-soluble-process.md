| 列表与性能 | 通过 | FCS 表格与 PFOS 卡片均严格使用 10 / 20 / 50 分页，伪造负数或 Infinity 回退 10；Playwright 在页面内用 `performance.now()` 对真实 DOM event 到目标局部 DOM 出现显式计时并断言小于 200ms，同时锁定 main/root 身份、connected、焦点和值。 |

## 6. 最终结论

结论：有条件通过

说明：

- 本阶段已完成 PCS 技术包 BOM 与工序预览；草稿只更新技术包工序预览，不生成加工单。
- 水溶按 BOM 物料触发，适用于所有 BOM 物料类型，目标对象不收窄为面料。
- 水溶归准备阶段、归属染厂、排序早于染色，默认生成任务单，不新增需求单、分厂、仓库、结算或批次能力。
- 规格审查发现的公共产值模板缺口已修复；水溶复用既有批次型模板 D，并已补齐专项运行时检查。
- BOM 与技术包工序类型契约已移入独立 `scripts/check-water-soluble-types.ts`，由项目本地 `tsc --noEmit` 真实编译，不再依赖 Node 的类型擦除结果。
- 所有 BOM 物料类型均可维护单位与水溶要求；同一物料同时水溶、染色时固定先水溶后染色。
- 正式技术包快照、运行时克隆和评审差异保留物料编码、单位、水溶要求及工序关联 BOM 行。
- FCS 生产单拆解已按正式快照中的具体 BOM 行生成准备产物：仅水溶生成独立水溶任务，仅染色生成普通染色需求，同一物料水溶加染色只生成带“先水溶、后染色”路线的染色需求，均未选择则不生成。
- 计划数量由系统按适用 SKU 生产数量 × BOM 单耗 ×（1 + 损耗率）计算，最多保留 3 位小数并保留 BOM 单位；产物 ID 包含生产单、正式快照、工序和 BOM 行，重复拆解不累加。
- BOM 产物 ID 已明确纳入正式技术包版本 ID；同一生产单固定 snapshotId 下切换正式版本会生成新身份，同 snapshotId 与同 version id 重复拆解保持幂等，legacy 非 BOM 产物 ID 不变。
- FCS 真实生产入口已锁定 WATER / DYE 资格门槛并对同一 BOM 稳定去重；BOM 产物 ID 使用无歧义分段编码，数量异常会明确阻断，不再静默生成错误单据。
- 数量校验只对当前工序确实会生成的 BOM 产物执行；残留关联的 NONE BOM 保持 0 产物且不影响其他合法产物，组合 BOM 的 WATER 分支也会直接跳过。
- FCS 已建立独立水溶加工单唯一领域来源：仅消费字段完整的 `WATER_SOLUBLE` 且 `TASK` 物料级产物，既包括真实物料级生成产物，也包括携带真实技术包 BOM、系统计算数量与单位的字典覆盖演示产物；演示产物仅服务原型状态覆盖，不等于实际业务单。`sourceDemandIds` 始终为空，不生成水溶需求单；重复读取保持稳定且不累加。
- 加工单已覆盖待分配染厂、待原料、待水溶、水溶中、生产暂停、待交出、交出待收货、收货差异和已完成；非法状态、重复动作、非有限数、非正数与超量交出均以中文原因阻断。
- 工厂分配已接入统一工厂能力：水溶能力归类为染厂能力；只有染色而无水溶能力的工厂不能承接。产能档案可投影水溶能力，使用批次型“批/分钟”口径与通用设备类型。
- 数量不足由系统进入主管处理，支持继续加工、按实际完成量交出、退回返工；交出/收货已接入通用 `WATER_SOLUBLE_WORK_ORDER + MATERIAL` 交接对象并回写同一加工单。
- 独立 `WATER_SOLUBLE + TASK` 物料产物已从整单与连续工序编排源头剥离，只生成独立单工序任务；整单/组合任务同步移除该产物的 `sourceArtifactIds`、覆盖工序和明细来源，其他普通工序与整任务数量保持不变。“染色加工（含水溶）”仍只生成 DYE demand，不受该规则影响。
- 稳定的 `PROC_WATER_SOLUBLE` 领域移动任务已接入 PDA 统一执行任务入口；既有任务列表、详情、索引和 taskId / 扫码入口读取同一任务，数量使用 BOM 计划量和原单位。PDA 聚合按稳定 artifact 标识用领域 TASK-WATER 替换同源单工序 TASKGEN，不再承担隐藏整单/组合重复的职责。
- 未新增独立 PDA 菜单；列表继续只进入统一任务详情。任务 8 已接入独立水溶与含水溶染色的真实当前动作；任务 9 已将待交出动作接入既有通用交接闭环。
- 加工单会在读取与动作前对账当前物料级产物：新增、移除按 generationKey 收敛，既有加工单执行状态、数量、工厂和日志不会被重复读取重置。
- 分厂资格以正式工厂能力为准，同时校验工厂启用状态和派单资格；交出必须等于批准数量，继续补做按累计完成数量上报且不得倒退。
- 规格审查补充的倒序、同一步并行、直接确认、无共享物料、重新同步、取消水溶和快照隔离反例均已纳入专项检查。
- 代码质量审查补充的多组共享关系、首个染色不共享、拆分后单项染色并行元数据及连续承接反例均已纳入专项检查。
