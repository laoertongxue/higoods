| 类型契约原先仅由 Node 擦除类型后运行，存在假绿风险 | - | 技术包维护与后续开发 | 新增独立类型契约文件，由项目本地 `tsc --noEmit` 先编译，再执行运行时检查。 | 否 |
| 加工单与跨端执行链路不在当前范围 | - | 跟单、染厂主管、染厂一线员工 | 当前不生成加工单，也不宣称 FCS / PFOS / PDA 页面已实现；后续任务另行实现和复审。 | 否 |
| 不同 BOM 行分别选择水溶、染色时可能被误当成同一物料组合 | `选不对`、`追溯不足` | 技术资料维护人员 | 按工序逐行计算并覆盖 `linkedBomItemIds`，新增、编辑、删除都会重新联动。 | 否 |

| 规格审查发现预置倒序或同一步并行草稿可绕过页面动作保护直接确认 | `点错风险`、`追溯不足` | 技术资料维护人员 | 将共享 BOM 物料的“先水溶、后染色”约束下沉到通用路线归一化；确认、保存和运行时归一化统一生效。仅拆分共享水溶；染色仍有并行伙伴时保留原组语义，拆分后降为单项时清除失效并行标记。 | 否 |
| 代码审查发现多组与交叉并行组可能形成互换循环，达到次数上限后仍静默保留非法路线 | `点错风险`、`追溯不足` | 技术资料维护人员 | 先从含染色的混合组拆出参与共享 BOM 依赖的水溶子组，再按步骤组依赖图做稳定拓扑排序；返回前强校验全部共享 BOM 不变量，失败时明确阻断。增加多组、首个不共享、交叉组、仅两项并行及连续承接反例。 | 否 |
| legacy 与页面工序联合类型未直接继承正式技术包水溶字段 | `协作断裂` | 技术资料维护人员、后续生产单读取方 | legacy `TechPackProcessEntry` 与页面 `TechniqueItem` 直接引用 `TechnicalProcessEntry` 对应属性，专项 tsc 构造正式与 legacy 契约并检查页面单一来源。 | 否 |
| 生产单准备产物原先按工序汇总，可能把不同 BOM 行的水溶与染色要求误组合，且沿用成衣总数量 | `选不对`、`算不准`、`追溯不足` | 生产跟单、后续染厂主管 | 生产拆解改为读取生产单引用的正式快照，按生产单、快照、工序、BOM 行生成稳定产物；计划数量由适用 SKU 数量、BOM 单耗和损耗率自动计算并保留 BOM 单位。同一物料同时水溶和染色时只生成带固定路线的染色需求。 | 否 |
| BOM 产物 ID 只使用生产单固定快照号，切换正式技术包版本后可能错误复用旧产物身份 | `追溯不足`、`协作断裂` | 生产跟单、后续染厂主管 | BOM 产物版本键同时包含生产单快照号和正式技术包版本 ID，版本 ID 缺失时才回退版本编码或快照号；专项检查固定 snapshotId、只切换正式版本 A / B，锁定产物 ID 必须变化。 | 否 |

## 6. 最终结论

结论：通过

说明：

- 本阶段已完成 PCS 技术包 BOM 与工序预览；草稿只更新技术包工序预览，不生成加工单。
- 水溶按 BOM 物料触发，适用于所有 BOM 物料类型，目标对象不收窄为面料。
- 水溶归准备阶段、归属染厂、排序早于染色，默认生成任务单，不新增需求单、分厂、仓库、结算或批次能力。
- 规格审查发现的公共产值模板缺口已修复；水溶复用既有批次型模板 D，并已补齐专项运行时检查。
- BOM 与技术包工序类型契约已移入独立 `scripts/check-water-soluble-types.ts`，由项目本地 `tsc --noEmit` 真实编译，不再依赖 Node 的类型擦除结果。
- 所有 BOM 物料类型均可维护单位与水溶要求；同一物料同时水溶、染色时固定先水溶后染色。
- 正式技术包快照、运行时克隆和评审差异保留物料编码、单位、水溶要求及工序关联 BOM 行。
- FCS 生产单拆解已按正式快照中的具体 BOM 行生成准备产物：仅水溶生成独立水溶任务，仅染色生成普通染色需求，同一物料水溶加染色只生成带“先水溶、后染色”路线的染色需求，均未选择则不生成。
- 计划数量由系统按适用 SKU 生产数量 × BOM 单耗 ×（1 + 损耗率）计算，最多保留 3 位小数并保留 BOM 单位；产物 ID 包含生产单、正式快照、工序和 BOM 行，重复拆解不累加。
- BOM 产物 ID 已明确纳入正式技术包版本 ID；同一生产单固定 snapshotId 下切换正式版本会生成新身份，同 snapshotId 与同 version id 重复拆解保持幂等，legacy 非 BOM 产物 ID 不变。
- 规格审查补充的倒序、同一步并行、直接确认、无共享物料、重新同步、取消水溶和快照隔离反例均已纳入专项检查。
- 代码质量审查补充的多组共享关系、首个染色不共享、拆分后单项染色并行元数据及连续承接反例均已纳入专项检查。
- 质量复审补充的交叉并行组依赖反例已纳入专项检查，路线不再通过迭代互换和次数上限收敛。
- 本次只完成 FCS 拆解产物规则与数量口径，不宣称水溶领域加工单页面、FCS 页面、PFOS 页面或 PDA 页面已实现；后续现场执行链路仍需按角色、动作、防错与异常兜底另行复审。
