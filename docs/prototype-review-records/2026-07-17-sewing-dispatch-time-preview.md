# 车缝派单时效预览原型审查记录

## 1. 基本信息

| 项目 | 内容 |
| --- | --- |
| 审查日期 | 2026-07-17 |
| 相关需求 / 任务 | 车缝、车缝到后道在直接派单和改派时支持回填业务分配时间，并自动展示具体交付与回货时效 |
| 涉及系统 | FCS |
| 涉及页面路径 | `/fcs/dispatch/sewing`、`/fcs/dispatch/continuous` |
| 端类型 | 管理端 |
| 主要角色 | 生产计划员、跟单 |
| 主要任务 | 为独立车缝 SKU 或车缝到后道连续任务选择承接工厂，确认主工厂，并在提交前核对具体履约日期 |

## 2. 参考规范

- `docs/higood-indonesia-factory-product-design-guidelines.md`
- `docs/higood-indonesia-factory-prototype-review-checklist.md`

本次重点核对管理端信息密度、弹窗信息负荷、数量与时间自动计算、防错阻断、低分辨率可提交和交接事实口径。

## 3. 自查结论

| 检查项 | 结论 | 说明 |
| --- | --- | --- |
| 角色匹配 | 通过 | 页面面向生产计划员和跟单，保留工厂选择、主工厂确认和履约日期核对。 |
| 任务清晰度 | 通过 | 弹窗聚焦业务分配时间、承接工厂、主工厂、交付时效和回货时效。 |
| 信息架构与导航 | 通过 | 沿用车缝分配工作台和连续工序任务分配既有入口，未新增无关层级。 |
| 页面模式 | 通过 | 管理端采用列表加轻量弹窗，直接派单与改派表达一致。 |
| 信息负荷 | 通过 | 删除连续工序弹窗重复的分配数量卡片和实际操作时间；独立车缝不重复展示列表已有齐套与风险字段。 |
| 文案 | 通过 | 使用“业务分配时间”“交付完成”“30% / 70% / 100% 回货”等业务语言。 |
| 数量与状态 | 通过 | 独立车缝按完整 SKU 分配且一个 SKU 仅选一家工厂；改派按确认实收后的剩余数量计算。 |
| 扫码与识别 | 通过 | 本次为管理端派单场景，不涉及扫码。 |
| 防错 | 通过 | 业务分配时间晚于提交当前时间时展示明确错误并禁用确认；未选工厂时禁用确认。 |
| UI 样式 | 通过 | 时效区采用克制的单列日期清单，未增加装饰性卡片。 |
| 组件交互 | 通过 | 修改业务分配时间仅刷新时效预览槽；连续工序弹窗页头与页脚固定，正文独立滚动。 |
| 协作关系 | 通过 | 明确直接派单和改派后工厂自动接单，以业务分配时间启动时效。 |
| 异常与追溯 | 通过 | 实际操作时间仍由系统提交时记录但不在弹窗展示；履约只累计接收方确认实收。 |
| 现场设备可用性 | 通过 | 已检查 1366×768 和 1280×720；正文可滚动且取消、确认按钮保持可见。 |

## 4. 问题标签

- `字段过载`
- `算不准`
- `视觉干扰`
- `点错风险`

## 5. 主要问题与处理

| 问题 | 标签 | 影响角色 | 处理方式 | 是否仍有风险 |
| --- | --- | --- | --- | --- |
| 原弹窗重复展示列表已有齐套、风险和数量信息 | 字段过载、视觉干扰 | 生产计划员、跟单 | 收敛为分配动作必需信息，移除连续工序重复数量卡片 | 否 |
| 原时效只展示相对天数或部分节点，用户仍需换算日期 | 算不准 | 生产计划员、跟单 | 根据业务分配时间直接展示交付及 30%、70%、100% 的具体日期时间 | 否 |
| 未来业务分配时间可能形成错误履约起点 | 点错风险 | 生产计划员、跟单 | 提交前实时校验并禁用确认，提交逻辑再次校验 | 否 |
| 低分辨率下连续工序弹窗页脚会随正文滚出首屏 | 点错风险、组件误用 | 生产计划员、跟单 | 仅正文滚动，固定页头与操作页脚 | 否 |

## 6. 最终结论

结论：通过

说明：

- 车缝采用 9×24 小时交付及 4×24、8×24、9×24 小时回货节点。
- 车缝到后道采用 10×24 小时交付及 5×24、9×24、10×24 小时回货节点。
- 仅直接派单和改派展示本预览；发起竞价继续以工厂确认接单作为时效起点。
- 本次无规范例外。
