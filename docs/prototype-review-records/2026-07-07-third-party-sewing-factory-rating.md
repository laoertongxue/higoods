## 自查结论

- 角色边界清晰：PPIC 负责派单判断，主管负责评级解释，工厂管理在“三方工厂评级”集中查看评级，结算人员只被阻断黑名单结算。
- 导航边界清晰：评级信息已从工厂档案编辑抽屉移出，新增菜单“三方工厂评级”位于“工厂档案”之后、“工厂产能档案”之前；工厂档案只作为主档来源，不再承载评级卡片。
- 信息负荷可控：评级页使用标准列表页组件承载筛选、统计卡片、表格、分页和详情抽屉，不引入完整后台配置；统计卡片上方不再保留说明性标题文案。
- 首屏顺序明确：页面标题为“三方工厂评级”；桌面端筛选区保持一行展示，统计卡片位于筛选下方，并按当前筛选结果联动计算“全部三方车缝工厂、正常合作、考核中、黑名单、S/A/B/C 分布”。
- 筛选覆盖完整：支持工厂名称 / 编码、评级、合作状态、工厂规模、是否允许派单、是否允许结算，能从管理视角快速定位黄牌、黑名单和试单工厂。
- 交互可达：筛选表单使用 `data-nav-from-fields` 从当前字段构造稳定导航，提交后 URL 查询参数更新，联动统计和标准列表按筛选结果重算；列设置入口和分页上一页 / 下一页使用 `data-nav`，避免自定义事件未进入全局分发导致点击无效。
- 列表治理达标：三方工厂评级页声明 `@page-pattern: list`，使用 `renderStandardListPage`、`renderStandardListTable`、`renderTablePagination`，保留分页、排序、列显示、列冻结和右侧固定操作栏。
- 数量口径明确：试单上限区分 1000 件和 300 件，近 90 天只做生产时效查看。
- 防错明确：C 级/黑名单禁止派单和发起结算，B 级只做黄牌提醒。
- 2026-07-16 补充：车缝分配域函数已改为执行结构化派单评估结果，不再只按 `cooperationStatusLabel` 做硬编码拦截；考核中工厂仅允许试产单额度内派单，B 级黄牌需确认交期风险，主管指定工厂不能进入竞价，无评级快照在域函数层阻断。
- 2026-07-16 防错自查：直接绕过页面调用派单域函数时，黑名单/暂停、考核中常规单、黄牌未确认、主管指定竞价和无评级候选都会返回中文业务阻断文案；确认上下文只放行黄牌风险和主管指定场景，不放行无评级快照。
- 2026-07-16 复审修复：三方评级硬门禁已收敛到三方车缝评级治理对象，内部 / 非治理车缝工厂不因缺少评级快照被拦；考核中工厂按同一工厂本次派单合计数量校验首单上限；B 级黄牌无论是否小单、是否急单，都必须传入风险确认后才能派单。
- 2026-07-16 返工复审修复：派单策略数量改为读取真实执行的 `row.remainingQty` 合计，不再允许 `policyOverrideByRowId.dispatchQty` 低报、负数或异常值绕过试产首单上限；默认发起竞价不假装检查未知候选，定标入口 `awardRuntimeTaskTender` 对被定标三方车缝评级治理对象执行同一套派单评级策略，黑名单 / 禁止派单、主管指定不可竞价和黄牌未确认都会在定标前阻断，非三方普通车缝工厂不因缺少评级快照被误拦。
- 2026-07-16 页面接入复审：车缝分配工作台直接派单弹窗已消费统一派单评估结果，工厂下拉按当前 SKU 的生产单类型、急单标记和 `row.remainingQty` 展示可派、黄牌确认、主管指定确认或阻断信息；确认派单时只传递黄牌风险确认、主管指定确认、单据类型和急单上下文，不再由页面用合作状态硬编码拦截黑名单或考核中。
- 2026-07-16 页面缺评级返工：车缝分配工作台对三方车缝治理对象但缺少评级快照的工厂直接显示 `BLOCK` 阻断，文案提示“缺少三方评级快照，不能派单，请先完成评级”，不允许通过页面确认上下文放行。
- 中文状态完整：页面不展示 `TRIAL`、`ACTIVE`、`BLACKLISTED`。
- 补充核查：评级快照使用工厂主键 `ID-F02x`，同时保留工厂编码 `ID-FAC-002x` 查询兼容；黑名单工厂已纳入可演示的对账单生成范围，确保结算拦截能在页面自然触发。
- 补充覆盖：评级快照已按工厂档案主档覆盖全部 11 个三方车缝工厂，包含 `KOL-GOTO-001` 和 `ID-F026` 至 `ID-F030`。
- 补充串联：车缝分配候选改用工厂档案主档口径；对账单 mock 已覆盖 11 个三方车缝工厂，可合作工厂有历史对账单，黑名单/暂停工厂保留待生成候选用于拦截演示。
- 补充口径：结算主数据查询支持工厂主键和工厂编码互通，`KOL-GOTO-001` 等工厂档案 seed 也按三方车缝三旬结算生成对账周期。
- 对抗式核查：专项检查已覆盖数据完整性、页面可见性、交互可达性、业务闭环四类反例；黑名单工厂即使绕过页面直接调用车缝分配和对账单创建域函数，也会被域层拦截；同时检查工厂档案中不再残留旧评级面板。本轮补充浏览器反例验证，覆盖筛选联动、重置、详情抽屉、列设置和分页导航。
