### 1.1 上下游与责任边界

- 上游：生产单、工艺路线、车缝分配工作台、连续工序任务、工厂主数据。
- 下游：PDA 接单 / 执行 / 交出、接收方确认实收、生产进度与异常责任复核。
- 分配事实：独立车缝只进入车缝分配工作台；含车缝连续任务只进入连续工序任务页面。
- 接单事实：直接派单按业务分配时间自动接单；竞价定标后由中标工厂明确确认接单，确认时才启动时效。
- 数量事实：工厂交出不等于回货完成，只有接收方确认实收才计入履约；超量实收正常计入，履约比例允许超过 100%。
- 主工厂事实：一个生产单只有一个主工厂；只有一个有效车缝工厂时系统自动选择，多工厂时要求管理人员显式选择。

| 检查项 | 结论 | 说明 |

| 角色匹配 | 通过 | 管理端负责分配、改派和全链路追溯；主管端负责确认延迟责任复核；PDA 员工端只保留当前接单、执行、交出或确认动作。 |
| 任务清晰度 | 通过 | 独立车缝与连续任务入口分开；连续任务提供直接派单、发起竞价、暂不分配三个明确动作，不使用含糊的“整任务分配”。 |
| 信息架构与导航 | 通过 | 分配页、进度页、PDA 接单 / 执行 / 交出读取同一运行时任务、履约快照和实收事实。 |
| 页面模式 | 通过 | FCS 使用管理表格、节点信息和追溯抽屉；PDA 使用低密度卡片与单一主动作。 |
| 信息负荷 | 通过 | PDA 不展示管理统计与复杂责任算法；节点详情、受影响记录和复核动作放在管理 / 主管视图。 |
| 文案 | 通过 | 使用“直接派单”“发起竞价”“暂不分配”“确认接单”“确认实收”“主管复核责任”等具体动作；不向一线暴露英文状态码。 |
| 数量与状态 | 通过 | 系统自动计算分配量、已交、已确认实收、还差、履约比例和下一节点；101 件的 30% / 70% 取 31 / 71 件；超收比例不封顶。 |
| 扫码与识别 | 通过 | PDA 沿用任务 / 交接扫码入口；本次未改变扫码模型，确认页显示任务和数量供核对。 |
| 防错 | 通过 | 分配时间和操作时间不得晚于操作当前时间；竞价未确认接单不启动时效；重复接单、无定标接单、已完成后改派均阻断。 |
| UI 样式 | 通过 | 管理端信息密度较高但节点与异常突出；PDA 主动作突出、状态色克制，关键数量带“件”。 |
| 组件交互 | 通过 | 高频时间 / 工厂字段局部刷新，输入不触发整页重绘；弹窗关闭不丢滚动位置，提交前有节点或实收预览。 |
| 协作关系 | 通过 | 明确工厂交出、接收方实收、主管责任复核三方责任；进度只按确认实收累计。 |
| 异常与追溯 | 通过 | 截止前交出但截止后确认形成接收延迟证据，保留交出时间、确认时间、受影响数量、延迟时长和记录编号。 |
| 现场设备可用性 | 通过 | PDA 采用短文案、低密度、单主动作；关键弹窗在低分辨率可滚动提交；弱网场景保留已成功实收事实并提示后续同步。 |

## 5. 主要问题与处理

| 问题 | 标签 | 影响角色 | 处理方式 | 是否仍有风险 |
| --- | --- | --- | --- | --- |
| 独立车缝和连续工序任务入口边界容易混淆 | `协作断裂` | 生产计划员 / 跟单 | 车缝工作台排除连续任务，连续任务页单独承接分配动作。 | 否 |
| 业务分配时间可回填但可能误填未来时间 | `点错风险` | 生产计划员 / 跟单 | 统一以操作端本地墙钟校验，未来时间直接阻断且不产生部分状态。 | 否 |
| 竞价定标可能被误认为已经接单 | `状态抽象` | 工厂主管 | 定标后保持“待确认接单”，由工厂确认时创建履约快照并启动滚动 24 小时时钟。 | 否 |
| 工厂交出数量被误当作回货完成数量 | `协作断裂` | 跟单、工厂、接收方 | 只累计接收方确认实收；同时展示累计交出与累计已确认实收。 | 否 |
| 接收方晚确认导致工厂看似逾期 | `缺主管兜底` | 平台主管 | 仍按真实确认时间计算，同时展示接收延迟记录并提供“主管复核责任”。 | 否 |
| 改派后旧实收和新任务时钟可能混用 | `追溯不足` | 跟单、平台主管 | 旧快照失效但保留历史；新任务按剩余未实收数量创建独立快照，旧任务后续实收不污染新快照。 | 否 |
| 多家车缝工厂时主工厂归属不明确 | `选不对` | 生产计划员 / 跟单 | 单厂自动选择；多厂要求显式选择且全程保持唯一主工厂。 | 否 |
| 弱网下接收事实与仓储联动可能不同步 | `协作断裂` | 接收方仓管、主管 | 已成功的实收事实不回滚，页面提示仓储联动待同步并保留主管处理入口。 | 否 |

## 6. Mock 与边界验收

- 独立车缝：30%、70%、100% 三节点均按时。
- 车缝到包装：30% 逾期、70% 追上、最终按时。
- 裁片到包装：30% 按时、70% 逾期、最终逾期完成。
- 最终截止后未完成仍显示逾期待达标。
- 截止前交出、截止后确认形成接收方确认延迟。
- 101 件的 30% / 70% / 100% 目标为 31 / 71 / 101 件。
- 超量实收正常完成，履约比例可以超过 100%。
- 部分实收后改派，新旧快照、剩余数量和后续实收互相隔离。
- 单车缝工厂自动成为主工厂，多车缝工厂必须显式选择。

## 7. 现场与交互专项结论

- 弱网：确认实收成功后，后续仓储联动失败不回滚现场事实，并给出可恢复提示。
- 低分辨率：PDA 首屏保持低密度与单主动作；管理端弹窗主体可滚动，底部动作可达。
- 局部交互：时间、工厂、数量等高频输入只刷新预览 / 错误区域，不整页重绘，不丢输入焦点或滚动位置。
- 性能：关键弹窗打开、字段校验、预览切换和本地 Mock 提交均以 200ms 内响应为验收标准。

## 8. 最终结论

结论：通过
