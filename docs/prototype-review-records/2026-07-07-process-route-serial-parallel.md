## 5. 主要问题与处理

| 问题 | 标签 | 影响角色 | 处理方式 | 是否仍有风险 |
| --- | --- | --- | --- | --- |
| 基础路线顺序可能被误解为最终工艺路线 | 读不懂 | 跟单、PPIC | 字典页说明其仅为默认参考，技术包工序 Tab 以款式级确认为准 | 否 |
| 并行组承接口径可能影响后续连续工序合并理解 | 状态抽象 | 跟单、PPIC | 页面显示“分别承接 / 并行组整体承接”，并说明其只影响连续工序任务合并资格 | 否 |
| 未确认路线继续流入发布会造成生产任务口径不稳定 | 协作断裂 | 跟单、PPIC | 跟单审核和正式技术包发布均增加路线确认门禁 | 否 |
| 生产单创建后继续读取技术包草稿会造成任务顺序漂移 | 追溯不足 | PPIC、跟单 | 生产单冻结正式且已确认路线，任务产物排序和 ProcessTask 依赖均读取冻结字段 | 否 |
| 任务清单按旧依赖 / seq 误合并非冻结路线任务 | 状态抽象 | PPIC、跟单 | 合并判断改为统一只读校验，必须读取冻结路线步骤 / 并行线，缺步骤或缺并行组完整选择时给出中文原因 | 否 |
| 连续工序任务被当成明细拆分任务再次分配 | 点错风险 | PPIC、跟单 | 合并任务固定为整任务粒度，压制明细拆分配置，连续分配页只提供整任务分配 / 暂不分配 | 否 |
| 含裁片连续任务误生成我方加工单或遗漏三方上报口径 | 协作断裂 | PPIC、跟单、三方工厂 | 分配页在其他连续工序任务中说明我方裁床提供唛架方案，三方上报裁片完成数量和可做成衣数，不生成我方加工单 | 否 |

## 6. 最终结论

结论：通过

说明：

- 第 1 批完成数据模型、路线纯函数、字典基础路线顺序和字典页展示。
- 第 2 批完成技术包工序 Tab 的款式级路线编辑、并行组设置、路线确认、跟单审核门禁和正式发布门禁。
- 第 3 批完成技术包版本克隆 / 保存的路线保留，生产单冻结正式且已确认路线，任务产物按冻结路线排序，ProcessTask 写入冻结路线并按路线步骤生成依赖。
- 第 4 批完成连续任务人工合并判断与连续工序分配页面：合并只基于冻结路线，连续任务不允许按明细拆分，分配页只展示任务清单人工合并任务。
