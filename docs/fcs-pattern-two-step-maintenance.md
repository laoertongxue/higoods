# FCS 纸样管理两步维护

## 改造范围

纸样管理拆成两个维护步骤，避免跟单和版师字段混在一个大表单里。

1. 跟单基础信息：纸样名称、纸样类型、是否针织、门幅（cm）、关联物料。
2. 版师技术信息：排料长度（m）、纸样 PRJ 文件、唛架图片、DXF 文件、RUL 文件、裁片明细。

原图片字段统一改为“唛架图片”，表示版师上传的唛架预览图。代码内部可保留兼容字段，但页面、表单、详情和快照展示统一使用“唛架图片”。

## 字段口径

- 纸样 PRJ 文件：版师维护，必填，只记录 .prj / .PRJ 文件信息，不解析内容。
- 唛架图片：版师维护，必填，允许 png、jpg、jpeg、webp。
- DXF 文件、RUL 文件：继续沿用现有上传与解析能力，本次不重构解析算法。
- 关联物料：来自当前技术包物料清单，保存 linkedMaterialId、linkedMaterialName、linkedMaterialSku。
- 维护状态：维护 maintainerStepStatus、merchandiserInfoStatus、patternMakerInfoStatus。

## 快照范围

技术包保存或发布时，纸样快照保留：

- 纸样名称、纸样类型、是否针织、门幅（cm）、关联物料
- 排料长度（m）、纸样 PRJ 文件、唛架图片、DXF 文件、RUL 文件
- 裁片明细、维护状态

## 本次不处理

- 不做捆条多条维护。
- 不改裁片明细片数列。
- 不做逐片特殊工艺配置。
- 不做菲票逐片生成。
- 不重构 DXF/RUL 解析逻辑。

## 流程图

打开纸样管理
-> 新增纸样
-> 跟单填写基础信息
-> 保存并进入版师技术信息
-> 版师填写排料长度
-> 上传纸样PRJ文件
-> 上传唛架图片
-> 上传DXF和RUL文件
-> 解析裁片明细
-> 保存纸样
-> 技术包版本快照保留纸样信息

## 状态机

待跟单维护
-> 待版师维护
-> 待解析
-> 已解析待确认
-> 已完成

异常流转：

字段校验失败
-> 停留当前步骤
-> 显示中文错误提示
-> 修改后重新保存
