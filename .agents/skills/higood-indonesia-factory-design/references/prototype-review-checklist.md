# Prototype Review Checklist Reference

Use this reference only as a navigation aid. The authoritative file is:

`docs/higood-indonesia-factory-prototype-review-checklist.md`

## Review Flow

1. Fill page basics: system, path, role,端类型, upstream, downstream.
2. Check role fit and task clarity first.
3. Check information architecture, page mode, and information load.
4. Check copy, quantity/status expression, scan/image recognition, and error prevention.
5. Check UI style, component interaction, collaboration relationship, abnormal traceability, and device usability.
6. Record conclusion: 通过 / 有条件通过 / 不通过.
