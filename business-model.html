三、关键业务实体详解

📐 技术包 TechPack 核心引擎

技术包是生产最核心的"配方"，定义了"用什么材料、按什么纸样、经过什么工序、做成什么尺码"。

数据域内容用途
纸样文件 PatternFiles 裁片部位（前片/后片/领口/袖口…）、尺码范围、颜色分配、门幅(cm)、排料长度(m)、捆条配置 确定裁单应该裁哪些部位，门幅决定唛架排布
BOM物料清单 BomItems 面料/辅料名称、规格、单位消耗量、损耗率、印花/染色要求、适用的SKU范围 确定用什么料、用多少、是否需特殊处理
颜色-物料映射 ColorMaterialMappings 将具体颜色绑定到具体物料SKU、关联纸样ID和裁片部位、每单位片数 裁单生成的直接依据：每条映射产生一条裁单
工艺工序 ProcessEntries 工序代码、工艺代码、标准工时(SAM)、是否特种工艺、派单粒度(整单/颜色/SKU/明细) 确定生产任务如何拆解和派单
尺码表 SizeTable 每个SKU的S/M/L/XL等尺码的具体测量值 唛架尺码配比的依据
📦 生产单 ProductionOrder 枢纽实体

生产单是连接需求、技术包、工厂、任务的核心枢纽。

stateDiagram-v2 [*] --> DRAFT: 创建 DRAFT --> WAIT_TECH_PACK_RELEASE: 技术包未就绪 WAIT_TECH_PACK_RELEASE --> READY_FOR_BREAKDOWN: 技术包发布 READY_FOR_BREAKDOWN --> WAIT_ASSIGNMENT: 任务待拆解 WAIT_ASSIGNMENT --> ASSIGNING: 开始派单 ASSIGNING --> EXECUTING: 工厂确认开工 EXECUTING --> COMPLETED: 全部完成 ASSIGNING --> ON_HOLD: 挂起 EXECUTING --> ON_HOLD: 挂起 ON_HOLD --> ASSIGNING: 恢复 ON_HOLD --> EXECUTING: 恢复 DRAFT --> CANCELLED: 取消 WAIT_TECH_PACK_RELEASE --> CANCELLED: 取消

核心快照机制：创建生产单时，技术包被"冻结"为 ProductionOrderTechPackSnapshot，确保生产过程中技术包即使后续更新也不会影响已开工的生产单。快照包含 BOM、纸样、工序、尺码、颜色映射、裁片部位的完整副本。

✂️ 裁单 GeneratedCutOrder 派生实体

裁单从生产单自动派生，每一条 = 一个(物料SKU + 纸样文件 + 门幅)的组合。

关键关联：

cutOrderId → productionOrderId（来源生产单）

materialSku + materialColor + materialType → BOM物料

patternFileId + patternVersion + effectiveWidthText → 纸样+门幅

markerPlanId → 唛架（初始为空，排唛架后回写）

物料类型 materialType： PRINT 印花料 DYE 染色料 SOLID 本色料 LINING 里料

🗺️ 唛架 MarkerPlan 分组容器

唛架是一个分组概念，将同SPU下多个裁单按纸样分组编排到一个排料方案中。

关键关联：

markerPlanId → sourceCutOrderIds[]（多个裁单共用同一个唛架方案）

markerPlanId → sourceProductionOrderIds[]（来自哪些生产单）

唛架信息（反映在裁片单）：尺码配比(sizeMix)、总片数(totalPieces)、净长(netLength)、每片消耗(perPieceConsumption)

八、CuttingCoreRegistry — 内存中的多维度索引

这是裁床域最核心的数据结构。它将生产单、裁单、唛架、裁片任务、PDA执行全部加载到内存中，建立双向索引，实现 O(1) 的关联查询。

markerPlanIds[]"| C C -->|"sourceCutOrderIds[]

反向查询能力：给定一个裁单ID，可以通过 Registry 查到它属于哪个生产单、关联哪些唛架方案、对应的 PDA 执行记录。给定一个生产单ID，可以查到下属所有裁单、所有唛架、所有 PDA 执行。
